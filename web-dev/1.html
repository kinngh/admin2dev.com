Introduction au HTML

Qu'est-ce que le HTML

HTML ou Hyper Text Markup Language est le squelette d'Internet. Le HTML est extrêmement simple à apprendre et met à l'échelle la complexité des exigences les plus complexes que nous avons. Tout comme Apex, HTML a sa propre syntaxe pour tout, mais contrairement à Apex, il n'y a pas d'instructions if-else, il n'y a pas de boucles, il n'y a pas d'opération "Logicale" dans HTML. Le HTML est, en termes trop simplifiés, un moyen de formater un document en y ajoutant des attributs.

A HTML file ends with the extension .html, and uses tags within < >and </ > to identify how to display the content between the tags.

Écrire du HTML

First, we need to define the .html file to be a HTML5 file. For this, we start off using <html>. In your text editor or IDE of choice (I'll be using Atom), create a new file and name it index.html and add the following content:

<html>
</html>

A thing to remember is, when you upload your site to a server, the server will ALWAYS look for a file named index.extension where extension could be .html, .php, .asp, etc. This is why all the first pages in every folder we create is called index.html. I spent an entire day when I was learning HTML back in 4th grade figuring out why the extremely expensive server won't display my page at harshdeephura.com but harshdeephura.com/about.html worked just fine.

Maintenant que nous avons défini le début et la fin de notre dossier ! Maintenant, nous divisons le fichier HTML en deux sections :

  • Tête
  • Corps

Tête

La section principale du HTML contient tous les préchargeurs comme CSS, les balises SEO, le titre de la page Web et les scripts d'analyse (nous parlons de scripts dans le cours JavaScript). Ajoutons une tête à notre fichier HTML et donnons-lui un titre.

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
</html>

Save and open this in your browser. You'll notice the title says Harshdeep's Portfolio(or the text you chose to write)!

Corps

Le corps se charge après la balise head et contient tout le contenu que l'utilisateur verra. Pour écrire du texte dans notre corps, nous utilisons la balise thep, qui signifie paragraphe. Bien que vous puissiez techniquement écrire du texte directement dans la balise body et qu'il affiche toujours le texte, cela créera des problèmes lorsque nous formaterons notre texte comme l'ajout de sauts de ligne et d'attributs. Ajoutons une étiquette corporelle et écrivons notre nom !

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
	</body>
</html>

Great! Now to add a new paragraph, we can simply create more p tags! Let's write about how many Salesforce certifications we have like every cool kid on LinkedIn!

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>
	</body>
</html>

Maintenant, écrivons une ligne sur nous-mêmes. Peut-être d'où nous sommes, ce que nous faisons quand nous avons du temps libre, où nous travaillons, n'importe quoi ! ou vous pouvez Google "intérêts qui ont l'air bons sur le CV" parce qu' soyons réels, aucun humain normal ne peut se souvenir de leurs intérêts lorsqu'on le demande explicitement. Mais cette fois, assurez-vous d'ajouter plusieurs espaces entre vos lignes. Regardez ce code par exemple :

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.
		I also binge TV Shows on Netflix instead of being a functioning member of the society.
		I also really REALLY love dogs!</p>
	</body>
</html>

Notice how all the lines were converted to a single line of text? This is because HTML doesn't care about how you format your text as long as it is between a start and end tag. To add a new line, we use a br tag called as break. Unlike most tags, br doesn't have a closing tag.

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.<br>
		I also binge TV Shows on Netflix instead of being a functioning member of the society.<br />
		I also really REALLY love dogs!</p>
	</body>
</html>

Notice how I wrote br first as <br> and then as <br /> but they yield the same result? Because these tags don't require a separate closing tag, you can choose to write it either ways and HTML still accepts it like a loving mother.

Mise en forme du texte

Maintenant, je veux vraiment parler de mon amour pour les chiens. Les majuscules ne rendent VRAIMENT pas assez justice, nous allons donc ajouter quelques italiques, soulignés et gras au texte !

Mise en forme Étiquette
BOLD <strong>...</strong>
Italique <em>...</em>
Souligner <u>...</u>
Grève <strike>...</strike>

Ajoutons maintenant cela à notre code !

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.<br>
		I also binge TV Shows on Netflix instead of being a functioning member of the society.<br />
		I also really <strong>REALLY</strong> love dogs!</p>
	</body>
</html>

Hmm. Je pense toujours qu'il ne rend pas assez justice. Ajoutons plus de formatage !

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.<br>
		I also binge TV Shows on Netflix instead of being a functioning member of the society.<br />
		I also really <strong><u><em>REALLY</em></u></strong> love dogs!</p>
	</body>
</html>

There we go! Notice how to add multiple formatting options the tags are nested. It's a good practice to close the inner tags first and then the outer tags. We can further manipulate it, by underlining the whole REALLY love dogs! sentence.

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.<br>
		I also binge TV Shows on Netflix instead of being a functioning member of the society.<br />
		I also really <strong><u><em>REALLY</em></strong> love dogs!</u></p>
	</body>
</html>

Pour mieux comprendre les étiquettes d'imbrication, nous pouvons la comparer à la fabrication de café instantané. Peu importe l'ordre dans lequel vous mettez votre café instantané, votre sucre, votre eau et votre lait, cela fera toujours exactement le même café instantané, mais en même temps votre contenu doit être écrit dans le bon ordre, de la même manière que les céréales vont d'abord, puis le lait.

Rubriques

Bien que j'aime vraiment les chiens, je ne veux pas que mon amour pour les chiens vole le spectacle ! Mettons un peu plus l'accent sur le nom. Je veux que mon nom ait une taille de police plus grande. Pour cela, nous utiliserons la balise En-tête.

La balise d'en-tête est écrite avec `h` suivi d'un nombre de 1 à 6, 1 étant le plus grand et 6 étant le plus petit. Mettons à jour notre code.

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<h1>Harshdeep Singh Hura</h1>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.<br>
		I also binge TV Shows on Netflix instead of being a functioning member of the society.<br />
		I also really <strong><u><em>REALLY</em></strong> love dogs!</u></p>
	</body>
</html>

Remarquez comment j'ai remplacé la balise « p » par « h1 ». Il n'est pas nécessaire d'avoir des balises « p » autour d'un « h# » parce qu'elles commencent toujours par une nouvelle ligne.

Résumé

  • The html tag defines the start and end of our file. This tells the browser to render everything inside it using the HTML language.
  • The head tag is where our pre-loaders go, like our CSS and SEO tags
    • The title tag defines the title of this web page.
  • The body tag is now loaded
    • The h1 tag defines a heading
    • The p tag defines a new paragraph
    • The br tells the browser to start a new line
    • The strong is to bold the text
    • The u is to underline the text
    • The em is to italicize the text
      • Nous imbriquons des balises pour combiner les effets
      • The u tag is extended to underline more text
