Introducción al HTML

¿Qué es HTML?

HTML o Hyper Text Markup Language es el marco esquelético de Internet. HTML es extremadamente simple de aprender y escala en complejidad los requisitos más complejos que tenemos. Al igual que Apex, HTML tiene su propia sintaxis para todo, pero a diferencia de Apex, no hay instrucciones if-else, no hay bucles, no hay operación "lógica" dentro de HTML. HTML es, en términos demasiado simplificados, una forma de dar formato a un documento añadiéndole atributos.

A HTML file ends with the extension .html, and uses tags within < >and </ > to identify how to display the content between the tags.

Escritura de HTML

First, we need to define the .html file to be a HTML5 file. For this, we start off using <html>. In your text editor or IDE of choice (I'll be using Atom), create a new file and name it index.html and add the following content:

<html>
</html>

A thing to remember is, when you upload your site to a server, the server will ALWAYS look for a file named index.extension where extension could be .html, .php, .asp, etc. This is why all the first pages in every folder we create is called index.html. I spent an entire day when I was learning HTML back in 4th grade figuring out why the extremely expensive server won't display my page at harshdeephura.com but harshdeephura.com/about.html worked just fine.

¡Ahora que hemos definido el inicio y el final de nuestro archivo! Ahora, dividimos el archivo HTML en dos divisiones:

  • Cabeza
  • Cuerpo

Cabeza

La sección principal del HTML contiene todos los precargadores como CSS, etiquetas SEO, título de la página web y scripts de análisis (hablamos de scripts en el curso JavaScript). Añadamos un cabezal a nuestro archivo HTML y le demos un título.

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
</html>

Save and open this in your browser. You'll notice the title says Harshdeep's Portfolio(or the text you chose to write)!

Cuerpo

El cuerpo se carga después de la etiqueta principal y contiene todo el contenido que el usuario verá. Para escribir texto en nuestro cuerpo, usamos la etiqueta thep, que significa párrafo. Si bien técnicamente puede escribir texto directamente en la etiqueta de cuerpo y seguirá mostrando el texto, creará problemas cuando formateemos nuestro texto, como agregar saltos de línea y atributos. ¡Añadamos una etiqueta corporal y escribamos nuestro nombre!

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
	</body>
</html>

¡Genial! ¡Ahora para agregar un nuevo párrafo, simplemente podemos crear más etiquetas p! ¡Escribamos sobre cuántas certificaciones de Salesforce tenemos como cada chico genial en LinkedIn!

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>
	</body>
</html>

Ahora escribamos una línea sobre nosotros mismos. ¡Tal vez de dónde somos, qué hacemos cuando tenemos tiempo libre, dónde trabajamos, cualquier cosa! o puedes buscar en Google "intereses que se ven bien en el currículum vitae" porque seamos reales, ningún ser humano normal puede recordar sus intereses cuando se le pide explícitamente. Pero esta vez, asegúrese de agregar múltiples espacios entre sus líneas. Mira este código, por ejemplo:

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.
		I also binge TV Shows on Netflix instead of being a functioning member of the society.
		I also really REALLY love dogs!</p>
	</body>
</html>

Notice how all the lines were converted to a single line of text? This is because HTML doesn't care about how you format your text as long as it is between a start and end tag. To add a new line, we use a br tag called as break. Unlike most tags, br doesn't have a closing tag.

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.<br>
		I also binge TV Shows on Netflix instead of being a functioning member of the society.<br />
		I also really REALLY love dogs!</p>
	</body>
</html>

Notice how I wrote br first as <br> and then as <br /> but they yield the same result? Because these tags don't require a separate closing tag, you can choose to write it either ways and HTML still accepts it like a loving mother.

Formato de texto

Ahora realmente quiero hablar de mi amor por los perros. Las mayúsculas REALMENTE no hacen suficiente justicia, ¡así que añadiremos algunas cursivas, subrayaremos y negrita al texto!

Formato Etiqueta
AUDAD <strong>...</strong>
Cursiva <em>...</em>
Subrayado <u>...</u>
Huelga <strike>...</strike>

¡Ahora agreguemos esto a nuestro código!

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.<br>
		I also binge TV Shows on Netflix instead of being a functioning member of the society.<br />
		I also really <strong>REALLY</strong> love dogs!</p>
	</body>
</html>

Hmm. Todavía creo que no hace suficiente justicia. ¡Añadamos más formato!

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.<br>
		I also binge TV Shows on Netflix instead of being a functioning member of the society.<br />
		I also really <strong><u><em>REALLY</em></u></strong> love dogs!</p>
	</body>
</html>

There we go! Notice how to add multiple formatting options the tags are nested. It's a good practice to close the inner tags first and then the outer tags. We can further manipulate it, by underlining the whole REALLY love dogs! sentence.

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<p>Harshdeep Singh Hura</p>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.<br>
		I also binge TV Shows on Netflix instead of being a functioning member of the society.<br />
		I also really <strong><u><em>REALLY</em></strong> love dogs!</u></p>
	</body>
</html>

Para entender mejor las etiquetas de anidación, podemos compararlas con hacer café instantáneo. No importa en qué orden pongas tu café instantáneo, azúcar, agua y leche, seguirá haciendo exactamente el mismo café instantáneo, pero al mismo tiempo tu contenido tiene que escribirse en el orden correcto, de la misma manera que el cereal va primero y luego la leche.

Títulos

Aunque realmente amo a los perros, ¡no quiero que mi amor por los perros robe el espectáculo! Pongamos un poco de énfasis adicional en el nombre. Quiero que mi nombre tenga un tamaño de fuente más grande. Para esto, usaremos la etiqueta Heading.

La etiqueta de encabezado está escrita con `h` seguida de un número del 1 al 6, 1 siendo el más grande y 6 siendo el más pequeño. Actualicemos nuestro código.

<html>
	<head>
		<title>Harshdeep's Portfolio</title>
	</head>
	<body>
		<h1>Harshdeep Singh Hura</h1>
		<p>1x Salesforce Certified</p>

		<p>I build iOS apps.<br>
		I also binge TV Shows on Netflix instead of being a functioning member of the society.<br />
		I also really <strong><u><em>REALLY</em></strong> love dogs!</u></p>
	</body>
</html>

Observe cómo reemplacé la etiqueta `p` por `h1`. No es necesario tener etiquetas `p` alrededor de una `h#` porque siempre comienzan en una nueva línea.

Resumen

  • The html tag defines the start and end of our file. This tells the browser to render everything inside it using the HTML language.
  • The head tag is where our pre-loaders go, like our CSS and SEO tags
    • The title tag defines the title of this web page.
  • The body tag is now loaded
    • The h1 tag defines a heading
    • The p tag defines a new paragraph
    • The br tells the browser to start a new line
    • The strong is to bold the text
    • The u is to underline the text
    • The em is to italicize the text
      • Anidamos etiquetas para combinar efectos
      • The u tag is extended to underline more text
