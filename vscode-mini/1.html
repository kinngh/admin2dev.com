Configuración de VSCode para el desarrollo

Tome este escenario. Acabas de generar este interés en el café y ahora quieres expandirte de tu café instantáneo en polvo y leche a algo más serio. Te conectas, buscas cervecerías de café locales y programas un programa de capacitación de café con ellas. Una semana después, tienes un sofisticado proceso de elaboración de café que es único para ti. Su café se hace en una cámara de vacío para que obtenga hasta la última gota de café de sus granos antes de revolverlo en el trago perfecto de espresso.

Su conjunto de herramientas importa. Seguro que la consola del desarrollador es una gran mezcla instantánea de café, necesitas algo más potente porque la consola del desarrollador solo puede llevarte hasta cierto punto. Cuando empiezas a crecer como desarrollador, anhelas ese lugar perfecto para construir y depurar tus programas y este mini curso te presenta tu cámara de vacío para preparar tu café.

Código de Visual Studio

Visual Studio Code, o VSCode, es un IDE construido y publicado por Microsoft y está disponible de forma gratuita. Lo bueno de VSCode es que es realmente personalizable y la industria está adoptando rápidamente VSCode como un IDE estándar fuera de la consola de desarrolladores. Ahora, configuremos VSCode.

En primer lugar, necesitamos descargar JDK o Java Development Kit del sitio web de Oracle disponible aquí. Incluso si tiene JDK instalado, instálelo de nuevo. Esto es para asegurarte de que siempre estés en la última versión del software y realmente, es más fácil de esa manera. He visto a la gente encontrarse con problemas y no ser capaz de resolver las cosas durante semanas a la vez, solo para luego reiniciar completamente su sistema y arrancar sus máquinas desde cero.

Ahora descargue VSCode desde el sitio web de su Visual Studio para su plataforma e instálelo y ábralo.

The first thing to do after you install and open it, is to go in Settings and in Commonly Used section, change Auto Save to any option you prefer. I have kept it on afterDelay set to 10000 so I don't have to worry about saving the file I am working on. The delay is in ms where 1 second = 1000 ms, so autosaving my file every 10 seconds works well.

Now open the extensions section and search for Salesforce Extension Pack and install it.

Mientras eso se instala, necesitamos herramientas CLI de Salesforce en nuestra máquina. Diríjase al sitio web de la CLI de Salesforce e instálelo para su plataforma.

Una vez hecho esto, mostrará una ventana emergente en la parte inferior de la pantalla.

Restart your computer. I've noticed this with multiple instances that developers don't restart their computer for days and run into errors, and yes, I've helped debug people from our Discord community with this, where the solution was simply, restart your computer. After your restart, open VSCode and press Command + Shift + P for macOS users or Control + Shift + P for Windows users and type SFDX. If you see commands popping up, you have correctly configured everything!
