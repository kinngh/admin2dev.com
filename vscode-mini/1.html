Einrichten von VSCode für die Entwicklung

Nehmen Sie dieses Szenario. Sie haben gerade dieses Interesse an Kaffee aufgebaut und möchten jetzt von Ihrem Instant-Kaffeepulver und Ihrer Milch zu etwas Ernsterem werden. Sie gehen online, suchen nach lokalen Kaffeebrauereien und planen mit ihnen ein Kaffeetrainingsprogramm. Eine Woche später haben Sie einen ausgeklügelten Kaffeeherstellungsprozess, der für Sie einzigartig ist. Ihr Kaffee wird in einer Vakuumkammer hergestellt, so dass Sie jeden letzten Tropfen Kaffee aus Ihren Bohnen erhalten, bevor Sie ihn in den perfekten Schuss Espresso aufrühren.

Ihr Toolset ist wichtig. Sicher, dass die Entwicklerkonsole ein großartiger Instant-Kaffeemix ist, Sie brauchen etwas Mächtigeres, weil die Entwicklerkonsole Sie nur so weit bringen kann. Wenn Sie als Entwickler wachsen, sehnen Sie sich nach dem perfekten Ort, um Ihre Programme zu bauen und zu debuggen, und dieser Minikurs führt Sie in Ihre Vakuumkammer ein, um Ihren Kaffee zu brauen.

Visual Studio-Code

Visual Studio Code, oder VSCode, ist eine von Microsoft erstellte und veröffentlichte IDE, die kostenlos verfügbar ist. Das Tolle an VSCode ist, dass es wirklich anpassbar ist und die Branche VSCode schnell als Standard-IDE außerhalb der Entwicklerkonsole annimmt. Richten wir nun VSCode ein.

Zuerst müssen wir JDK oder Java Development Kit von der hier verfügbaren Website von Oracle herunterladen. Auch wenn Sie JDK installiert haben, installieren Sie es erneut. Dies soll sicherstellen, dass Sie immer auf der neuesten Version der Software sind, und wirklich ist es einfach einfacher. Ich habe gesehen, wie Menschen auf Probleme gestoßen sind und wochenlang nicht in der Lage sind, die Dinge herauszufinden, nur um dann ihr System vollständig zurückzusetzen und ihre Maschinen von Grund auf neu zu starten.

Laden Sie nun VSCode von der Visual Studio-Website für Ihre Plattform herunter und installieren und öffnen Sie es.

The first thing to do after you install and open it, is to go in Settings and in Commonly Used section, change Auto Save to any option you prefer. I have kept it on afterDelay set to 10000 so I don't have to worry about saving the file I am working on. The delay is in ms where 1 second = 1000 ms, so autosaving my file every 10 seconds works well.

Now open the extensions section and search for Salesforce Extension Pack and install it.

Während der Installation benötigen wir Salesforce CLI-Tools auf unserem Computer. Besuchen Sie die Salesforce CLI Website und installieren Sie sie für Ihre Plattform.

Sobald Sie fertig sind, wird unten rechts neben dem Bildschirm ein Pop-up angezeigt.

Restart your computer. I've noticed this with multiple instances that developers don't restart their computer for days and run into errors, and yes, I've helped debug people from our Discord community with this, where the solution was simply, restart your computer. After your restart, open VSCode and press Command + Shift + P for macOS users or Control + Shift + P for Windows users and type SFDX. If you see commands popping up, you have correctly configured everything!
