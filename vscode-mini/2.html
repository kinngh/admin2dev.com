Explorer

L'Explorateur contient une liste de tous les répertoires et fichiers de votre projet. Bien que nous l'apprenions dans les prochains chapitres, voici une capture d'écran de l'apparence de votre écran lorsque vous aurez un projet ouvert, avec un modèle testClass.

  • Open Editors shows all currently open files. In the screenshot, we have our actual testClass.cls file that contains the testClass class, and testClass.cls-meta.xml that contains information about the test class, that VSCode auto generates. This is a SOAP API that is used to transfer our code to the org we are working in, and is handled automatically.
  • Test Org est le nom que j'ai donné à mon projet que je viens de créer. Nous passons en revue cela lorsque nous discuterons de la connexion de VSCode à notre organisation et nous serons clairs dans les chapitres à venir. Sachez simplement que chaque projet que vous créez ajoutera une nouvelle sous-section avec le nom.
  • Outline contient toutes les classes et tous les objets que nous avons créés.
  • La chronologie contient un historique de tous nos commits git et de nos exécutions de code.
  • Exécution des tâches répertorie toutes les tâches en cours d'exécution.

Another really great feature in VSCode is the mini map available on the right side. This is handy when you have a file that contains a lot of code and having a mini map allows you to navigate more quickly. Let's populate the testClass with random methods and see it in action:

Rechercher

La recherche est exactement ce que vous attendez d'elle, une barre de recherche à gauche, tapez votre requête, appuyez sur Entrée et vous avez des résultats disponibles.

Contrôle à la source

Le contrôle des sources est destiné à versionner nos systèmes de travail et de fichiers. Il utilise git et dispose d'un accès intégré pour publier directement sur GitHub. Nous allons plus loin dans un mini-cours séparé Git et Collaboration Mini-cours qui explique ce qu'est git, la configuration et comment collaborer avec d'autres utilisateurs à l'aide de git. Pour l'instant, nous ignorons cela de manière compétitive parce que c'est hors de portée pour ce mini-cours.

Courir

This is used to run our code based on configurations. For most parts, we will be using the SFDX: Execute Anonymous Apex with Editor contents to run our code.

Extensions

Extensions est l'endroit où nous obtenons nos modules complémentaires pour VSCode. C'est d'où nous avons obtenu nos packages Salesforce.

Réglages

All your settings live here. This is also where you can select theme for VSCode. I am using the Dark+ (default dark) theme. This only changes your color scheme, so it's safe to play around with. You can get more themes from extensions panel by searching for category:themes or heading over to my recommended VSCode Theme website, VSCodeThemes. (Not affiliated in any way.), or you can search the internet for more VSCode themes if you like.
