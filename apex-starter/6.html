OOPs, Klassen und Methoden

Objektorientierte Programmierung

Apex ist eine objektorientierte Programmiersprache (OOP), ein Programmierparadigma, bei dem Daten in Objekten und Code in Prozeduren (Methoden) gespeichert werden. Die Idee von OOP ist es, Daten und Funktionen, die mit den Daten arbeiten, miteinander zu verbinden, um die Zugriffskontrolle durch Klassen aufrechtzuerhalten.

OOP bringt eine ganze Reihe von Programmierkonzepten mit. Hier sind einige Konzepte, die wir häufiger als andere verwenden werden:

  • Vererbung: Dadurch können Sie Methoden von der übergeordneten Klasse "erben". Das ist ähnlich wie Kinder zu haben. Das Kind könnte grüne Augen von ihrer Mutter und blonde Haare von ihrem Vater haben. Wir werden die Vererbung später in diesem Kurs ausführlich diskutieren, da sie wirklich kompliziert werden kann und einen ganzen Tag gewidmet werden muss.
  • Kapselung: Das bedeutet, dass die Variablen innerhalb einer Klasse immer ausgeblendet sind und nur mit einer Methode aufgerufen werden können. Ein Beispiel wäre, dass Sie einen Raum nicht betreten können, ohne zuerst das Gebäude zu betreten. Der Raum ist eine Variable, der Boden eine Methode und der Bau eine Klasse.

Unterricht

Klassen sind Vorlagen zum Erstellen von Objekten, die eine Instanz der Klasse sind und für die Organisation verwandter Variablen und Funktionen nützlich sind. Denken Sie an Klassen wie Ausstechformen, sie haben eine definierte Form und Struktur und alles, was sie betreiben, hat ein bestimmtes Ergebnis (Form des Cookies). Wenn wir einen runden Ausstecher als Klasse in unserem Code definieren, würden wir sagen, dass er einen Umfang, einen Radius und eine Fläche haben muss. Die Werte können jedoch spontan berechnet werden, indem der Benutzer einfach einen Wert fragt: den Radius.

Hier ist die Syntax für die Erstellung einer Klasse:

scope modifier class ClassName{
}

Lassen Sie uns das aufschlüsseln:

  1. scopeKann privat, öffentlich oder global sein.
  2. modifierKann virtuell, abstrakt, mit Freigabe, ohne Freigabe sein (optional)
  3. classDeklariert, dass diese Variable ein Klassendatentyp ist
  4. ClassNameIst der Name der Variablen. Beachten Sie, dass wir camelCase beim Schreiben von Klassennamen nicht folgen. Dies soll klar zwischen regulären Variablen und Klassen unterscheiden.

Geltungsbereich

Der Umfang kann privat, öffentlich oder global sein.

  • PrivateBereich bedeutet, dass jedes Objekt außerhalb der Klasse nicht auf darin enthaltene Methoden zugreifen kann.
  • PublicBereich bedeutet, dass jedes Objekt in der gesamten Anwendung Zugriff darauf hat.
  • GlobalBereich bedeutet, dass jedes Objekt in Apex Zugriff darauf hat.

Modifikator

Der Freigabebereich kann virtuell, abstrakt, mit Freigabe oder ohne Freigabe erfolgen.

  • virtualErmöglicht es Ihnen, Methoden zu überschreiben, die von der übergeordneten Klasse geerbt wurden.
  • abstractErmöglicht es Ihnen, Methoden ohne jemanden zu erstellen.
  • with sharingErmöglicht es Ihnen, den Code im Kontext des Berechtigungsbereichs des Benutzers in Salesforce auszuführen.
  • without sharingErmöglicht es Ihnen, den Code auszuführen und auf Objekte zuzugreifen, auch außerhalb des Berechtigungsbereichs des Benutzers in Salesforce.

Methoden

Methoden (auch Funktionen genannt, wenn sie außerhalb der Klasse geschrieben werden) sind Vorlagenstrukturen, die Eingabeparameter annehmen und berechnete Werte zurückgeben. Die Syntax für Method lautet:

scope modifier return_dataType methodName(inputParameters){

return return_dataType
}

Lassen Sie es uns aufschlüsseln

  1. scopeKann öffentlich, privat, geschützt oder global sein.
  2. modifierKann überschreiben, statisch, endgültig (optional) sein.
  3. return_dataTypeDefiniert, welche Art von Wert die Methode zurückgibt.
  4. methodNameIst der Name, der dieser Methode gegeben wird. Folgt camelCase.
  5. inputParametersSind Parameter, die die Methode für die Verarbeitung auf sie benötigt. (optional)
  6. returnIst ein Schlüsselwort, das angibt, dass ein Wert zurückgegeben wird

Geltungsbereich

Der Umfang kann öffentlich, privat, geschützt oder global sein.

  • PublicMacht die Methode für den Code in der gesamten Anwendung verfügbar.
  • PrivateMacht die Methode außerhalb der Klasse unzugänglich.
  • ProtectedWird verwendet, wenn die Methode nur für ihre untergeordneten Klassen zugänglich sein soll.
  • GlobalBereich macht die Methode in Apex verfügbar.

Modifikator

  • overrideWird verwendet, wenn von der übergeordneten Klasse geerbt und die übergeordnete Methode in der untergeordneten Klasse umgeschrieben wird.
  • staticWird verwendet, wenn eine Methode/Variable erstellt wird, die nur einmal initialisiert wird und ihrer äußeren Klasse zugeordnet ist. Es besteht keine Notwendigkeit, eine neue Instanz der Klasse zu erstellen, um auf statische Methoden zuzugreifen.
  • finalWird verwendet, wenn eine Methode erstellt wird, bei der der Wert nicht geändert werden kann, nachdem er in die Methode/Variable eingefügt wurde.

    • Eine Beispielmethode wäre:

    public String hello(String h){
	String finalString = "Hello " + h;
	return finalString;
}

    To access this method, we would simply type the method name and our String in ( ) like so:

    hello('harshdeep');

    Jetzt auf eine komplexere Situation zusteuern, der Ausstechform. Um einen runden Ausstecher herzustellen, benötigen wir einen Radius vom Benutzer und geben seinen Umfang und seine Fläche zurück. Anstatt dies in unserem anon exec-Fenster zu schreiben, erstellen wir eine neue Datei.

    In the developer console, click on new > apex class. Name the class CookieCutter and press enter.

    Schreiben Sie in der Klasse drei Methoden (siehe Syntaxhandbuch, wenn nötig):

    1. A private method named cir that returns Decimal and takes a Decimal as it's input. It returns the value 2 * 3.14 * r as a Decimal.
    2. A private method named area that returns Decimal and takes a Decimal as it's input. It returns the value 3.14 * r * r as a Decimal.
    3. A public method named radius that returns List<Decimal> and takes a Decimal as it's input. It adds the value from area and cir into the list and returns it.
    public class CookieCutter {

    private Decimal cir(Decimal r){
        Decimal fence = 2 * 3.14 * r;

        return fence;
    }

    private Decimal area(Decimal r){
        Decimal finalArea = 3.14 * r * r;

        return finalArea;
    }

    public List<Decimal> radius(Decimal r){
        List<Decimal> returnList = new List<Decimal>();

        returnList.add(area(r));
        returnList.add(cir(r));

        return returnList;
    }

}

    Lassen Sie uns diesen Code aufschlüsseln:

    • Zuerst machen wir eine öffentliche Klasse namens CookieCutter.
    • Out first method is cir, that returns a Decimal data type and takes a Decimal r as it's input parameter.
    • cir makes a new decimal called fence (short for circumference) and calculated the circumference of the circle using 2 pie r and returns the final value.
    • area makes a new decimal called finalArea that takes Decimal r as an input and calculates area using pie r square and returns the final value.
    • radius method takes decimal r as input and will return a list of decimal.
    • raidus method calls area and cir, adds them to the list and returns the final list.

    Wir werden folgenden Code aus dem Anon Exec-Fenster (Control + E) ausführen. Da sich diese Methode innerhalb einer Klasse befindet, müssen wir eine neue Instanz dieser Klasse erstellen, die den Klassennamen als Datentyp verwendet. Dies entspricht der Erstellung einer neuen Variablen, außer dass der Datentyp die Klasse wäre.

    CookieCutter finalCutter = new CookieCutter();

    Dies weist den Computer an, eine neue Vorlage namens finalCutter basierend auf der CookieCutter-Vorlage zu erstellen. Um nun auf die ourradius-Methode mit 4.0 als Radius zuzugreifen, die eine Liste von Dezimalstellen zurückgibt, nennen wir sie wie folgt:

    List<Decimal> finalList = finalCutter.radius(4.0);

    Now the list finalList has values of area and cir from inside the class! Go ahead and output it in the log and see the results!

    CookieCutter finalCutter = new CookieCutter();
List<Decimal> finalList = finalCutter.radius(4.0);
System.debug(finalList);

    Sie können mehrere Methoden mit demselben Namen innerhalb der Klasse haben, solange es Möglichkeiten gibt, sie zu unterscheiden.

    In unserer CookieCutter-Klasse können wir eine andere Methode definieren, die bereits existiert, aber unterschiedliche Eingaben akzeptiert. Hier ist ein Code:

    public class CookieCutter {

    private Decimal area(Decimal r){
        Decimal finalArea = 3.14 * r * r;
        return finalArea;
    }

    private Decimal area(Decimal r, String someVariable){
        return 2.0;
    }
}

    Notice how we have two methods named area but both take different inputs! This is useful when your program makes use of optional parameters. An example would be when you need to generate numbers for your organization, but marketing department believes odd numbers are evil and must not be used. Your manager approves of this and asks you to write a new generator. This may or may not be from personal experience.

    • Klassen sind Vorlagen, die nachgestellt werden können.
    • Methoden sind generische Verarbeitungseinheiten, die Eingabevariablen annehmen und verarbeitete Ergebnisse zurückgeben.
