OOP, Clases y Métodos

Programación orientada a objetos

Apex es un lenguaje de programación orientada a objetos (OOP), un paradigma de programación donde los datos se almacenan en objetos y código en procedimientos (métodos). La idea de OOP es enlazar datos y funciones que operan en los datos juntos para mantener el control de acceso a través de clases.

OOP trae consigo bastantes conceptos de programación. Aquí hay algunos conceptos que usaremos con más frecuencia que otros:

  • Herencia: Esto le permite "heredar" métodos de la clase padre. Esto es similar a tener hijos. El niño podría tener ojos verdes de su madre y cabello rubio de su padre. Hablaremos de la herencia en detalle más adelante en este curso porque puede complicarse mucho y necesita un día entero dedicado.
  • Encapsulación: Esto significa que las variables dentro de una clase siempre están ocultas y solo se puede acceder usando un método. Un ejemplo sería que no se puede entrar en una habitación sin entrar primero en el edificio. La habitación es una variable, el piso un método y el edificio una clase.

Clases

Las clases son plantillas para crear objetos que son una instancia de la clase y son útiles para organizar variables y funciones relacionadas. Piensa en clases como los cortadores de galletas, tienen una forma y estructura definidas y todo lo que operan tiene un resultado específico (forma de la cookie). Si definimos un cortador de galletas redondo como una clase en nuestro código, diríamos que necesita tener una circunferencia, un radio y un área. Los valores, sin embargo, se pueden calcular sobre la marcha con solo preguntar al usuario un valor: el radio.

Aquí está la sintaxis para hacer una clase:

scope modifier class ClassName{
}

Vamos a desglosar esto:

  1. scopepuede ser privado, público o global.
  2. modifierpuede ser virtual, abstracto, con uso compartido, sin compartir (opcional)
  3. classdeclara que esta variable es un tipo de datos de clase
  4. ClassNamees el nombre de la variable. Tenga en cuenta que no seguimos camelCase al escribir Nombres de Clase. Esto es para hacer una clara distinción entre variables regulares y clases.

Alcance

El alcance puede ser privado, público o global.

  • Privatescope significa que cualquier objeto fuera de la clase no puede acceder a los métodos dentro de ella
  • Publicscope significa que cualquier objeto en toda la aplicación tiene acceso a él.
  • Globalscope significa que cualquier objeto en Apex tiene acceso a él.

Modificador

El ámbito compartido puede ser virtual, abstracto, con uso compartido o sin compartir.

  • virtualmodificador le permite reemplazar métodos heredados de la clase padre.
  • abstractmodificador le permite hacer métodos sin nadie definido.
  • with sharingmodificador le permite ejecutar el código en el contexto del ámbito de permisos del usuario en Salesforce.
  • without sharingmodificador le permite ejecutar el código y acceder a objetos incluso fuera del ámbito de permisos del usuario en Salesforce.

Métodos

Los métodos (también llamados funciones cuando se escriben fuera de la clase) son estructuras de plantillas que toman parámetros de entrada y devuelven valores calculados. La sintaxis de Method es:

scope modifier return_dataType methodName(inputParameters){

return return_dataType
}

Vamos a desglosarlo

  1. scopepuede ser público, privado, protegido o global.
  2. modifierpuede ser anulación, estática, final (opcional).
  3. return_dataTypedefine qué tipo de valor devolverá el método.
  4. methodNamees el nombre dado a este método. Sigue a camelCase.
  5. inputParametersson parámetros que el método toma para procesarlos. (opcional)
  6. returnes una palabra clave que denota que se está devolviendo un valor

Alcance

El alcance puede ser público, privado, protegido o global.

  • Publichace que el método esté disponible para codificar en toda la aplicación.
  • Privatehace que el método sea inaccesible fuera de la clase.
  • Protectedscope se utiliza cuando el método solo debe ser accesible por sus clases hijas.
  • Globalhace que el método esté disponible en todo Apex.

Modificador

  • overridese utiliza cuando se hereda de la clase padre y se reescribe el método padre en la clase secundaria.
  • staticse utiliza al crear un método/variable que se inicializa solo una vez y está asociado con su clase externa. No hay necesidad de hacer una nueva instancia de la clase para acceder a los métodos estáticos.
  • finalse utiliza al crear un método donde el valor no se puede cambiar una vez que se ha insertado en el método/variable.

    • Un método de ejemplo sería:

    public String hello(String h){
	String finalString = "Hello " + h;
	return finalString;
}

    To access this method, we would simply type the method name and our String in ( ) like so:

    hello('harshdeep');

    Ahora hacia una situación más compleja, el Cortador de Galletas. Para hacer un cortador de galletas redondo, necesitaríamos un radio del usuario, y devolveríamos su circunferencia y área. En lugar de escribir esto en nuestra ventana anon exec, hagamos un nuevo archivo.

    In the developer console, click on new > apex class. Name the class CookieCutter and press enter.

    En la clase, escriba tres métodos (consulte la guía de sintaxis si tiene que hacerlo):

    1. A private method named cir that returns Decimal and takes a Decimal as it's input. It returns the value 2 * 3.14 * r as a Decimal.
    2. A private method named area that returns Decimal and takes a Decimal as it's input. It returns the value 3.14 * r * r as a Decimal.
    3. A public method named radius that returns List<Decimal> and takes a Decimal as it's input. It adds the value from area and cir into the list and returns it.
    public class CookieCutter {

    private Decimal cir(Decimal r){
        Decimal fence = 2 * 3.14 * r;

        return fence;
    }

    private Decimal area(Decimal r){
        Decimal finalArea = 3.14 * r * r;

        return finalArea;
    }

    public List<Decimal> radius(Decimal r){
        List<Decimal> returnList = new List<Decimal>();

        returnList.add(area(r));
        returnList.add(cir(r));

        return returnList;
    }

}

    Desglosemos este código:

    • Primero, hacemos una clase pública llamada CookieCutter.
    • Out first method is cir, that returns a Decimal data type and takes a Decimal r as it's input parameter.
    • cir makes a new decimal called fence (short for circumference) and calculated the circumference of the circle using 2 pie r and returns the final value.
    • area makes a new decimal called finalArea that takes Decimal r as an input and calculates area using pie r square and returns the final value.
    • radius method takes decimal r as input and will return a list of decimal.
    • raidus method calls area and cir, adds them to the list and returns the final list.

    Ejecutaremos el siguiente código de Anon Exec Window (Control + E). Debido a que este método está dentro de una clase, necesitamos crear una nueva instancia de esta clase usando el nombre de la clase como tipo de datos. Esto es lo mismo que hacer una nueva variable, excepto que el tipo de datos sería la clase.

    CookieCutter finalCutter = new CookieCutter();

    This tells the computer to make a new template called finalCutter based on CookieCutter template. Now to access our radius method with 4.0 as it's radius, that returns a list of decimals, we call it like so:

    List<Decimal> finalList = finalCutter.radius(4.0);

    Now the list finalList has values of area and cir from inside the class! Go ahead and output it in the log and see the results!

    CookieCutter finalCutter = new CookieCutter();
List<Decimal> finalList = finalCutter.radius(4.0);
System.debug(finalList);

    Puede tener múltiples métodos con el mismo nombre dentro de la clase, siempre y cuando haya maneras de distinguirlos.

    En nuestra clase CookieCutter, podemos definir otro método que ya existe pero toma diferentes entradas. Aquí hay algo de código:

    public class CookieCutter {

    private Decimal area(Decimal r){
        Decimal finalArea = 3.14 * r * r;
        return finalArea;
    }

    private Decimal area(Decimal r, String someVariable){
        return 2.0;
    }
}

    Notice how we have two methods named area but both take different inputs! This is useful when your program makes use of optional parameters. An example would be when you need to generate numbers for your organization, but marketing department believes odd numbers are evil and must not be used. Your manager approves of this and asks you to write a new generator. This may or may not be from personal experience.
