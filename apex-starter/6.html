OOP, classes et méthodes

Programmation orientée objet

Apex est un langage de programmation orientée objet (OOP), un paradigme de programmation où les données sont stockées dans des objets et du code dans des procédures (méthodes). L'idée de OOP est de lier les données et les fonctions qui fonctionnent sur les données ensemble pour maintenir le contrôle d'accès par le biais de classes.

OOP apporte beaucoup de concepts de programmation avec lui. Voici quelques concepts que nous utiliserons plus souvent que d'autres :

  • Héritage : Cela vous permet d'"hériter" des méthodes de la classe parente. C'est similaire à avoir des enfants. L'enfant pouvait avoir les yeux verts de sa mère et les cheveux blonds de son père. Nous discuterons de l'héritage en détail plus loin dans ce cours parce qu'il peut devenir vraiment compliqué et nécessite une journée entière dédiée.
  • Encapsulation : Cela signifie que les variables à l'intérieur d'une classe sont toujours masquées et ne peuvent être consultées qu'à l'aide d'une méthode. Par exemple, vous ne pouvez pas entrer dans une pièce sans entrer d'abord dans le bâtiment. La pièce est une variable, le sol une méthode et le bâtiment une classe.

Cours

Les classes sont des modèles pour créer des objets qui sont une instance de la classe et sont utiles pour organiser les variables et les fonctions connexes. Pensez aux classes comme les emporte-pièces, elles ont une forme et une structure définies et tout ce sur quoi elles opèrent a un résultat spécifique (forme du cookie). Si nous définissons un emporte-pièce rond comme une classe dans notre code, nous dirons qu'il doit avoir une circonférence, un rayon et une zone. Les valeurs, cependant, peuvent être calculées à la volée en demandant simplement à l'utilisateur une valeur : le rayon.

Voici la syntaxe pour créer une classe :

scope modifier class ClassName{
}

Décomposons ceci :

  1. scopepeut être privé, public ou mondial.
  2. modifierpeut être virtuel, abstrait, avec partage, sans partage (facultatif)
  3. classdéclare que cette variable est un type de données de classe
  4. ClassNameest le nom de la variable. Notez que nous ne suivons pas camelCase lors de l'écriture de noms de classe. Il s'agit de faire une distinction claire entre les variables régulières et les classes.

Champ d'application

La portée peut être privée, publique ou mondiale.

  • Privatescope signifie que tout objet en dehors de la classe ne peut pas accéder aux méthodes à l'intérieur de celle-ci
  • Publicscope signifie que tout objet de l'application y a accès.
  • Globalscope signifie que tout objet dans Apex y a accès.

Modifier

L'étendue de partage peut être virtuelle, abstraite, avec partage ou sans partage.

  • virtualvous permet de remplacer les méthodes héritées de la classe parente.
  • abstractvous permet de faire des méthodes sans que personne ne soit défini.
  • with sharingvous permet d'exécuter le code dans le contexte de l'étendue d'autorisation de l'utilisateur dans Salesforce.
  • without sharingvous permet d'exécuter le code et d'accéder aux objets même en dehors de la portée d'autorisation de l'utilisateur dans Salesforce.

Pour créer une classe d'emporte-pièce, nous la définissons de la manière suivante :

public class CookieCutter{
}

Méthodes

Les méthodes (également appelées fonctions lorsqu'elles sont écrites en dehors de la classe) sont des structures de modèle qui prennent des paramètres d'entrée et renvoient des valeurs calculées. La syntaxe de Method est la suivante :

scope modifier return_dataType methodName(inputParameters){

return return_dataType
}

Décomposons-le

  1. scopepeut être public, privé, protégé ou mondial.
  2. modifierpeut être override, static, final (facultatif).
  3. return_dataTypedéfinit le type de valeur que la méthode retournera.
  4. methodNameest le nom donné à cette méthode. Suit camelCase.
  5. inputParameterssont des paramètres que la méthode prend pour les traiter. (facultatif)
  6. returnest un mot-clé qui indique qu'une valeur est renvoyée

Champ d'application

La portée peut être publique, privée, protégée ou mondiale.

  • Publicmet la méthode à disposition pour coder dans toute l'application.
  • Privaterend la méthode inaccessible en dehors de la classe.
  • Protectedest utilisée lorsque la méthode ne devrait être accessible qu'à ses classes enfants.
  • Globalrend la méthode disponible dans tout Apex.

Modifier

  • overrideest utilisé lors de l'héritage de la classe parente et de la réécriture de la méthode parente dans la classe enfant.
  • staticest utilisé lors de la création d'une méthode/variable qui n'est initialisée qu'une seule fois et qui est associée à sa classe externe. Il n'est pas nécessaire de créer une nouvelle instance de la classe pour accéder aux méthodes statiques.
  • finalest utilisé lors de la création d'une méthode où la valeur ne peut pas être modifiée une fois qu'elle a été insérée dans la méthode/variable.

    • Un exemple de méthode serait :

    public String hello(String h){
	String finalString = "Hello " + h;
	return finalString;
}

    To access this method, we would simply type the method name and our String in ( ) like so:

    hello('harshdeep');

    Cela reviendrait ;

    Maintenant, vers une situation plus complexe, le Cookie Cutter. Pour faire un emporte-pièce rond, nous aurions besoin d'un rayon de l'utilisateur et de lui rendre sa circonférence et sa surface. Au lieu d'écrire cela dans notre fenêtre anon exec, faisons un nouveau fichier.

    In the developer console, click on new > apex class. Name the class CookieCutter and press enter.

    Dans la classe, écrivez trois méthodes (reportez-vous au guide syntaxique si vous devez le faire) :

    1. A private method named cir that returns Decimal and takes a Decimal as it's input. It returns the value 2 * 3.14 * r as a Decimal.
    2. A private method named area that returns Decimal and takes a Decimal as it's input. It returns the value 3.14 * r * r as a Decimal.
    3. A public method named radius that returns List<Decimal> and takes a Decimal as it's input. It adds the value from area and cir into the list and returns it.
    public class CookieCutter {

    private Decimal cir(Decimal r){
        Decimal fence = 2 * 3.14 * r;

        return fence;
    }

    private Decimal area(Decimal r){
        Decimal finalArea = 3.14 * r * r;

        return finalArea;
    }

    public List<Decimal> radius(Decimal r){
        List<Decimal> returnList = new List<Decimal>();

        returnList.add(area(r));
        returnList.add(cir(r));

        return returnList;
    }

}

    Décomposons ce code :

    • Tout d'abord, nous faisons une classe publique appelée CookieCutter.
    • Out first method is cir, that returns a Decimal data type and takes a Decimal r as it's input parameter.
    • cir makes a new decimal called fence (short for circumference) and calculated the circumference of the circle using 2 pie r and returns the final value.
    • area makes a new decimal called finalArea that takes Decimal r as an input and calculates area using pie r square and returns the final value.
    • radius method takes decimal r as input and will return a list of decimal.
    • raidus method calls area and cir, adds them to the list and returns the final list.

    Nous exécuterons le code suivant de Anon Exec Window (Control + E). Comme cette méthode se trouve à l'intérieur d'une classe, nous devons créer une nouvelle instance de cette classe en utilisant le nom de classe comme type de données. C'est la même chose que de faire une nouvelle variable, sauf que le type de données serait la classe.

    CookieCutter finalCutter = new CookieCutter();

    This tells the computer to make a new template called finalCutter based on CookieCutter template. Now to access our radius method with 4.0 as it's radius, that returns a list of decimals, we call it like so:

    List<Decimal> finalList = finalCutter.radius(4.0);

    Now the list finalList has values of area and cir from inside the class! Go ahead and output it in the log and see the results!

    CookieCutter finalCutter = new CookieCutter();
List<Decimal> finalList = finalCutter.radius(4.0);
System.debug(finalList);

    Vous pouvez avoir plusieurs méthodes portant le même nom à l'intérieur de la classe, à condition qu'il existe des moyens de les distinguer.

    Dans notre classe CookieCutter, nous pouvons définir une autre méthode qui existe déjà mais qui prend différentes entrées. Voici un code :

    public class CookieCutter {

    private Decimal area(Decimal r){
        Decimal finalArea = 3.14 * r * r;
        return finalArea;
    }

    private Decimal area(Decimal r, String someVariable){
        return 2.0;
    }
}

    Notice how we have two methods named area but both take different inputs! This is useful when your program makes use of optional parameters. An example would be when you need to generate numbers for your organization, but marketing department believes odd numbers are evil and must not be used. Your manager approves of this and asks you to write a new generator. This may or may not be from personal experience.
