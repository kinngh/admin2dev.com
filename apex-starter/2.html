IDE, Variablen und Datentypen

Integrierte Entwicklungsumgebung

Beziehen wir uns auf den Code für die Kaffeezubereitung:

get cup = 1 unit;
get coffee = 100 grams;
get water = 500 ml;
get pot;
get timer;
get frenchPress;

put cup, frenchPress, pot on table;
pour water(500 ml) in pot;
start pot;
set timer = 60 seconds;

put coffee(100 grams) in frenchPress;
pour 1/5 water from pot in frenchPress;
set timer = 30 seconds;

pour water from pot in frenchPress;
set timer = 5 minutes;

pour everything from frenchPress to cup;

drink;

Der erste Schritt zur Kaffeezubereitung war der Eintritt in die Küche. Das Äquivalent, zu dem eine IDE (Integrated Development Environment) verwendet wird.

Eine IDE oder integrierte Entwicklungsumgebung ist eine Kombination aus Texteditor und Tools, die das Schreiben, Ausführen und die Fehlerberichterstattung (Debugging) erleichtern. Während dieses Kurses werden wir 2 IDEs verwenden.

  • Salesforce-Entwicklerkonsole
  • Microsoft Visual Studio-Code

Da wir gerade erst anfangen, werden wir die Salesforce Developer Console verwenden.

Dafür werden wir unsere Entwicklerorganisation verwenden. Eine Entwicklerorganisation ist eine Salesforce-Instanz, die zum Testen und Debuggen von SF-Anwendungen und -Code verwendet wird. Wenn Sie noch keine Salesforce Developer Org haben, machen Sie es wie folgt:

  • Besuchen Sie die Salesforce Developer Website und melden Sie sich an.

  • Klicken Sie auf Ihren Namen und dann auf Mein Entwicklerkonto

Variablen und Datentypen

Jetzt, da wir in unserer Küche (der Entwicklerkonsole) sind, machen wir Kaffee.
Der erste Schritt bestand darin, unsere Materialien zu sammeln. 

 get cup = 1 unit;
get coffee = 100 grams;
get water = 500 ml;
get pot;
get timer;
get frenchPress;
Wir müssen unsere Grundmaterialien sammeln, um weiter darauf aufzubauen. Sie haben Materialien, die ausschließlich den Zweck haben, Kaffee/Zucker/Wasser darin zu halten. Sie haben Materialien mit dem Zweck, seinen Inhalt zu verarbeiten, wie den Topf mit kochendem Wasser oder die Verwendung von French Press zum Filtern des Kaffees. Dann verlassen wir die Küche mit nur einer Tasse, die mit verarbeiteten/gemischten Zutaten gefüllt ist. Die "Tassen" im Code, die Ihre Materialien enthalten, werden Variablen genannt.

Variablen werden so genannt, weil sie variieren können, was bedeutet, dass sich ihr Inhalt im Laufe der Zeit ändern kann, genau wie die Tasse zuerst verwendet wurde, um Wasser zu halten, dann gemahlenen Kaffee und schließlich den vorbereiteten Kaffee. Im Gegensatz zum realen Leben, dem Cup, der verwendet wurde, um mehrere Inhalte verschiedener Typen aufzunehmen, kann eine Variable Inhalte nur eines Typs enthalten, und dies wird als Datentyp bezeichnet.

Ein Datentyp ist nur der Programmierer, der dem Computer mitteilt, wie er die Variable verwenden will. Es ähnelt dem Spiel fit the shape aus dem Kindergarten. Eine Sternlochung (Datentyp) ermöglicht es nur, dass ein Stern (Daten) hineinpasst. Es gibt zwei Arten von Datentypen in Apex

  • Primitive Datentypen
  • Komplexe Datentypen

Primitive Datentypen sind die einfachsten zu definierenden Datentypen.

  • Blob
  • Boolescher
  • Datum
  • DatumZeit
  • Dezimal
  • Doppelgänger
  • ID
  • Ganzzahl
  • Lang
  • Objekt
  • Schnur
  • Zeit

Komplexe Datentypen basieren auf den Bedürfnissen des Programmierers.

  • Klassenobjekt
  • Zähler
  • Liste
  • Zuordnung
  • Set
  • sObjekt

Eine Liste kann beispielsweise eine Gruppe von IDs, Strings, Dezimalstellen und Ganzzahlen gleichzeitig enthalten. Eine Liste könnte auch mehrere Listen in sich enthalten! Die Verwendung komplexer Datentypen hängt vollständig davon ab, wie der Programmierer sie verwenden will.

Syntax

Jede Sprache hat ihre eigene Grammatik, dasselbe gilt für Programmiersprachen. Jede Programmiersprache hat ihre eigene streng definierte Grammatik, die Syntax genannt wird.

Die Grammatik (Syntax), um neue Variablen zu erstellen und ihre Datentypen zu definieren, ist

dataType variableName = value;
  • dataTypeIst einer der primitiven oder komplexen Datentypen.
  • variableNameIst, wie wir diese Variable später verwenden.
  • =Ist der Zuweisungsoperator. Das bedeutet, dass Sie den Wert auf der rechten Seite der Variablen auf der linken Seite zuweisen. Mehr über Operatoren und wie sie funktionieren, finden Sie später im Kurs.
  • valueIst der endgültige Wert, der der Variablen zugewiesen wird.
  • ;Definiert das Zeilenende / Blockende.
A good practice while naming multi-word variables is to use camel case[1]. The variable variableName could be written as VariableName, but it's a good practice to lower case the first letter of the first word and then capitalize first letter of following words. Apex is a case-insensitive language, so variableName and VariableName both mean the same variable. A lot of other programming languages like Python and Swift would see variableName and VariableName as two separate entities.
In zukünftigen Lektionen werden Sie erkennen, wie der größte Teil des Codes, den Sie schreiben, einfach Syntax ist, die mit Logik und Datentypen gefüllt ist.

Datentypen: Grundlagen

Lassen Sie uns in einer geordneten Reihenfolge des besseren Verständnisses in die Grundlagen von Datentypen eintauchen.

  • ID
    • Der ID-Datentyp enthält Salesforce-IDs in einfachen Anführungszeichen. Eine ID kann ein beliebiges gültiges 18-stelliges Zeichen sein, das eine ID darstellt.
    ID nameID = '00300000003T2PGAA0';

  • Schnur
    • Ein String-Datentyp enthält Text, Zahlen und andere alphanumerische Zeichen in einfachen Anführungszeichen. Wenn Zahlen in einen String eingefügt werden, kann er nicht verwendet werden, um mathematische Operationen auszuführen. Deshalb verwenden wir die Datentypen Integer, Decimal und Double.
    String siteName = 'Admin2Dev';

  • Ganzzahl
    • An Integer data type holds whole numbers that can be negative, zero or positive. The range of Integer is -2,147,483,647 to 2,147,483,647 and separators like , and spaces are not allowed. 
    Integer positiveNumber = 10000;
Integer zero = 0;
Integer negativeNumber = -100;

  • Dezimal
    • A Decimal data type holds positive and negative numbers that include a decimal point and separators like , and spaces are not allowed. 
    Decimal positiveNumber = 1.45;
Decimal zero = 0.0;
Decimal negativeNumber = -1.45;

  • Doppelgänger
    • A Double data type holds positive and negative numbers that include a decimal point. It has a range of -263 to 263-1 and separators like , and spaces are not allowed. 
    Double positiveNumber = 1.45;
Double zero = 0.0;
Double negativeNumber = -1.45;

  • Lang
    • A long data type holds whole numbers that can be negative, zero or positive. It has a range of -263 to 263-1 and separators like , and spaces are not allowed. 
    Long positiveNumber = 100;
Long zero = 0;
Long negativeNumber = -100;

Zusammenfassung


  • In einer IDE (Integrated Development Environment) schreiben, führen und debuggen wir Code.
  • Variablen enthalten Daten, die sich im Laufe der Zeit ändern können.
  • Datentypen definieren, welche Art von Daten eine Variable enthalten wird.
  • Es gibt primitive und komplexe Datentypen.
  • Primitive Datentypen sind einfache Zahlen, Texte und Salesforce-IDs. Komplexe Datentypen hängen davon ab, wie der Programmierer sie verwenden will.
  • Die Grammatik einer Programmiersprache wird Syntax genannt und muss strikt befolgt werden.
