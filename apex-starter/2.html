IDE, variables et types de données

Environnement de développement intégré

Reportons-vous au code de la fabrication du café :

get cup = 1 unit;
get coffee = 100 grams;
get water = 500 ml;
get pot;
get timer;
get frenchPress;

put cup, frenchPress, pot on table;
pour water(500 ml) in pot;
start pot;
set timer = 60 seconds;

put coffee(100 grams) in frenchPress;
pour 1/5 water from pot in frenchPress;
set timer = 30 seconds;

pour water from pot in frenchPress;
set timer = 5 minutes;

pour everything from frenchPress to cup;

drink;

La première étape pour faire du café a été d'entrer dans la cuisine. L'équivalent est l'utilisation d'un IDE (Integrated Development Environment).

Un IDE ou environnement de développement intégré est une combinaison d'éditeur de texte et d'outils qui facilitent l'écriture, l'exécution et le signalement d'erreurs (déboguage). Tout au long de ce cours, nous utiliserons 2 IDE.

  • Console de développement Salesforce
  • Code Microsoft Visual Studio

Puisque nous commençons tout juste, nous utiliserons Salesforce Developer Console.

Pour cela, nous utiliserons notre organisation de développeurs. Une organisation de développeur est une instance Salesforce qui est utilisée pour tester et déboguer les applications et le code SF. Si vous n'avez pas encore d'organisation de développement Salesforce, voici comment vous y parvenez :

  • Rendez-vous sur le site Web des développeurs Salesforce et inscrivez-vous.

  • Cliquez sur votre nom et cliquez sur Mon compte développeur

Variables et types de données

Maintenant que nous sommes dans notre cuisine (la console du développeur), faisons un café.
La première étape a consista à rassembler nos matériaux. 

 get cup = 1 unit;
get coffee = 100 grams;
get water = 500 ml;
get pot;
get timer;
get frenchPress;
Nous devons rassembler nos matériaux de base pour les développer davantage. Vous avez des matériaux qui ont pour seul but de contenir du café/sucre/eau. Vous avez des matériaux dans le but de traiter son contenu, comme l'eau bouillante de la casserole ou l'utilisation de la presse française pour filtrer le café. Ensuite, nous nous éloignons de la cuisine en utilisant une seule tasse remplie d'ingrédients transformés/mélangés. Les "tasses" dans le code qui contiennent vos matériaux sont appelées variables.

Les variables sont appelées ainsi parce qu'elles peuvent varier, ce qui signifie que leur contenu peut changer au fil du temps, tout comme la tasse a d'abord été utilisée pour retenir l'eau, puis le café moulu, puis enfin le café préparé. Contrairement à la vie réelle de la tasse qui a été utilisée pour contenir plusieurs contenus de différents types, une variable ne peut contenir que du contenu d'un seul type et c'est ce qu'on appelle son type de données.

Un type de données est juste le programmeur qui dit à l'ordinateur comment il a l'intention d'utiliser la variable. C'est similaire à jouer au jeu de forme de la maternelle. Un trou d'étoile (type de données) ne permettra qu'une étoile (données) d'être ajustée à l'intérieur. Il existe deux types de types de données dans Apex

  • Types de données primitives
  • Types de données complexes

Les types de données primitifs sont les types de données les plus simples à définir.

  • Blob
  • Booléen
  • Date
  • DateTime
  • Décimal
  • Double
  • ID
  • Entier
  • Long
  • Objet
  • Chaîne
  • Heure

Les types de données complexes sont basés sur les besoins du programmeur.

  • Objet de classe
  • Enumerator
  • Liste
  • Carte
  • Ensemble
  • sObject

Par exemple, une liste peut contenir un groupe d'ID, de chaînes, de décimales et d'entiers en même temps. Une liste pourrait également contenir plusieurs listes en elle-même ! L'utilisation de types de données complexes dépend entièrement de la façon dont le programmeur a l'intention de l'utiliser.

Syntaxe

Chaque langage a sa propre grammaire, il en va de même pour les langages de programmation. Chaque langage de programmation a son propre ensemble de grammaire strictement définie appelée syntaxe.

La grammaire (syntaxe) pour créer de nouvelles variables et définir leurs types de données est

dataType variableName = value;
  • dataTypeest l'un des types de données primitifs ou complexes.
  • variableNameest la façon dont nous utilisons cette variable plus tard.
  • =est l'opérateur d'affectation. Cela signifie qu'il faut attribuer la valeur à droite à la variable de gauche. Plus d'informations sur les opérateurs et leur fonctionnement sont plus tard dans le cours.
  • valueest la valeur finale attribuée à la variable.
  • ;définit la fin de la ligne / du bloc.
A good practice while naming multi-word variables is to use camel case[1]. The variable variableName could be written as VariableName, but it's a good practice to lower case the first letter of the first word and then capitalize first letter of following words. Apex is a case-insensitive language, so variableName and VariableName both mean the same variable. A lot of other programming languages like Python and Swift would see variableName and VariableName as two separate entities.
Dans les leçons futures, vous vous rendrez compte que la plupart du code que vous écrivez est simplement rempli de logique et de types de données.

Types de données : Principes de base

Plongeons dans les bases des types de données dans un ordre de facilité de compréhension.

  • ID
    • Le type de données d'identification contient des ID Salesforce entre guillemets simples. Un ID peut être n'importe quel caractère valide à 18 chiffres qui représente un ID.
    ID nameID = '00300000003T2PGAA0';

  • Chaîne
    • Un type de données String contient du texte, des nombres et d'autres caractères alphanumériques entre guillemets simples. Lorsque des nombres sont placés à l'intérieur d'une chaîne, il ne peut pas être utilisé pour exécuter des opérations mathématiques, c'est pourquoi nous utilisons des types de données entier, décimal et double.
    String siteName = 'Admin2Dev';

  • Entier
    • An Integer data type holds whole numbers that can be negative, zero or positive. The range of Integer is -2,147,483,647 to 2,147,483,647 and separators like , and spaces are not allowed. 
    Integer positiveNumber = 10000;
Integer zero = 0;
Integer negativeNumber = -100;

  • Décimal
    • A Decimal data type holds positive and negative numbers that include a decimal point and separators like , and spaces are not allowed. 
    Decimal positiveNumber = 1.45;
Decimal zero = 0.0;
Decimal negativeNumber = -1.45;

  • Double
    • A Double data type holds positive and negative numbers that include a decimal point. It has a range of -263 to 263-1 and separators like , and spaces are not allowed. 
    Double positiveNumber = 1.45;
Double zero = 0.0;
Double negativeNumber = -1.45;

  • Long
    • A long data type holds whole numbers that can be negative, zero or positive. It has a range of -263 to 263-1 and separators like , and spaces are not allowed. 
    Long positiveNumber = 100;
Long zero = 0;
Long negativeNumber = -100;

Résumé


  • Un IDE (Integrated Development Environment) est l'endroit où nous écrivons, exécutons et déboguons du code.
  • Les variables contiennent des données qui peuvent changer au fil du temps.
  • Les types de données définissent le type de données qu'une variable contiendra.
  • Il existe des types de données primitifs et complexes.
  • Les types de données primitives sont des nombres simples, des textes et des identifiants Salesforce. Les types de données complexes dépendent de la façon dont le programmeur a l'intention de les utiliser.
  • La grammaire d'un langage de programmation est appelée syntaxe et doit être strictement suivie.
