IDE, Variables y Tipos de Datos

Entorno de Desarrollo Integrado

Consultemos el código para hacer café:

get cup = 1 unit;
get coffee = 100 grams;
get water = 500 ml;
get pot;
get timer;
get frenchPress;

put cup, frenchPress, pot on table;
pour water(500 ml) in pot;
start pot;
set timer = 60 seconds;

put coffee(100 grams) in frenchPress;
pour 1/5 water from pot in frenchPress;
set timer = 30 seconds;

pour water from pot in frenchPress;
set timer = 5 minutes;

pour everything from frenchPress to cup;

drink;

El primer paso para hacer café fue entrar en la cocina. El equivalente es usar un IDE (Entorno de Desarrollo Integrado).

Un IDE o Entorno de Desarrollo Integrado es una combinación de editor de texto y herramientas que facilitan la escritura, la ejecución y la presentación de informes de errores (depuración). A lo largo de este curso, usaremos 2 IDE.

  • Consola para desarrolladores de Salesforce
  • Código de Microsoft Visual Studio

Ya que apenas estamos empezando, usaremos Salesforce Developer Console.

Para esto, usaremos nuestra Organización de Desarrolladores. Una organización de desarrolladores es una instancia de Salesforce que se utiliza para probar y depurar aplicaciones y código de SF. Si aún no tiene una organización de desarrolladores de Salesforce, así es como la hace:

  • Diríjase al sitio web para desarrolladores de Salesforce y regístrese.

  • Haga clic en su nombre y haga clic en Mi cuenta de desarrollador

Variables y tipos de datos

Ahora que estamos en nuestra cocina (la consola del desarrollador), hagamos un poco de café.
El primer paso fue reunir nuestros materiales. 

 get cup = 1 unit;
get coffee = 100 grams;
get water = 500 ml;
get pot;
get timer;
get frenchPress;
Necesitamos reunir nuestros materiales básicos para construir sobre ellos más. Tiene materiales que tienen el único propósito de contener café/azúcar/agua en ellos. Tienes materiales con el propósito de procesar su contenido, como la olla hirviendo agua o usando French Press para filtrar el café. Luego nos alejamos de la cocina usando solo una taza llena de ingredientes procesados / mezclados. Las "tazas" en el código que contienen sus materiales se llaman Variables.

Las variables se llaman así porque pueden variar, lo que significa que su contenido puede cambiar con el tiempo al igual que la taza se usó primero para retener agua, luego café molido y luego finalmente el café preparado. A diferencia de la vida real, la taza que se usó para contener múltiples contenidos de diferentes tipos, una variable puede contener contenido de un solo tipo y esto se llama su tipo de datos.

Un tipo de datos es solo el programador que le dice a la computadora cómo pretenden usar la variable. Es similar a jugar al juego fit the shape desde el jardín de infantes. Un agujero de estrella (tipo de datos) solo permitirá que una estrella (datos) quepa dentro. Hay dos tipos de tipos de datos en Apex

  • Tipos de datos primitivos
  • Tipos de datos complejos

Los tipos de datos primitivos son los tipos de datos más simples de definir.

  • Blob
  • Booleano
  • Fecha
  • DateTime
  • Decimal
  • Doble
  • ID
  • Entero
  • Largo
  • Objeto
  • Cadena
  • Hora

Los tipos de datos complejos se basan en las necesidades del programador.

  • Objeto Class
  • Enumerador
  • Lista
  • Mapa
  • Conjunto
  • sObject

Por ejemplo, una lista puede contener un grupo de ID, cadenas, decimales y enteros al mismo tiempo. ¡Una lista también podría contener varias listas dentro de sí misma! El uso de tipos de datos complejos depende completamente de cómo el programador tenga la intención de usarlos.

Sintaxis

Cada lenguaje tiene su propia gramática, lo mismo se aplica a los lenguajes de programación. Cada lenguaje de programación tiene su propio conjunto de gramática estrictamente definida llamada sintaxis.

La gramática (sintaxis) para hacer nuevas variables y definir sus tipos de datos es

dataType variableName = value;
  • dataTypees uno de los tipos de datos primitivos o complejos.
  • variableNamees cómo usamos esta variable más tarde.
  • =es el operador de asignación. Esto significa asignar el valor de la derecha a la variable de la izquierda. Más sobre los operadores y cómo funcionan es más adelante en el curso.
  • valuees el valor final que se asigna a la variable.
  • ;define el final de la línea / bloque.
A good practice while naming multi-word variables is to use camel case[1]. The variable variableName could be written as VariableName, but it's a good practice to lower case the first letter of the first word and then capitalize first letter of following words. Apex is a case-insensitive language, so variableName and VariableName both mean the same variable. A lot of other programming languages like Python and Swift would see variableName and VariableName as two separate entities.
En lecciones futuras, te darás cuenta de cómo la mayor parte del código que escribes es simplemente sintaxis rellena de lógica y tipos de datos.

Tipos de datos: Fundamentos

Sumérjase en los conceptos básicos de los tipos de datos en un orden de facilidad de comprensión.

  • ID
    • El tipo de datos de ID contiene los ID de Salesforce entre comillas simples. Un ID puede ser cualquier carácter válido de 18 dígitos que represente un ID.
    ID nameID = '00300000003T2PGAA0';

  • Cadena
    • Un tipo de datos String contiene texto, números y otros caracteres alfanuméricos entre comillas simples. Cuando los números se ponen dentro de una cadena, no se puede usar para ejecutar operaciones matemáticas, es por eso que usamos tipos de datos enteros, decimales y dobles.
    String siteName = 'Admin2Dev';

  • Entero
    • An Integer data type holds whole numbers that can be negative, zero or positive. The range of Integer is -2,147,483,647 to 2,147,483,647 and separators like , and spaces are not allowed. 
    Integer positiveNumber = 10000;
Integer zero = 0;
Integer negativeNumber = -100;

  • Decimal
    • A Decimal data type holds positive and negative numbers that include a decimal point and separators like , and spaces are not allowed. 
    Decimal positiveNumber = 1.45;
Decimal zero = 0.0;
Decimal negativeNumber = -1.45;

  • Doble
    • A Double data type holds positive and negative numbers that include a decimal point. It has a range of -263 to 263-1 and separators like , and spaces are not allowed. 
    Double positiveNumber = 1.45;
Double zero = 0.0;
Double negativeNumber = -1.45;

  • Largo
    • A long data type holds whole numbers that can be negative, zero or positive. It has a range of -263 to 263-1 and separators like , and spaces are not allowed. 
    Long positiveNumber = 100;
Long zero = 0;
Long negativeNumber = -100;

Resumen


  • Un IDE (Entorno de Desarrollo Integrado) es donde escribimos, ejecutamos y depuramos código.
  • Las variables contienen datos que pueden cambiar con el tiempo.
  • Los tipos de datos definen qué tipo de datos retendrá una variable.
  • Hay tipos de datos primitivos y complejos.
  • Los tipos de datos primitivos son números simples, textos e ID de Salesforce. Los tipos de datos complejos dependen de cómo el programador tenga la intención de usarlos.
  • La gramática de un lenguaje de programación se llama sintaxis y debe seguirse estrictamente.
