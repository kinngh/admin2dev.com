Consola de desarrolladores, operaciones aritméticas, interpolación de cadenas y refactorización de código

Instalación del modo oscuro para Salesforce Extension para Chrome [Recomendado]

Como programador, pasará horas en una estirada mirando una pantalla averiguando soluciones y escribiendo código que hace que sea realmente importante proteger sus ojos. Una buena manera de reducir el estrés en los ojos es usando el modo oscuro siempre que sea posible. Salesforce no proporciona una interfaz de usuario en modo oscuro incorporada, pero es posible llegar allí usando una extensión del navegador. El que recomiendo es el modo oscuro para Salesorce.

Descargue la extensión de Chrome Web Store y siga las instrucciones en pantalla

Todas las capturas de pantalla a lo largo del curso usarán Darkmode siempre que sea posible. Tenga en cuenta que esto no cambia ninguna funcionalidad en su organización de Salesforce, solo cambia el color de los elementos de la interfaz de usuario.

La Consola de Desarrolladores

Abra la consola del desarrollador.

Deberías ser recibido con una pantalla similar a esta:

    Barra de menú superior

  1. Archivo: Este menú contiene opciones para abrir, guardar, cerrar y eliminar archivos y recursos.
  2. Editar: Este menú contiene opciones para buscar y reemplazar, corregir la sangría y el formato de código.
  3. Depuración: Este menú contiene opciones que se relacionan con la depuración y la corrección de errores de código.
  4. Prueba: Este menú contiene opciones relacionadas con probar nuestro código y asegurarnos de que nuestro código haga lo que se supone que debe.
  5. Espacio de trabajo: Este menú contiene opciones para configurar la interfaz de usuario para la consola del desarrollador.
  6. Ayuda: Este menú contiene temas generales de ayuda como enlaces a documentación, teclas de acceso directo y preferencias para la consola del desarrollador.

    7. Editor de texto

  7. Pestañas: Este espacio contiene todos los archivos abiertos en un formato de pestañas.
  8. Editor: Aquí es donde escribimos nuestro código.

    9. Barra de menú inferior

  9. Registros: Esta pestaña contiene todos los registros de ejecuciones actuales y anteriores.
  10. Pruebas: Esta pestaña contiene todos los informes de pruebas de nuestro código, como cobertura de código y resultados de aprobación/no aprobación.
  11. Puntos de control: Esta pestaña contiene información sobre los puntos de control cuando se configuran en código.
  12. Editor de consultas: Esta opción permite al desarrollador ejecutar consultas SOSL y SOQL y validarlas.
  13. Estado de la vista: Esta pestaña le permite examinar el estado de la vista para la solicitud de página de Visualforce.
  14. Progreso: Esta pestaña muestra el progreso en la ejecución del código actual y anterior.
  15. Problemas: Esta pestaña muestra, en tiempo casi real, los errores disponibles, si los hay, mientras se escribe nuestro código.

Escribir código

Now that we know where we write our code, let's write some of our first programs! Head over to Debug > Open execute anonymous window. This window allows us to write snippets of code. Alternativelt, you can press Ctrl and E to open quicker.

Hagamos 2 enteros y añádalos:

Integer a = 5;
Integer b = 4;
a + b;

Asegúrese de que "Abrir registros después de la ejecución" esté marcada. Su pantalla debería tener un aspecto similar al siguiente:

Notará que es recibido con esta pantalla:

Line 3, Column 1:
Expression cannot be a statment

The code in line 3, a+b is the equivalent to mixing coffee and water in nothing. We need a variable (cup) to hold them. Let's fix that:

Integer a = 5;
Integer b = 4;
Integer c = a + b;

Notará que lo llevan automáticamente a esta pantalla

Click on Debug Only checkbox and you will notice it's empty.

Here's what we did wrong. We took a calculator and pressed a + b, but we didn't press the = to see the result, the equivalent of = is using System.debug();.

System.debug();se utiliza para generar texto y variables en el registro. Los programadores utilizan esta práctica línea de código para depurar y ver los valores de las variables durante todo su tiempo de ejecución. La sintaxis para usarlo es:

System.debug(value);

Where value can be any Data type. Here are some examples:

System.debug(c);
System.debug("This is some text");
System.debug(45);

Añadamos esto a nuestro código y ejecutemos:

Integer a = 5;
Integer b = 4;
Integer c = a + b;
System.debug(c);

¡Ahora, haga clic en Depurar solo y notará que su respuesta está ahí!

Descripción de los registros

  • 03:51:26:002es la marca de tiempo de cuando se ejecutó el código. Esto está en el formato HH:MM:ss:SS
  • USER_DEBUG [4]|DEBUG|9 is the event identifier. This includes information on who ran the code, the line which output the log, and the value from System.debug(); itself.

Operaciones aritméticas básicas

  • Adición

    • Addition is done by using the + symbol between values / variables 

    Integer a = 4;
Integer b = 5;
Integer c = a + b;
Integer d = 4 + 5;
Integer e = a + 5;

System.debug(c);
System.debug(d);
System.debug(e);
    Los resultados son 9 en cada caso.

  • Resta

    • Subtraction is done by using the - symbol between values / variables. This will subtract the value on the right from the value on the left. 

    Integer a = 4;
Integer b = 5;
Integer c = a - b;
Integer d = b - a;
Integer e = 4 - 5;
Integer f = a - 5;

System.debug(c);
System.debug(d);
System.debug(e);
System.debug(f);
    Los resultados para c, e y f son -1, mientras que d es 1.

  • Multiplicación

    • Multiplication is done by using the * symbol between values / variables 

    Integer a = 4;
Integer b = 5;
Integer c = a * b;
Integer d = 4 * 5;
Integer e = a * 5;

System.debug(c);
System.debug(d);
System.debug(e);

Integer f = -4;
Integer g = a * f;
System.debug(g);
    Los resultados para c, d y e son 20, mientras que g resultan -20.

  • División

    • Division is done by using the / symbol between values / variables. The value on the left is the numerator, and the one on the right is the denominator. 

    Integer a = 4;
Integer b = 5;
Integer c = a / b;
Integer d = 4 / 5;
Integer e = a / 5;

System.debug(c);
System.debug(d);
System.debug(e);
Antes de seguir leyendo, ejecute el ejemplo anterior y lea el registro. Las respuestas son bastante interesantes 0, 0 y 0. ¡Tampocurre que no hay decimales involucrados!
Esto sucede porque un tipo de datos enteros solo acepta números enteros. Si quisiéramos el número exacto, un tipo de datos decimal o doble sería más relevante. Pruebe este código en Anon exec:

Integer a = 4;
Integer b = 5;

Decimal c = a / b;
System.debug(c);

Al leer el registro, ¡La respuesta sigue siendo 0! ¡Esto se debe a que mientras c es un decimal, a y b siguen siendo enteros! Cambiemos nuestro código de nuevo:

Decimal a = 4.0;
Decimal b = 5.0;
Decimal c = a / b;

System.debug(c);

¡La respuesta es 0.8! Esto sucede porque tenemos solo 1 carácter después del punto decimal. Tómese su tiempo para experimentar alrededor de esto. ¿Qué pasa si a es 4.0 pero b es 5.00? ¿Qué pasa si a es un entero y b es un decimal? ¡Aquí (todavía) no hay respuestas incorrectas!

Utilicemos varios operadores ahora. Haga 5 enteros, asigne números enteros y decimales al azar y utilice múltiples operadores entre ellos. Aquí está mi código:

Integer a = 45;
Integer b = 18;
Decimal c = 4.0;
Decimal d = 5.0;
System.debug(a + b / c * d);
Antes de ejecutar el código, use un lápiz y papel y calcule usando la regla BODMAS y PEMDAS antes de comparar valores en la consola. ¿Notó una diferencia? Al igual que BODMAS y PEMDAS, Apex tiene su propio orden de operaciones aritméticas:

Precedencia Operadores Descripción
1 {} () ++ -- Agrupar y prefijar incrementos y decrementos
2 nuevo Negación unaria, tipo fundido y creación de objetos
3 * / Multiplicación y división
4 + - Suma y resta
5 < <= > >= instancia de Comparaciones mayores e inferiores a las pruebas de referencia
6 == ! = Comparaciones: igual y no igual
7 && Lógica Y
8 || ÓR lógico
9 = += -= *= /= &= Operadores de asignación

Si queremos seguir una forma específica de resolver una operación, la encerramos entre paréntesis ( ). Los soportes siempre se resuelven de adentro hacia afuera.

Interpolación de cadenas

¿Alguna vez se ha preguntado cómo su instancia de Salesforce org dice "Buenos días <Su nombre>"? Esto es posible agregando una variable a una cadena y se llama Interpolación de Cadenas. Pruebe este código:

String myName = 'Harshdeep';
String hello = 'Hello ' + myName;
System.debug(hello);
y el resultado seráHello Harshdeep

Aparte de la personalización, es realmente útil para escribir registros para hacerlos más legibles. Escribamos un poco más de código:

Integer a = 5;
Integer b = 10;
Integer c = a + b;
String answer = 'The answer is ' + c;
System.debug(answer);

The logs are so much easier to read now!. Take your time to make different variations using string interpolation. What if you want to output The answer is 15!?

Refactorización de código

Mientras que preparar su propio café es divertido, el proceso de limpieza puede ser ligeramente infurante porque terminó usando 5 tazas, 2 cuencos, una olla, una prensa francesa, pocas cucharas y una taza medidora que necesita ser limpiada. Para lidiar con esto, la mayoría de nosotros simplemente usaríamos menos utensilios. Esto se puede hacer usando solo 1 taza para medir y volver a cablear todo nuestro proceso de fabricación de café para hacer menos desorden. Esto también se aplica al código. Echemos un vistazo a un simple ejemplo de adición:

integer a = 4;
integer b = 5;
integer c = a + b;
System.debug(c);
Hicimos 3 variables aquí. Uno tiene el número 4. Uno tiene el número 5. Uno sostiene el resultado de la suma. Mientras que un programa simple como este no tarda mucho en ejecutarse, cuando escalamos para escribir programas más grandes y complejos, se necesitará demasiada memoria y potencia informática para ejecutarse.

Esto es comparable al simple acto de apagar las luces cuando sale de su habitación para ahorrar electricidad. Haces una pequeña diferencia hacia tu consumo de electricidad, pero cuando toda la familia lo hace, luego todo el vecindario y luego toda la ciudad, hay una reducción dramática en el consumo de electricidad.

Refactorizar código significa escribir código que tiene bloques reutilizables que terminan funcionando más rápido y tomando menos memoria, mientras tiene la misma salida.

Refactoricemos nuestro código. Primero, eliminemos la variable c y demos directamente a+b en nuestro registro:

Integer a = 4;
Integer b = 5;
System.debug(a+b);

Now, let's declare both our integers in the same line, using a , as a separator:

Integer a = 4, b = 5;
System.debug(a+b);
Observe cómo pasamos de 4 líneas de código a 2. También usamos una variable menos y obtenemos el mismo resultado, mientras que también reutilizamos las variables que ya teníamos. La refactorización de código jugará un papel súper importante más adelante en un código más complejo.

La refactorización de código es importante porque las organizaciones de Salesforce se ejecutan en un entorno multiinquilino. Esto es similar a vivir en un apartamento con extraños y compartir pasillos y cocina, pero tener una habitación personal. Del mismo modo, varias organizaciones se ejecutan en un servidor y utilizan recursos informáticos compartidos con límites llamados Límites del Gobernador. Exploraremos los límites del gobernador en profundidad más adelante en este curso.

Refactorización de operaciones aritméticas

Ciertas operaciones aritméticas se pueden reescribir para facilitar la legibilidad. Esto será útil cuando se utilice bucles.

  • ++añade 1 al valor y lo devuelve a asignar a la variable.
  • --resta 1 del valor y lo devuelve a la variable.
  • *=multiplica la variable con el número de la derecha y la asigna de nuevo a la variable.

    • a *= 10;es equivalente aa = a * 10;

  • /=divide la variable con el número a la derecha y la asigna de nuevo a la variable.

    • a /= 10;es equivalente aa = a / 10;

Resumen

  • Es importante cuidar tu salud como programador desde el día 1 podrías confiar en que el autor te lo diga el día 3.
  • Los resultados de las operaciones aritméticas dependen del tipo de datos de los valores primarios.
  • System.debug();es una herramienta potente y necesaria para registrar valores durante el tiempo de ejecución.
  • Es importante refactorizar el código y valorar la potencia y los recursos informáticos.

Nota

El autor es consciente de que hay muchos conceptos marcados, ya que trataremos esto más adelante. El curso asume que esta es la primera vez que escribe código y la sobrecarga con mucha información que no tiene sentido inmediatamente puede ser contraproducente.
