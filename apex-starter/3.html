Console de développement, opérations arithmétiques, interpolation de chaînes et refactoring de code

Installation du mode sombre pour Salesforce Extension pour Chrome [Recommandé]

En tant que programmeur, vous passerez des heures à regarder un écran à trouver des solutions et à écrire du code, ce qui rend vraiment important de protéger vos yeux. Une bonne façon de réduire le stress sur les yeux est d'utiliser le mode sombre dans la mesure du possible. Salesforce ne fournit pas d'interface utilisateur en mode sombre intégrée, mais il est possible d'y arriver à l'aide d'une extension de navigateur. Celui que je recommande est le mode sombre pour la vente.

Téléchargez l'extension à partir du Chrome Web Store et suivez les instructions à l'écran

Toutes les captures d'écran tout au long du cours utiliseront Darkmode dans la mesure du possible. Notez que cela ne change aucune fonctionnalité dans votre organisation Salesforce, il suffit de changer la couleur des éléments de l'interface utilisateur.

La console de développement

Console de développement ouverte.

Vous devriez être accueilli avec un écran similaire à celui-ci :

    Barre de menu supérieure

  1. Fichier : Ce menu contient des options pour ouvrir, enregistrer, fermer et supprimer des fichiers et des ressources.
  2. Édition : Ce menu contient des options pour trouver et remplacer, corriger l'indentation et la mise en forme du code.
  3. Debug : Ce menu contient des options relatives au débogage et au code de correction d'erreurs.
  4. Test : Ce menu contient des options liées au test de notre code et à l'assurance que notre code fait ce qu'il est censé faire.
  5. Espace de travail : Ce menu contient des options pour configurer l'interface utilisateur de la console de développement.
  6. Aide : Ce menu contient des rubriques d'aide générales telles que des liens vers la documentation, des touches de raccourci et des préférences pour la console de développement.

    7. Éditeur de texte

  7. Onglets : Cet espace contient tous les fichiers ouverts dans un format à onglets.
  8. Éditeur : C'est là que nous écrivons notre code.

    9. Barre de menu inférieure

  9. Journaux : Cet onglet contient tous les journaux des exécutions en cours et précédentes.
  10. Tests : Cet onglet contient tous les rapports de test de notre code comme la couverture de code et les résultats de réussite/échec.
  11. Points de contrôle : Cet onglet contient des informations sur les points de contrôle lors de la configuration dans le code.
  12. Éditeur de requêtes : Cette option permet au développeur d'exécuter des requêtes SOSL et SOQL et de les valider.
  13. État de la vue : Cet onglet vous permet d'examiner l'état de la vue pour la demande de page Visualforce.
  14. Progression : Cet onglet montre la progression de l'exécution du code actuel et précédent.
  15. Problèmes : Cet onglet affiche, en temps quasi réel, les erreurs disponibles, le cas échéant, lors de l'écriture de notre code.

Écrire du code

Now that we know where we write our code, let's write some of our first programs! Head over to Debug > Open execute anonymous window. This window allows us to write snippets of code. Alternativelt, you can press Ctrl and E to open quicker.

Faisons 2 entiers et ajoutons-les :

Integer a = 5;
Integer b = 4;
a + b;

Assurez-vous que l'option "Ouvrir les journaux après l'exécution" est cochée. Votre écran devrait ressembler à ceci :

Vous remarquerez que vous êtes accueilli avec cet écran :

Line 3, Column 1:
Expression cannot be a statment

The code in line 3, a+b is the equivalent to mixing coffee and water in nothing. We need a variable (cup) to hold them. Let's fix that:

Integer a = 5;
Integer b = 4;
Integer c = a + b;

Vous remarquerez que vous êtes automatiquement redirigé vers cet écran

Click on Debug Only checkbox and you will notice it's empty.

Here's what we did wrong. We took a calculator and pressed a + b, but we didn't press the = to see the result, the equivalent of = is using System.debug();.

System.debug();est utilisé pour afficher du texte et des variables dans le journal. Les programmeurs utilisent cette ligne de code pratique pour déboguer et voir les valeurs des variables tout au long de son exécution. La syntaxe pour l'utiliser est la suivante :

System.debug(value);

Where value can be any Data type. Here are some examples:

System.debug(c);
System.debug("This is some text");
System.debug(45);

Ajoutons ceci à notre code et exécutons :

Integer a = 5;
Integer b = 4;
Integer c = a + b;
System.debug(c);

Maintenant, cliquez sur Déboguer uniquement et vous remarquerez que votre réponse est là !

Comprendre les journaux

  • 03:51:26:002est l'horodatage de la date d'exécution du code. C'est dans le format HH:MM:ss:SS
  • USER_DEBUG [4]|DEBUG|9 is the event identifier. This includes information on who ran the code, the line which output the log, and the value from System.debug(); itself.

Opérations arithmétiques de base

  • Ajout

    • Addition is done by using the + symbol between values / variables 

    Integer a = 4;
Integer b = 5;
Integer c = a + b;
Integer d = 4 + 5;
Integer e = a + 5;

System.debug(c);
System.debug(d);
System.debug(e);
    Les résultats sont de 9 dans tous les cas.

  • Soustraction

    • Subtraction is done by using the - symbol between values / variables. This will subtract the value on the right from the value on the left. 

    Integer a = 4;
Integer b = 5;
Integer c = a - b;
Integer d = b - a;
Integer e = 4 - 5;
Integer f = a - 5;

System.debug(c);
System.debug(d);
System.debug(e);
System.debug(f);
    Les résultats pour c, e et f sont -1, tandis que d est 1.

  • Multiplication

    • Multiplication is done by using the * symbol between values / variables 

    Integer a = 4;
Integer b = 5;
Integer c = a * b;
Integer d = 4 * 5;
Integer e = a * 5;

System.debug(c);
System.debug(d);
System.debug(e);

Integer f = -4;
Integer g = a * f;
System.debug(g);
    Les résultats pour c, d et e sont de 20, tandis que g résultats de -20.

  • Division

    • Division is done by using the / symbol between values / variables. The value on the left is the numerator, and the one on the right is the denominator. 

    Integer a = 4;
Integer b = 5;
Integer c = a / b;
Integer d = 4 / 5;
Integer e = a / 5;

System.debug(c);
System.debug(d);
System.debug(e);
Avant de lire plus loin, exécutez l'exemple ci-dessus et lisez le journal. Les réponses sont assez intéressantes 0, 0 et 0. Il se trouve qu'il n'y a pas de décimales non plus !
Cela se produit parce qu'un type de données entier n'accepte que des nombres entiers. Si nous voulions le nombre exact, un type de données décimal ou double serait plus pertinent. Essayez ce code dans Anon exec :

Integer a = 4;
Integer b = 5;

Decimal c = a / b;
System.debug(c);

A la lecture du journal, la réponse est toujours 0 ! C'est parce que si c est une décimale, a et b sont toujours des entiers ! Commutons à nouveau notre code :

Decimal a = 4.0;
Decimal b = 5.0;
Decimal c = a / b;

System.debug(c);

La réponse est 0,8 ! Cela se produit parce que nous n'avons qu'un seul caractère après la virgule. Prenez votre temps pour expérimenter autour de cela. Que se passe-t-il si a vaut 4,0 mais b vaut 5,00 ? Que se passe-t-il si a est un entier et b est une décimale ? Il n'y a pas de mauvaises réponses ici (encore) !

Allons utiliser plusieurs opérateurs maintenant. Faites 5 entiers, assignez-leur des nombres entiers et des décimales au hasard et utilisez plusieurs opérateurs entre eux. Voici mon code :

Integer a = 45;
Integer b = 18;
Decimal c = 4.0;
Decimal d = 5.0;
System.debug(a + b / c * d);
Avant d'exécuter le code, utilisez un stylo et du papier et calculez à l'aide de la règle BODMAS et PEMDAS avant de comparer les valeurs dans la console. Vous avez remarqué une différence ? Tout comme BODMAS et PEMDAS, Apex a son propre ordre d'opérations arithmétiques :

Préséance Opérateurs Description
1 {} () ++ -- Regroupement et préfixe incréments et décréments
2 nouveau Négation unaire, conversion de type et création d'objets
3 * / Multiplication et division
4 + - Addition et soustraction
5 < <= > >= instanceof Comparaisons supérieures et inférieures aux comparaisons, tests de référence
6 == ! = Comparaisons : égales et non égales
7 && Logique ET
8 || Logique OU
9 = += -= *= /= &= Opérateurs d'affectation

If we want to follow a specific way of solving an operation, we enclose it in brackets ( ). Brackets are always solved inside out.

Interpolation de chaînes

Vous êtes-vous déjà demandé comment votre instance d'organisation Salesforce dit "Bonjour <Votre nom>" ? Ceci est possible en ajoutant une variable à une chaîne et s'appelle String Interpolation. Essayez ce code :

String myName = 'Harshdeep';
String hello = 'Hello ' + myName;
System.debug(hello);
et le résultat seraHello Harshdeep

Outre la personnalisation, il est vraiment utile pour écrire des journaux pour les rendre plus lisibles. Écrivons un peu plus de code :

Integer a = 5;
Integer b = 10;
Integer c = a + b;
String answer = 'The answer is ' + c;
System.debug(answer);

The logs are so much easier to read now!. Take your time to make different variations using string interpolation. What if you want to output The answer is 15!?

Refactoring du code

Bien que l'infusion de votre propre café soit amusant, le processus de nettoyage peut être légèrement furieux car vous avez fini par utiliser 5 tasses, 2 bols, une casserole, une presse française, quelques cuillères et une tasse à mesurer qui doit être nettoyée. Pour y faire face, la plupart d'entre nous utiliseraient simplement moins d'ustensiles. Cela peut être fait en utilisant seulement 1 tasse pour mesurer et recâbrer l'ensemble de notre processus de fabrication du café afin de faire moins de gâchis. Cela s'applique également au code. Jetons un coup d'œil à un exemple d'ajout simple :

integer a = 4;
integer b = 5;
integer c = a + b;
System.debug(c);
Nous avons fait 3 variables ici. L'un d'eux détient le numéro 4. L'un d'eux détient le numéro 5. On détient le résultat de l'addition. Bien qu'un programme simple comme celui-ci ne prenne pas beaucoup à exécuter, lorsque nous faisons évoluer pour écrire des programmes plus grands et plus complexes, il faudra trop de mémoire et de puissance de calcul pour s'exécuter.

Cela est comparable au simple fait d'éteindre vos lumières lorsque vous quittez votre pièce pour économiser de l'électricité. Vous faites une petite différence dans votre consommation d'électricité, mais lorsque toute la famille le fait, puis tout le quartier, puis toute la ville, il y a une réduction spectaculaire de la consommation d'électricité.

Le refactoring de code signifie écrire du code qui a des blocs réutilisables qui finissent par s'exécuter plus rapidement et prendre moins de mémoire, tout en ayant la même sortie.

Refactorisons notre code. Tout d'abord, supprimons la variable c et sortons directement a+b dans notre journal :

Integer a = 4;
Integer b = 5;
System.debug(a+b);

Now, let's declare both our integers in the same line, using a , as a separator:

Integer a = 4, b = 5;
System.debug(a+b);
Remarquez comment nous sommes passés de 4 lignes de code à 2. Nous utilisons également une variable de moins et nous obtenons le même résultat, tout en réutilisant les variables que nous avions déjà. Le refactoring de code jouera un rôle super important plus tard dans le code plus complexe.

Le refactoring de code est important parce que les organisations Salesforce s'exécutent sur un environnement multilocataire. C'est similaire à vivre dans un appartement avec des étrangers et à partager des couloirs et une cuisine, mais avoir une chambre personnelle. De même, plusieurs organisations s'exécutent sur un serveur et utilisent des ressources informatiques partagées avec des limites appelées limites de gouverneur. Nous explorerons en profondeur les limites des gouverneurs plus tard dans ce cours.

Refactorisation des opérations arithmétiques

Certaines opérations arithmétiques peuvent être réécrites pour faciliter la lisibilité. Cela sera pratique lors de l'utilisation de boucles.

  • ++ajoute 1 à la valeur et l'affecte à la variable.
  • --soustrait 1 de la valeur et l'affecte à la variable.
  • *=multiplie la variable par le nombre à droite et l'affecte à la variable.

    • a *= 10;équivaut àa = a * 10;

  • /=divise la variable avec le numéro à droite et l'affecte à la variable.

    • a /= 10;équivaut àa = a / 10;

Résumé

  • Il est important de prendre soin de votre santé en tant que programmeur dès le premier jour, vous pouvez compter sur l'auteur qui vous en parle le troisième jour.
  • Les résultats des opérations arithmétiques dépendent du type de données des valeurs parentes.
  • System.debug();est un outil puissant et nécessaire pour enregistrer les valeurs pendant l'exécution.
  • Il est important de refactoriser le code et de valoriser la puissance de calcul et les ressources.

Note

L'auteur est conscient qu'il y a beaucoup de concepts marqués comme nous en parlerons plus tard. Le cours suppose que c'est la première fois que vous écrivez du code et qu'une surcharge de beaucoup d'informations qui n'ont pas de sens immédiatement peut être contre-productive.
