Entwicklerkonsole, arithmetische Operationen, String-Interpolation und Code-Refactoring

Installieren von Dunkler Modus for Salesforce Extension for Chrome [Empfohlen]

Als Programmierer werden Sie Stunden damit verbringen, auf einen Bildschirm zu starren, um Lösungen zu finden und Code zu schreiben, was es wirklich wichtig macht, Ihre Augen zu schützen. Eine gute Möglichkeit, den Stress für die Augen zu reduzieren, ist, wo immer möglich, der dunkle Modus zu verwenden. Salesforce bietet keine integrierte Benutzeroberfläche im Dunkelmodus, aber es ist möglich, mit einer Browsererweiterung dorthin zu gelangen. Die, die ich empfehle, ist der Dunkelmodus für Salesorce.

Laden Sie die Erweiterung aus dem Chrome Web Store herunter und folgen Sie den Anweisungen auf dem Bildschirm

Alle Screenshots während des gesamten Kurses verwenden, wo immer möglich, den Dunkelmodus. Beachten Sie, dass dies keine Funktionalität in Ihrer Salesforce-Organisation ändert, sondern nur die Farbe der UI-Elemente.

Die Entwicklerkonsole

Öffnen Sie die Entwicklerkonsole.

Sie sollten mit einem Bildschirm begrüßt werden, der dem ähnelt:

    Obere Menüleiste

  1. Datei: Dieses Menü enthält Optionen zum Öffnen, Speichern, Schließen und Löschen von Dateien und Ressourcen.
  2. Bearbeiten: Dieses Menü enthält Optionen zum Suchen und Ersetzen, Beheben von Einrückungen und Codeformatierung.
  3. Debuggen: Dieses Menü enthält Optionen, die sich auf Debugging und Fehlerbehebungscode beziehen.
  4. Test: Dieses Menü enthält Optionen zum Testen unseres Codes und zum Sicherstellen, dass unser Code das tut, was er soll.
  5. Arbeitsbereich: Dieses Menü enthält Optionen zum Einrichten der Benutzeroberfläche für die Entwicklerkonsole.
  6. Hilfe: Dieses Menü enthält allgemeine Hilfethemen wie Links zur Dokumentation, Tastenkombinationen und Einstellungen für die Entwicklerkonsole.

    7. Texteditor

  7. Registerkarten: Dieser Bereich enthält alle geöffneten Dateien in einem Tabbed-Format.
  8. Editor: Hier schreiben wir unseren Code.

    9. Untere Menüleiste

  9. Protokolle: Diese Registerkarte enthält alle Protokolle aus aktuellen und vorherigen Durchläufen.
  10. Tests: Diese Registerkarte enthält alle Berichte aus dem Testen unseres Codes wie Codeabdeckung und Ergebnisse von bestanden/nicht bestanden.
  11. Checkpoints: Diese Registerkarte enthält Informationen über Checkpoints bei der Einrichtung im Code.
  12. Abfrage-Editor: Mit dieser Option kann der Entwickler SOSL- und SOQL-Abfragen ausführen und validieren.
  13. Ansichtsstatus: Auf dieser Registerkarte können Sie den Ansichtsstatus für die Visualforce-Seitenanforderung untersuchen.
  14. Fortschritt: Diese Registerkarte zeigt den Fortschritt bei der Ausführung des aktuellen und vorherigen Codes an.
  15. Probleme: Diese Registerkarte zeigt nahezu in Echtzeit die Fehler an, die beim Schreiben unseres Codes verfügbar sind, falls vorhanden.

Code schreiben

Now that we know where we write our code, let's write some of our first programs! Head over to Debug > Open execute anonymous window. This window allows us to write snippets of code. Alternativelt, you can press Ctrl and E to open quicker.

Machen wir 2 ganze Zahlen und fügen sie hinzu:

Integer a = 5;
Integer b = 4;
a + b;

Stellen Sie sicher, dass die Option "Protokolle nach der Ausführung öffnen" aktiviert ist. Ihr Bildschirm sollte etwa so aussehen:

Sie werden feststellen, dass Sie mit diesem Bildschirm begrüßt werden:

Line 3, Column 1:
Expression cannot be a statment

The code in line 3, a+b is the equivalent to mixing coffee and water in nothing. We need a variable (cup) to hold them. Let's fix that:

Integer a = 5;
Integer b = 4;
Integer c = a + b;

Sie werden feststellen, dass Sie automatisch zu diesem Bildschirm weitergeleitet werden.

Click on Debug Only checkbox and you will notice it's empty.

Here's what we did wrong. We took a calculator and pressed a + b, but we didn't press the = to see the result, the equivalent of = is using System.debug();.

System.debug();Wird verwendet, um Text und Variablen im Protokoll auszugeben. Programmierer verwenden diese praktische Codezeile, um die Werte von Variablen während der gesamten Laufzeit zu debuggen und anzuzeigen. Die Syntax für die Verwendung lautet:

System.debug(value);

Where value can be any Data type. Here are some examples:

System.debug(c);
System.debug("This is some text");
System.debug(45);

Fügen wir dies unserem Code hinzu und führen wir Folgendes aus:

Integer a = 5;
Integer b = 4;
Integer c = a + b;
System.debug(c);

Klicken Sie jetzt auf Nur Debuggen und Sie werden feststellen, dass Ihre Antwort da ist!

Grundlegendes zu Protokollen

  • 03:51:26:002Ist der Zeitstempel, zu dem der Code ausgeführt wurde. Dies hat das Format HH:MM:ss:SS
  • USER_DEBUG [4]|DEBUG|9 is the event identifier. This includes information on who ran the code, the line which output the log, and the value from System.debug(); itself.

Grundlegende arithmetische Operationen

  • Addition

    • Die Addition erfolgt durch die Verwendung des +-Symbols zwischen Werten / Variablen

    Integer a = 4;
Integer b = 5;
Integer c = a + b;
Integer d = 4 + 5;
Integer e = a + 5;

System.debug(c);
System.debug(d);
System.debug(e);
    Die Ergebnisse sind in jedem Fall 9.

  • Subtraktion

    • Subtraction is done by using the - symbol between values / variables. This will subtract the value on the right from the value on the left. 

    Integer a = 4;
Integer b = 5;
Integer c = a - b;
Integer d = b - a;
Integer e = 4 - 5;
Integer f = a - 5;

System.debug(c);
System.debug(d);
System.debug(e);
System.debug(f);
    Die Ergebnisse für c, e und f sind -1, während d 1 ist.

  • Multiplikation

    • Die Multiplikation erfolgt mit dem *-Symbol zwischen Werten / Variablen

    Integer a = 4;
Integer b = 5;
Integer c = a * b;
Integer d = 4 * 5;
Integer e = a * 5;

System.debug(c);
System.debug(d);
System.debug(e);

Integer f = -4;
Integer g = a * f;
System.debug(g);
    Die Ergebnisse für c, d und e sind 20, während g auf -20 ergibt.

  • Abteilung

    • Division is done by using the / symbol between values / variables. The value on the left is the numerator, and the one on the right is the denominator. 

    Integer a = 4;
Integer b = 5;
Integer c = a / b;
Integer d = 4 / 5;
Integer e = a / 5;

System.debug(c);
System.debug(d);
System.debug(e);
Bevor Sie weiterlesen, führen Sie das obige Beispiel aus und lesen Sie das Protokoll. Die Antworten sind ziemlich interessant 0, 0 und 0. Es geht auch nicht um Dezimalstellen!
Dies geschieht, weil ein ganzzahliger Datentyp nur ganze Zahlen akzeptiert. Wenn wir die genaue Zahl wollten, wäre ein Dezimal- oder Doppeldatentyp relevanter. Probieren Sie diesen Code in Anon exec aus:

Integer a = 4;
Integer b = 5;

Decimal c = a / b;
System.debug(c);

Beim Lesen des Protokolls ist die Antwort immer noch 0! Das liegt daran, dass c zwar eine Dezimalstelle ist, a und b aber immer noch ganze Zahlen sind! Wechseln wir unseren Code erneut:

Decimal a = 4.0;
Decimal b = 5.0;
Decimal c = a / b;

System.debug(c);

Die Antwort ist 0,8! Dies geschieht, weil wir nur 1 Zeichen nach dem Dezimalzeichen haben. Nehmen Sie sich Zeit, um damit zu experimentieren. Was ist, wenn a 4,0, aber b 5,00 ist? Was ist, wenn a eine Ganzzahl und b eine Dezimalzahl ist? Es gibt hier (noch) keine falschen Antworten!

Lassen Sie uns jetzt mehrere Operatoren verwenden. Machen Sie 5 ganze Zahlen, weisen Sie ihnen zufällig ganze Zahlen und Dezimalstellen zu und verwenden Sie mehrere Operatoren zwischen ihnen. Hier ist mein Code:

Integer a = 45;
Integer b = 18;
Decimal c = 4.0;
Decimal d = 5.0;
System.debug(a + b / c * d);
Bevor Sie den Code ausführen, verwenden Sie Stift und Papier und berechnen Sie mit der BODMAS- und PEMDAS-Regel, bevor Sie die Werte in der Konsole vergleichen. Haben Sie einen Unterschied bemerkt? Genau wie BODMAS und PEMDAS hat Apex eine eigene Reihenfolge der arithmetischen Operationen:

Vorrang Betreiber Beschreibung
1 {} () ++ -- Gruppierungs- und Präfix-Schritte und -dekremente
2 neu Unäre Negation, Type Cast und Objekterstellung
3 * / Multiplikation und Division
4 + - Addition und Subtraktion
5 < <= > >= Instanz von Vergleich größer als und kleiner als, Referenztests
6 == ! = Vergleiche: gleich und ungleich
7 && Logisches UND
8 || Logisches ODER
9 = += -= *= /= &= Zuweisungsoperatoren

If we want to follow a specific way of solving an operation, we enclose it in brackets ( ). Brackets are always solved inside out.

String-Interpolation

Haben Sie sich jemals gefragt, wie Ihre Instanz der Salesforce-Organisation "Guten Morgen <Ihr Name>" sagt? Dies ist möglich, indem Sie einer Zeichenfolge eine Variable hinzufügen und wird String Interpolation genannt. Probieren Sie diesen Code aus:

String myName = 'Harshdeep';
String hello = 'Hello ' + myName;
System.debug(hello);
Und das Ergebnis wird seinHello Harshdeep

Abgesehen von der Personalisierung ist es wirklich nützlich, Protokolle zu schreiben, um sie lesbarer zu machen. Schreiben wir noch mehr Code:

Integer a = 5;
Integer b = 10;
Integer c = a + b;
String answer = 'The answer is ' + c;
System.debug(answer);

The logs are so much easier to read now!. Take your time to make different variations using string interpolation. What if you want to output The answer is 15!?

Code-Refactoring

Während das Brauen Ihres eigenen Kaffees Spaß macht, kann der Aufräumvorgang leicht auslaufend sein, da Sie am Ende 5 Tassen, 2 Schüsseln, einen Topf, eine französische Presse, ein paar Löffel und einen Messbecher verwendet haben, der gereinigt werden muss. Um damit fertig zu werden, würden die meisten von uns einfach weniger Utensilien verwenden. Dies kann erreicht werden, indem nur 1 Tasse zum Messen und Neuverdrahten unseres gesamten Kaffeeherstellungsprozesses verwendet wird, um weniger Chaos zu verursachen. Dies gilt auch für Code. Werfen wir einen Blick auf ein einfaches Additionsbeispiel:

integer a = 4;
integer b = 5;
integer c = a + b;
System.debug(c);
Wir haben hier 3 Variablen gemacht. Man hält die Nummer 4. Man hält die Nummer 5. Man hält das Ergebnis der Addition. Während ein einfaches Programm wie dieses nicht viel zum Ausführen benötigt, benötigt es zu viel Speicher und Rechenleistung, wenn wir skalieren, um größere, komplexere Programme zu schreiben.

Dies ist vergleichbar mit dem einfachen Akt, Ihr Licht auszuschalten, wenn Sie Ihr Zimmer verlassen, um Strom zu sparen. Sie machen einen kleinen Unterschied zu Ihrem Stromverbrauch, aber wenn die ganze Familie es tut, dann die gesamte Nachbarschaft und dann die ganze Stadt, gibt es eine dramatische Reduzierung des Stromverbrauchs.

Code-Refactoring bedeutet, Code zu schreiben, der wiederverwendbare Blöcke enthält, die am Ende schneller laufen und weniger Speicher benötigen, während sie die gleiche Ausgabe haben.

Lassen Sie uns unseren Code umgestalten. Entfernen wir zunächst die Variable c und geben wir direkt a+b in unserem Protokoll aus:

Integer a = 4;
Integer b = 5;
System.debug(a+b);

Deklarieren wir nun unsere beiden ganzen Zahlen in derselben Zeile mit einem als Trennzeichen:

Integer a = 4, b = 5;
System.debug(a+b);
Beachten Sie, wie wir von 4 Codezeilen auf 2 gestiegen sind. Wir verwenden auch eine Variable weniger und erhalten das gleiche Ergebnis, während wir gleichzeitig die Variablen, die wir bereits hatten, wiederverwenden. Code-Refactoring wird später in komplexerem Code eine super wichtige Rolle spielen.

Code-Refactoring ist wichtig, da Salesforce-Organisationen in einer mehrinstanzenfähigen Umgebung ausgeführt werden. Dies ähnelt dem Leben in einer Wohnung mit Fremden und der gemeinsamen Nutzung von Fluren und Küche, aber mit einem persönlichen Zimmer. Ebenso werden mehrere Organisationen auf einem Server ausgeführt und verwenden gemeinsam genutzte Computing-Ressourcen mit Limits, die als Governor Limits bezeichnet werden. Wir werden die Grenzen des Gouverneurs später in diesem Kurs eingehend untersuchen.

Refactoring arithmetischer Operationen

Bestimmte arithmetische Operationen können neu geschrieben werden, um die Lesbarkeit zu erleichtern. Dies wird nützlich sein, wenn Schleifen verwendet werden.

  • ++Addiert 1 zum Wert und weist ihn der Variablen zurück.
  • --Subtrahiert 1 vom Wert und weist ihn der Variablen zurück.
  • *=Multipliziert die Variable mit der Zahl auf der rechten Seite und weist sie der Variablen zurück.

    • a *= 10;Ist äquivalent zua = a * 10;

  • /=Teilt die Variable durch die Zahl auf der rechten Seite und weist sie der Variablen zurück.

    • a /= 10;Ist äquivalent zua = a / 10;

Zusammenfassung

  • Es ist wichtig, sich als Programmierer ab Tag 1 um Ihre Gesundheit zu kümmern, können Sie sich darauf verlassen, dass der Autor Sie am 3. Tag davon erzählt.
  • Die Ergebnisse arithmetischer Operationen hängen vom Datentyp der übergeordneten Werte ab.
  • System.debug();Ist ein leistungsstarkes und notwendiges Werkzeug, um Werte während der Laufzeit zu protokollieren.
  • Es ist wichtig, Code umzugestalten und Rechenleistung und Ressourcen zu schätzen.
