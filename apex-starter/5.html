Liste, ensemble, cartes et boucles

Listes, ensembles, cartes

La liste est un type de données ordonné qui contient plusieurs valeurs. Chaque valeur a sa place dans la liste où le comptage commence à partir de 0. Voici la syntaxe :

List<dataType> variableName = new List<dataType>();
  • List<dataType>définit le type de données que la liste contiendra
  • variableNamedéfinit le nom utilisé pour faire référence à cette liste
  • newdéfinit que nous faisons une nouvelle liste
  • List<dataType>()définit que nous faisons une liste vierge.

Pour ajouter une valeur dans la liste, nous suivons la syntaxe :

variableName.add(value);

Voici le code en action :

List<String> names = new List<String>();

names.add('Harshdeep');
names.add('Cobey');
names.add('Zerab');

System.debug(names);

Si nous voulons accéder à un élément spécifique de la liste, nous utilisons la syntaxe suivante :

 variableName[n];
  • variableNameest le nom de la liste
  • [n]est l'élément numérique auquel nous essayons d'accéder, entre crochets

Voyons cela dans la pratique :

List<String> names = new List<String>();

names.add('Harshdeep');
names.add('Cobey');
names.add('Zerab');

System.debug(names[3]);

L'erreur suivante s'affichera :

Line: 7, Column: 1
System.ListException: List index out of bounds: 3

C'est parce que tout comptage commence à partir de 0, donc pour accéder au troisième élément, nous devons en soustraire 1 parce qu'en interne la liste est stockée de la manière suivante :

Indice Nom
0 Dur profond
1 Cobey
2 Zerab

Maintenant, corrigeons le code en supprimant la dernière ligne avec :

System.debug(names[0]);
System.debug(names[2]);

Maintenant, le journal affiche le prénom et le nom de famille ! C'est super important à retenir. Tout comptage commence à partir de 0.

Set est une liste non ordonnée qui ne contient aucun doublon. Voici comment vous faites un ensemble :

Set<dataType> variableName = new Set<dataType>();
  • Set<dataType>est de déclarer le type Set of dataType.
  • variableNameest de faire référence à cet ensemble.
  • newdéfinit qu'il s'agit d'une nouvelle variable.
  • Set<dataType>()est d'initialiser l'ensemble vide avec ledit type de données.

Voyons Set in action :

Set<String> names = new Set<String>();
names.add('Harshdeep');
names.add('Cobey');
names.add('Zerab');
names.add('Cobey');

System.debug(names);

Lorsque vous regardez les journaux, le nom en double a été supprimé et l'ordre a également été modifié. Les ensembles ne sont généralement pas beaucoup utilisés dans la programmation pratique autant que les listes ou les cartes. Une utilisation pratique des ensembles serait dans un scénario où de mauvaises données ont été saisies, et au lieu d'exporter toutes les données et de parcourir une feuille de calcul, toutes les données sont stockées dans un ensemble et des valeurs uniques sont immédiatement extraites pour restructurer les données.

Map contient des paires de valeurs clés. Cela signifie que chaque élément aura deux valeurs. La syntaxe pour la création d'ensembles est la suivante :

Map<dataType1, dataType2> variableName = new Map<dataType1, dataType2>();
  • Map<dataType1, dataType2>est d'initialiser la variable map. dataType 1 et 2 peuvent être les mêmes types de données ou différents.
  • variableNameest le nom de la variable à faire référence à new est de déclarer qu'il s'agit d'une nouvelle variable
  • Map<dataType1, dataType2>()est d'initialiser une carte vide du même type de données.
    • Map<dataType1, dataType2> variableName = new Map<dataType1, dataType2>();
variableName.put(key, value);

    You insert values in your Map by using mapVariable.put(key, value);, where key and value are dataType variables as defined in their declaration. An easier way to understand is ordering food in a drive through. You ask for Number 1, 2 and 6 and you're given the food corresponding to those numbers.

    To access value from a Map, we use .get(key).
    Un exemple de cartes avec le même type de données :

    Map<String, String> placeCapital = new Map<String, String>();
placeCapital.put('New Delhi', 'India');
placeCapital.put('Captial', 'Country');

String ourCapital = placeCapital.get('New Delhi');
System.debug(ourCapital);

    Cela retournera "Inde" dans votre journal.

    Comprendre quand utiliser les ensembles, les listes et les cartes

    Ensemble

    D'après mon expérience, j'ai rarement utilisé des ensembles. Deux problèmes très distincts étaient plus faciles à résoudre avec Set que partout ailleurs :

    1. Une organisation avait un champ Pays de facturation défini sur Texte au lieu de la liste de sélection, et pour chaque pays, il y avait plusieurs valeurs, certaines récurrentes plus que d'autres. Au lieu d'exporter des données vers une feuille de calcul, d'exécuter des formules et de les télécharger, il était plus efficace d'écrire un ensemble qui rassemblait toutes les valeurs uniques dans une table, puis de le traiter davantage pour résoudre le problème.
    2. Une organisation avait une liste de diffusion personnalisée sans restriction sur le nombre de fois où vous pouviez vous inscrire. Lorsque certains gestionnaires manquaient des e-mails, ils choisissaient de s'inscrire à nouveau, ce qui entraînait l'envoi de plusieurs courriels au même utilisateur. Un moyen rapide de résoudre ce problème était de faire une boucle et d'attribuer toutes les valeurs de la liste à l'ensemble et de mettre à jour la liste finale.

    Dans les deux cas, c'était le résultat d'une mauvaise configuration par les administrateurs et les développeurs qui aurait pu être facilement évitée.

    Pour comprendre l'utilisation optimale de Lists v/s Maps, commençons par en savoir plus sur For Loops.

    Pour les boucles

    For loops est l'un des blocs de code les plus couramment utilisés dans la programmation Apex. La boucle A For peut être utilisée de 2 façons différentes :

    1. Pour exécuter un bloc de code n nombre de fois
    2. Pour itérer sur une liste

    Exécution d'un bloc de code n fois

    Pour faire cette boucle, nous avons besoin de trois choses :

    1. Une déclaration de départ
    2. Une condition de sortie
    3. Une instruction d'incrémentation

    La syntaxe est la suivante :

    for(start statement; exit condition; increment statement) {
  //code block
}
    • forest la déclaration de la boucle
    • startest ce qui s'exécutera au début de la boucle
    • exit conditionest ce qui arrêtera la boucle si elle renvoie false
    • increment statementsera exécuté chaque fois que le bloc de code finira de s'exécuter
    • ;est le séparateur entre chaque condition / instruction et est nécessaire

    Exécutez ce code dans la console du développeur :

    for(Integer i = 0; i < 10; i++){
  System.debug('Hello '+ i);
}

    Décomposons ce code. Voici ce qui s'est passé :

    1. Faites un entier i et affectez-lui la valeur 0
    2. Est-ce que je suis inférieur à 10 ? s'il est inférieur à 10, exécutez le bloc de code
    3. Après l'exécution du bloc de code, incrémentez la valeur de i par 1.
    4. Les instructions 2, 3 et 4 s'exécutent jusqu'à ce que le i soit égal à 10.

    Voici comment fonctionne pour la boucle en termes de café. Vous pesez votre café et la première cuillère n'avait que 50 grammes de café sur les 100 grammes. Une boucle for en termes de café serait :

    for (spoon = 0 grams; total_coffee < 100 grams; add another spoon){
  spoon.add(50 grams);
  weight.check();
}

    Nous commençons par 0 gramme dans notre cuillère et ajoutons 50 grammes de café avec chaque cuillère. Notre condition de sortie est de nous assurer que le café total dans notre tasse ne dépasse pas 100 grammes, nous continuons donc à ajouter du café dans des cuillères de 50 grammes jusqu'à ce que notre condition de sortie soit remplie.

    Boucle sur une liste ou un ensemble

    Pour faire cette boucle, nous avons besoin de deux choses :

    • Une liste ou un ensemble pour itérer sur
    • Une variable qui contiendra des valeurs temporairement

    La syntaxe est la suivante :

    for (dataType variableName: list_or_set){
    //code block
}
    • forest la déclaration de la boucle
    • dataTypeest le type de données de la variable
    • variableNameest la variable qui contiendra temporairement la valeur
    • list_or_setest une liste ou un ensemble existant

    Exécutez ce code dans la console du développeur :

    List<String> names=new List<String>();

names.add('Name 1');
names.add('Name 2');
names.add('Name 3');
names.add('Name 4');

for(String iterator: names){
  System.debug(iterator);
}

    Décomposons ce code. Voici ce qui s'est passé :

    1. Faites une liste et ajoutez-y des valeurs.
    2. La boucle for démarre
    3. Nous avons une variable nommée itérateur qui est de type string, qui confirme au type de données des noms de liste qui est une liste de chaînes
    4. Assignez la première valeur des noms à l'itérateur et exécutez le bloc de code
    5. Après l'exécution du bloc de code, affectez la valeur suivante dans names à l'itérateur et exécutez le bloc de code
    6. Si les noms de liste sont à court de valeurs, quittez la boucle.

    Une boucle For peut également être imbriquée à l'intérieur de la boucle for.

    Voici un exemple

    List<String> names=new List<String>();
names.add('Name 1');
names.add('Name 2');
names.add('Name 3');
names.add('Name 4');

List<String> surnames=new List<String>();
surnames.add('Name 3');
surnames.add('Name 1');
surnames.add('Name 4');
surnames.add('Name 2');

for (String outerLoop: names) {
  for (String innerLoop: surnames) {
    if (outerLoop==innerLoop) {
      System.debug(outerLoop + ' ' +innerLoop); //To add a space between both values
    }
  }
}

    Décomposons ce code :

    1. Faites 2 listes et attribuez-leur des valeurs
    2. outerLoop se voit attribuer la valeur Name 1 de la liste des noms.
    3. innerLoop se voit attribuer la valeur Name 3 de la liste des noms de famille.
    4. Si la condition vérifie la valeur n'est pas vraie
    5. innerLoop se voit maintenant attribuer la valeur Name 1 de la liste des noms de famille.
    6. Si la condition est vraie et qu'elle affiche "Nom 1 Nom 1" dans le log innerLoop se voit maintenant attribuer la valeur Nom 4 puis Nom 2.
    7. outerLoop se voit maintenant attribuer le nom 2 à partir de la liste des noms.
    8. innerLoop se voit attribuer le nom 3 de la liste des noms de famille

    La boucle extérieure s'exécutera une fois, puis laissera sa boucle intérieure se terminer complètement avant de passer à la valeur suivante. La variable itératrice de la boucle for doit correspondre au type de données de la liste ou à l'itération définie parce que chaque valeur de ladite liste ou de cet ensemble sera affectée à la variable itérateur une par une et lancera une erreur s'il y a une inadéquation des types de données.

    Listes vs cartes

    Maintenant que nous savons comment fonctionnent les boucles, comprenons la différence entre les listes et les cartes.

    Prenez cet exemple : j'ai une liste de 100 courriels avec des noms. J'ai une autre liste de 100 courriels avec des numéros de téléphone. Je dois faire une nouvelle liste avec des noms, des numéros de téléphone et des courriels.

    Le code suivant est à des fins de démonstration uniquement

    En utilisant une liste et imbriquée pour la boucle, le code ressemblerait à ceci :

    List<String>  emailList = //list with email and name
List<String> phoneList = //list with email and phone
List<String> finalList = new List<String> ();

for(String emailLooper: emailList) {
	for (String phoneLooper: phoneList) {
		if (emailLooper.email == phoneLooper.email){
			finalList.add(emailLooper.email + emailLooper.name + phoneLooper.phone);
		}

	}
}

    Cette boucle prend le premier e-mail de emailList et le compare à chaque e-mail de phoneList et lorsqu'elle trouve une correspondance, elle l'affecte à la liste finale et continue. Cela signifie 100 courriels * 100 numéros, ce qui peut atteindre 10 000 exécutions.

    L'utilisation d'une carte ressemblerait à ceci :

    Map<String, String> emailList = //list with email and name
Map<String, String> phoneList = //list with email and phone

List<String>  finalList = new List<String> ();

//Make empty variables
String emailFromEmailList;
String emailFromPhoneList;

for (String emailLooper: emailList.keyset()){ // .keyset() returns all the values in maps
	emailFromEmailList = emailList.get(emailLooper);
	emailFromPhoneList = phoneList.get(emailLooper);

	if (emailFromEmailList == emailFromPhoneList) {
		finalList.add(emailList.name, emailList.email, phoneList.phone); //This is not how use maps, for demonstartion purposes only
	}
}

    Cette boucle itère 100 fois sur la liste de diffusion et obtient la valeur correcte de la liste de téléphone en appelant sa paire clé-valeur. Les listes et les cartes ont leur propre heure et lieu à utiliser, mais dans ce scénario, l'utilisation de Maps a entraîné 100 fois moins d'itérations de la boucle, ce qui a entraîné une utilisation moindre de la puissance de calcul et des ressources.

    Résumé

    • Les listes, les ensembles et les cartes peuvent contenir plusieurs valeurs
    • La boucle for s'exécute jusqu'à ce que la condition de sortie soitfalse
