Lista, conjunto, mapas y bucles

Listas, conjuntos, mapas

List es un tipo de datos ordenado que contiene varios valores. Cada valor tiene su lugar en la lista donde el conteo comienza desde 0. Aquí está la sintaxis:

List<dataType> variableName = new List<dataType>();
  • List<dataType>define qué tipo de datos tendrá la lista
  • variableNamedefine el nombre utilizado para referirse a esta lista
  • newdefine que estamos haciendo una nueva lista
  • List<dataType>()define que estamos haciendo una lista en blanco.

Para agregar un valor en la Lista seguimos la sintaxis:

variableName.add(value);

Aquí está el código en acción:

List<String> names = new List<String>();

names.add('Harshdeep');
names.add('Cobey');
names.add('Zerab');

System.debug(names);

Si queremos acceder a un elemento específico de la Lista, usamos la siguiente sintaxis:

 variableName[n];
  • variableNamees el nombre de la lista
  • [n]es el elemento numérico al que estamos tratando de acceder, entre corchetes

Veamos esto en la práctica:

List<String> names = new List<String>();

names.add('Harshdeep');
names.add('Cobey');
names.add('Zerab');

System.debug(names[3]);

Se le presentará el siguiente error:

Line: 7, Column: 1
System.ListException: List index out of bounds: 3

Esto se debe a que todo el conteo comienza desde 0, por lo que para acceder al tercer elemento, necesitamos restar 1 de él porque internamente la lista se almacena de la siguiente manera:

Índice Nombre
0 Duro profundo
1 Cobey
2 Zerab

Ahora arreglemos el código eliminando la última línea con:

System.debug(names[0]);
System.debug(names[2]);

¡Ahora el registro muestra el nombre y el apellido! Esto es súper importante de recordar. Todo el conteo comienza desde 0.

Set es una lista desordenada que no tiene ningún duplicado. Así es como haces un Set:

Set<dataType> variableName = new Set<dataType>();
  • Set<dataType>es declarar el tipo Conjunto de tipo dataType.
  • variableNamees referirse a este conjunto.
  • newdefine que esta es una nueva variable.
  • Set<dataType>()es init el conjunto vacío con dicho tipo de datos.

Veamos Establecer en acción:

Set<String> names = new Set<String>();
names.add('Harshdeep');
names.add('Cobey');
names.add('Zerab');
names.add('Cobey');

System.debug(names);

Cuando mira los registros, el nombre duplicado se ha eliminado y el orden también se ha cambiado. Los conjuntos generalmente no se utilizan mucho en la programación práctica tanto como Listas o Mapas. Un uso práctico de los conjuntos sería en un escenario en el que se han introducido datos incorrectos, y en lugar de exportar todos los datos y ejecutarse a través de una hoja de cálculo, todos los datos se almacenan en un conjunto y los valores únicos se extraen inmediatamente para reestructurar los datos.

El mapa contiene pares de valores clave. Esto significa que cada elemento tendrá dos valores. La sintaxis para hacer conjuntos es:

Map<dataType1, dataType2> variableName = new Map<dataType1, dataType2>();
  • Map<dataType1, dataType2>es inicializar la variable de mapa. dataType 1 y 2 pueden ser los mismos tipos de datos o diferentes.
  • variableNamees el nombre de la variable a la que referirse new es declarar que es una variable nueva
  • Map<dataType1, dataType2>()es inicializar un mapa vacío del mismo tipo de datos.
    • Map<dataType1, dataType2> variableName = new Map<dataType1, dataType2>();
variableName.put(key, value);

    You insert values in your Map by using mapVariable.put(key, value);, where key and value are dataType variables as defined in their declaration. An easier way to understand is ordering food in a drive through. You ask for Number 1, 2 and 6 and you're given the food corresponding to those numbers.

    To access value from a Map, we use .get(key).
    Un ejemplo de mapas con el mismo tipo de datos:

    Map<String, String> placeCapital = new Map<String, String>();
placeCapital.put('New Delhi', 'India');
placeCapital.put('Captial', 'Country');

String ourCapital = placeCapital.get('New Delhi');
System.debug(ourCapital);

    Esto devolverá "India" en su registro.

    Comprender cuándo usar Conjuntos, Listas y Mapas

    Conjunto

    En mi experiencia, rara vez he usado Sets. Dos problemas muy distintos eran más fáciles de resolver con Set que en cualquier otro lugar:

    1. Una organización tenía el campo País de facturación establecido en Texto en lugar de la lista de selección, y para cada país, había múltiples valores, algunos recurrentes más que otros. En lugar de exportar datos a una hoja de cálculo, ejecutar fórmulas y cargarlas, era más eficiente escribir un conjunto que recopilara todos los valores únicos en una tabla y luego procesarlo aún más para solucionar el problema.
    2. Una organización tenía una lista de correo personalizada sin restricciones en el número de veces que podía inscribirse. Cuando ciertos gerentes se perdían los correos electrónicos, eligieron volver a suscribirse, lo que resultó en el envío de varios correos electrónicos al mismo usuario. Una forma rápida de resolver esto fue hacer un bucle y asignar todos los valores de la Lista al Conjunto y actualizar la Lista final.

    En ambos casos, fue el resultado de una mala configuración por parte de los administradores y desarrolladores que podría haberse evitado fácilmente.

    Para entender el uso óptimo de Lists v/s Maps, primero aprendamos sobre For Loops.

    Para bucles

    Para bucles es uno de los bloques de código más utilizados en la programación de Apex. Un bucle For se puede usar de 2 maneras diferentes:

    1. Para ejecutar un bloque de código n número de veces
    2. Para iterar sobre una lista

    Ejecutar un bloque de código n veces

    Para hacer este bucle, necesitamos tres cosas:

    1. Una sentencia de inicio
    2. Una condición de salida
    3. Una instrucción de incremento

    La sintaxis es:

    for(start statement; exit condition; increment statement) {
  //code block
}
    • fores la declaración del bucle
    • startes lo que se ejecutará al inicio del bucle
    • exit conditiones lo que detendrá el bucle si devuelve false
    • increment statementse ejecutará cada vez que el bloque de código termine de ejecutarse
    • ;es el separador entre cada condición / sentencia y es necesario

    Ejecute este código en la consola del desarrollador:

    for(Integer i = 0; i < 10; i++){
  System.debug('Hello '+ i);
}

    Vamos a desglosar este código. Esto es lo que pasó:

    1. Haga una i entera y asígnele el valor de 0
    2. ¿Es menor de 10? si es menor de 10, ejecute el bloque de código
    3. Después de ejecutar el bloque de código, incremente el valor de i en 1.
    4. Las sentencias 2, 3 y 4 se ejecutan hasta que la i es igual a 10.

    Así es como funciona for loop en términos de elaboración de café. Estás pesando tu café y la primera cucharada tenía solo 50 gramos de café de los 100 gramos. Un bucle para en términos de café sería:

    for (spoon = 0 grams; total_coffee < 100 grams; add another spoon){
  spoon.add(50 grams);
  weight.check();
}

    Comenzamos con 0 gramos en nuestra cuchara, y agregamos 50 gramos de café con cada cuchara. Nuestra condición de salida es asegurar que el café total en nuestra taza no exceda los 100 gramos, por lo que seguimos agregando café en cucharas de 50 gramos hasta que se cumpla nuestra condición de salida.

    Bucle sobre una lista o conjunto

    Para hacer este bucle, necesitamos dos cosas:

    • Una lista o conjunto para iterar sobre
    • Variable que mantendrá valores temporalmente

    La sintaxis es:

    for (dataType variableName: list_or_set){
    //code block
}
    • fores la declaración del bucle
    • dataTypees el tipo de datos de la variable
    • variableNamees la variable que mantendrá temporalmente el valor
    • list_or_setes una lista o conjunto existente

    Ejecute este código en la consola del desarrollador:

    List<String> names=new List<String>();

names.add('Name 1');
names.add('Name 2');
names.add('Name 3');
names.add('Name 4');

for(String iterator: names){
  System.debug(iterator);
}

    Vamos a desglosar este código. Esto es lo que pasó:

    1. Haga una lista y añádale valores.
    2. El bucle for se inicia
    3. Tenemos una variable llamada iterador que es de tipo string, que confirma el tipo de datos de los nombres de la lista, que es una lista de cadenas
    4. Asigne el primer valor de los nombres al iterador y ejecute el bloque de código
    5. Después de que el bloque de código se haya ejecutado, asigne el siguiente valor en nombres al iterador y ejecute el bloque de código
    6. Si los nombres de la lista se han quedado sin valores, salga del bucle.

    Un bucle For también se puede anidar dentro del bucle for.

    Aquí hay un ejemplo

    List<String> names=new List<String>();
names.add('Name 1');
names.add('Name 2');
names.add('Name 3');
names.add('Name 4');

List<String> surnames=new List<String>();
surnames.add('Name 3');
surnames.add('Name 1');
surnames.add('Name 4');
surnames.add('Name 2');

for (String outerLoop: names) {
  for (String innerLoop: surnames) {
    if (outerLoop==innerLoop) {
      System.debug(outerLoop + ' ' +innerLoop); //To add a space between both values
    }
  }
}

    Desglosemos este código:

    1. Haga 2 listas y asigne valores a ellas
    2. a outerLoop se le asigna el valor Nombre 1 de la lista de nombres.
    3. a innerLoop se le asigna el valor Name 3 de la lista de apellidos.
    4. Si la condición comprueba que el valor no es verdadero
    5. A innerLoop ahora se le asigna el valor Nombre 1 de la lista de apellidos.
    6. Si la condición es verdadera y muestra "Nombre 1 Nombre 1" en el bucle interno del registro ahora se le asigna el valor Nombre 4 y luego Nombre 2.
    7. outerLoop ahora recibe el nombre 2 de la lista de nombres.
    8. innerLoop recibe el nombre 3 de la lista de apellidos

    El bucle externo se ejecutará una vez, luego dejará que su bucle interior termine completamente antes de saltar al siguiente valor. La variable iteradora en el bucle for necesita coincidir con el tipo de datos de la lista o conjunto de iteración porque cada valor en dicha lista o conjunto se asignará a la variable iteradora uno por uno, y arrojará un error si hay una falta de coincidencia de tipos de datos.

    Listas vs Mapas

    Ahora que sabemos cómo funcionan los bucles, entendamos la diferencia entre listas y mapas.

    Tome este ejemplo: Tengo una lista de 100 correos electrónicos con nombres. Tengo otra lista de 100 correos electrónicos con números de teléfono. Necesito hacer una nueva lista con Nombres, Números de Teléfono y Correos Electrónicos.

    El siguiente código es solo para fines de demostración

    Usando una lista y anidado para el bucle, el código se vería algo así:

    List<String>  emailList = //list with email and name
List<String> phoneList = //list with email and phone
List<String> finalList = new List<String> ();

for(String emailLooper: emailList) {
	for (String phoneLooper: phoneList) {
		if (emailLooper.email == phoneLooper.email){
			finalList.add(emailLooper.email + emailLooper.name + phoneLooper.phone);
		}

	}
}

    Este bucle toma el primer correo electrónico de emailList y lo compara con cada correo electrónico de phoneList y cuando encuentra una coincidencia lo asigna a la lista final y continúa. Esto significa 100 correos electrónicos * 100 números, lo que llega a 10.000 ejecuciones.

    Usar un mapa se vería algo así:

    Map<String, String> emailList = //list with email and name
Map<String, String> phoneList = //list with email and phone

List<String>  finalList = new List<String> ();

//Make empty variables
String emailFromEmailList;
String emailFromPhoneList;

for (String emailLooper: emailList.keyset()){ // .keyset() returns all the values in maps
	emailFromEmailList = emailList.get(emailLooper);
	emailFromPhoneList = phoneList.get(emailLooper);

	if (emailFromEmailList == emailFromPhoneList) {
		finalList.add(emailList.name, emailList.email, phoneList.phone); //This is not how use maps, for demonstartion purposes only
	}
}

    Este bucle itera sobre la lista de correo electrónico 100 veces y obtiene el valor correcto de la lista telefónica llamando a su par clave-valor. Las listas y los mapas tienen su propio tiempo y lugar para usar, pero en este escenario el uso de mapas resultó en 100 veces menos iteraciones del bucle, lo que resulta en un menor uso de la potencia informática y los recursos.

    Resumen

    • Listas, conjuntos y mapas pueden contener varios valores
    • A for loop se ejecuta hasta que la condición de salida esfalse
