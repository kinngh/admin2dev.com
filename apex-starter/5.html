Liste, Set, Karten und für Schleifen

Listen, Sets, Karten

Liste ist ein geordneter Datentyp, der mehrere Werte enthält. Jeder Wert hat seinen Platz in der Liste, in der die Zählung bei 0 beginnt. Hier ist die Syntax:

List<dataType> variableName = new List<dataType>();
  • List<dataType>Definiert, welche Art von Daten die Liste enthalten wird
  • variableNameDefiniert den Namen, der für den Verweis auf diese Liste verwendet wird
  • newDefiniert, dass wir eine neue Liste erstellen
  • List<dataType>()Definiert, dass wir eine leere Liste erstellen.

Um einen Wert in der Liste hinzuzufügen, folgen wir der Syntax:

variableName.add(value);

Hier ist der Code in Aktion:

List<String> names = new List<String>();

names.add('Harshdeep');
names.add('Cobey');
names.add('Zerab');

System.debug(names);

Wenn wir auf ein bestimmtes Element in der Liste zugreifen möchten, verwenden wir die folgende Syntax:

 variableName[n];
  • variableNameIst der Name der Liste
  • [n]Ist das Zahlenelement, auf das wir zugreifen möchten, eingeschlossen in eckige Klammern

Lassen Sie uns dies in der Praxis sehen:

List<String> names = new List<String>();

names.add('Harshdeep');
names.add('Cobey');
names.add('Zerab');

System.debug(names[3]);

Es wird der folgende Fehler angezeigt:

Line: 7, Column: 1
System.ListException: List index out of bounds: 3

Dies liegt daran, dass alle Zählungen bei 0 beginnen. Um also auf das dritte Element zuzugreifen, müssen wir 1 davon subtrahieren, da die Liste intern wie folgt gespeichert wird:

Index Name
0 Harshdeep
1 Cobey
2 Zerab

Lassen Sie uns nun den Code korrigieren, indem wir die letzte Zeile entfernen mit:

System.debug(names[0]);
System.debug(names[2]);

Jetzt gibt das Protokoll den Vor- und Nachnamen aus! Das ist super wichtig, sich daran zu erinnern. Alle Zählungen beginnen bei 0.

Set ist eine ungeordnete Liste, die keine Duplikate enthält. So erstellen Sie ein Set:

Set<dataType> variableName = new Set<dataType>();
  • Set<dataType>Ist es, den Satz des Datentyps zu deklarieren.
  • variableNameIst es, sich auf dieses Set zu beziehen.
  • newDefiniert, dass dies eine neue Variable ist.
  • Set<dataType>()Ist es, die leere Menge mit dem Datentyp zu initieren.

Sehen wir uns Set in Aktion an:

Set<String> names = new Set<String>();
names.add('Harshdeep');
names.add('Cobey');
names.add('Zerab');
names.add('Cobey');

System.debug(names);

Wenn Sie sich die Protokolle ansehen, wurde der doppelte Name entfernt, und auch die Reihenfolge wurde geändert. Sets werden in der Regel nicht so viel in der praktischen Programmierung verwendet wie Listen oder Karten. Eine praktische Verwendung für Sets wäre in einem Szenario, in dem fehlerhafte Daten eingegeben wurden, und anstatt alle Daten zu exportieren und eine Tabelle zu durchlaufen, werden alle Daten in einer Gruppe gespeichert und eindeutige Werte sofort herausgezogen, um die Daten umzustrukturieren.

Karte enthält Schlüsselwertpaare. Das bedeutet, dass jedes Element zwei Werte hat. Die Syntax für die Erstellung von Mengen lautet:

Map<dataType1, dataType2> variableName = new Map<dataType1, dataType2>();
  • Map<dataType1, dataType2>Ist es, die Kartenvariable zu initialisieren. dataType 1 und 2 können dieselben oder unterschiedliche Datentypen sein.
  • variableNameIst der Name der Variablen, auf die verwiesen werden soll, um neu zu deklarieren, dass es sich um eine neue Variable handelt
  • Map<dataType1, dataType2>()Ist es, eine leere Karte desselben Datentyps zu initialisieren.
    • Map<dataType1, dataType2> variableName = new Map<dataType1, dataType2>();
variableName.put(key, value);

    You insert values in your Map by using mapVariable.put(key, value);, where key and value are dataType variables as defined in their declaration. An easier way to understand is ordering food in a drive through. You ask for Number 1, 2 and 6 and you're given the food corresponding to those numbers.

    To access value from a Map, we use .get(key).
    Ein Beispiel für Karten mit demselben Datentyp:

    Map<String, String> placeCapital = new Map<String, String>();
placeCapital.put('New Delhi', 'India');
placeCapital.put('Captial', 'Country');

String ourCapital = placeCapital.get('New Delhi');
System.debug(ourCapital);

    Dies gibt "Indien" in Ihrem Protokoll zurück.

    Verstehen, wann Sets, Listen und Karten zu verwenden sind

    Set

    Meiner Erfahrung nach habe ich Sets selten verwendet. Zwei sehr unterschiedliche Probleme waren mit Set leichter zu lösen als anderswo:

    1. In einer Organisation war das Feld Abrechnungsland auf Text anstelle der Auswahlliste eingestellt, und für jedes Land gab es mehrere Werte, einige mehr als andere. Anstatt Daten in eine Tabelle zu exportieren, Formeln auszuführen und hochzuladen, war es effizienter, einen Satz zu schreiben, der alle eindeutigen Werte in einer Tabelle sammelte und dann weiter darauf zu verarbeiten, um das Problem zu beheben.
    2. Eine Organisation hatte eine maßgeschneiderte Mailingliste ohne Einschränkungen, wie oft Sie sich anmelden konnten. Wenn bestimmte Manager E-Mails verpassten, entschieden sie sich, sich erneut anzumelden, was dazu führte, dass mehrere E-Mails an denselben Benutzer gesendet wurden. Eine schnelle Möglichkeit, dies zu lösen, bestand darin, eine Schleife zu erstellen und alle Werte aus der Liste dem Set zuzuweisen und die endgültige Liste zu aktualisieren.

    In beiden Fällen war es das Ergebnis einer schlechten Konfiguration durch Administratoren und Entwickler, die leicht hätte vermieden werden können.

    Um die optimale Nutzung für Listen v/s Maps zu verstehen, lassen Sie uns zuerst mehr über For Loops erfahren.

    Für Schleifen

    Für Schleifen ist einer der am häufigsten verwendeten Codeblöcke in der Apex-Programmierung. A For-Schleife kann auf 2 verschiedene Arten verwendet werden:

    1. So führen Sie einen Codeblock n mehrmals aus
    2. So iterieren Sie über eine Liste

    Einen Codeblock n mal ausführen

    Um diese Schleife zu machen, brauchen wir drei Dinge:

    1. Eine start-Anweisung
    2. Eine Ausstiegsbedingung
    3. Eine Inkrement-Anweisung

    Die Syntax lautet:

    for(start statement; exit condition; increment statement) {
  //code block
}
    • forIst die Deklaration der Schleife
    • startAnweisung ist das, was zu Beginn der Schleife ausgeführt wird
    • exit conditionIst, was die Schleife stoppt, wenn sie false zurückgibt
    • increment statementWird jedes Mal ausgeführt, wenn der Codeblock ausgeführt wird
    • ;Ist das Trennzeichen zwischen jeder Bedingung / Anweisung und ist notwendig

    Führen Sie diesen Code in der Entwicklerkonsole aus:

    for(Integer i = 0; i < 10; i++){
  System.debug('Hello '+ i);
}

    Lassen Sie uns diesen Code aufschlüsseln. Hier ist, was passiert ist:

    1. Machen Sie eine Ganzzahl i und weisen Sie ihr den Wert 0 zu
    2. Ist ich kleiner als 10? Wenn es kleiner als 10 ist, führen Sie den Codeblock aus
    3. Nachdem der Codeblock ausgeführt wurde, erhöhen Sie den Wert von i um 1.
    4. Die Anweisungen 2, 3 und 4 werden ausgeführt, bis das i gleich 10 ist.

    So funktioniert für Loop in Bezug auf die Kaffeezubereitung. Sie wiegen Ihren Kaffee und die erste Kugel hatte nur 50 Gramm Kaffee von den 100 Gramm. A for loop in Kaffeebegriffen wäre:

    for (spoon = 0 grams; total_coffee < 100 grams; add another spoon){
  spoon.add(50 grams);
  weight.check();
}

    Wir beginnen mit 0 Gramm in unserem Löffel und fügen mit jedem Löffel 50 Gramm Kaffee hinzu. Unsere Ausgangsbedingung besteht darin, sicherzustellen, dass der Gesamtkaffee in unserer Tasse 100 Gramm nicht überschreitet, daher fügen wir weiterhin Kaffee in 50 Gramm Löffeln hinzu, bis unsere Ausstiegsbedingung erfüllt ist.

    Schleife über eine Liste oder einen Satz

    Um diese Schleife zu machen, brauchen wir zwei Dinge:

    • Eine Liste oder ein Set, um über zu iterieren
    • Eine Variable, die Werte vorübergehend enthält

    Die Syntax lautet:

    for (dataType variableName: list_or_set){
    //code block
}
    • forIst die Deklaration der Schleife
    • dataTypeIst der Datentyp der Variablen
    • variableNameIst die Variable, die vorübergehend den Wert enthält
    • list_or_setIst eine vorhandene Liste oder ein Satz

    Führen Sie diesen Code in der Entwicklerkonsole aus:

    List<String> names=new List<String>();

names.add('Name 1');
names.add('Name 2');
names.add('Name 3');
names.add('Name 4');

for(String iterator: names){
  System.debug(iterator);
}

    Lassen Sie uns diesen Code aufschlüsseln. Hier ist, was passiert ist:

    1. Erstellen Sie eine Liste und fügen Sie ihr Werte hinzu.
    2. Die for-Schleife beginnt
    3. Wir haben eine Variable namens Iterator, die vom Typ string ist, die den Datentyp der Listennamen bestätigt, die eine Liste von Strings sind.
    4. Weisen Sie dem Iterator den ersten Namenswert zu und führen Sie den Codeblock aus
    5. Nachdem der Codeblock ausgeführt wurde, weisen Sie dem Iterator den nächsten Wert in Namen zu und führen Sie den Codeblock aus
    6. Wenn den Listennamen die Werte ausgegangen sind, beenden Sie die Schleife.

    A For-Schleife kann auch innerhalb der for-Schleife verschachtelt werden.

    Hier ist ein Beispiel

    List<String> names=new List<String>();
names.add('Name 1');
names.add('Name 2');
names.add('Name 3');
names.add('Name 4');

List<String> surnames=new List<String>();
surnames.add('Name 3');
surnames.add('Name 1');
surnames.add('Name 4');
surnames.add('Name 2');

for (String outerLoop: names) {
  for (String innerLoop: surnames) {
    if (outerLoop==innerLoop) {
      System.debug(outerLoop + ' ' +innerLoop); //To add a space between both values
    }
  }
}

    Lassen Sie uns diesen Code aufschlüsseln:

    1. Erstellen Sie 2 Listen und weisen Sie ihnen Werte zu
    2. outerLoop erhält den Wert Name 1 aus der Namensliste.
    3. innerLoop erhält den Wert Name 3 aus der Nachnamenliste.
    4. Wenn Bedingungsprüfungen überprüft werden, ist der Wert nicht wahr
    5. innerLoop erhält jetzt den Wert Name 1 aus der Nachnamenliste.
    6. Wenn die Bedingung wahr ist und sie "Name 1 Name 1" im Protokoll ausgibt, wird innerLoop jetzt der Wert Name 4 und dann Name 2 zugewiesen.
    7. outerLoop erhält jetzt den Namen 2 aus der Namensliste.
    8. innerLoop erhält den zugewiesenen Namen 3 aus der Nachnamenliste

    Die äußere Schleife wird einmal ausgeführt, dann lässt sie die innere Schleife vollständig beenden, bevor sie zum nächsten Wert springt. Die Iteratorvariable in der for-Schleife muss mit dem Datentyp der Liste übereinstimmen oder die Iteration übersetzen, da jeder Wert in der Liste oder Menge der Iteratorvariablen einzeln zugewiesen wird und einen Fehler auslöst, wenn die Datentypen nicht übereinstimmen.

    Listen im Vergleich zu Karten

    Jetzt, da wir wissen, wie Schleifen funktionieren, wollen wir den Unterschied zwischen Listen und Karten verstehen.

    Nehmen Sie dieses Beispiel: Ich habe eine Liste von 100 E-Mails mit Namen. Ich habe eine weitere Liste von 100 E-Mails mit Telefonnummern. Ich muss eine neue Liste mit Namen, Telefonnummern und E-Mails erstellen.

    Der folgende Code dient nur zu Demonstrationszwecken

    Wenn Sie eine Liste verwenden und für eine Schleife verschachtelt sind, würde der Code etwa so aussehen:

    List<String>  emailList = //list with email and name
List<String> phoneList = //list with email and phone
List<String> finalList = new List<String> ();

for(String emailLooper: emailList) {
	for (String phoneLooper: phoneList) {
		if (emailLooper.email == phoneLooper.email){
			finalList.add(emailLooper.email + emailLooper.name + phoneLooper.phone);
		}

	}
}

    Diese Schleife nimmt die erste E-Mail von emailList und gleicht sie mit jeder E-Mail in phoneList ab, und wenn sie eine Übereinstimmung findet, weist sie sie der endgültigen Liste zu und macht weiter. Das bedeutet, 100 E-Mails * 100 Nummern, was bis zu 10.000 Durchläufen ergibt.

    Die Verwendung einer Karte würde etwa so aussehen:

    Map<String, String> emailList = //list with email and name
Map<String, String> phoneList = //list with email and phone

List<String>  finalList = new List<String> ();

//Make empty variables
String emailFromEmailList;
String emailFromPhoneList;

for (String emailLooper: emailList.keyset()){ // .keyset() returns all the values in maps
	emailFromEmailList = emailList.get(emailLooper);
	emailFromPhoneList = phoneList.get(emailLooper);

	if (emailFromEmailList == emailFromPhoneList) {
		finalList.add(emailList.name, emailList.email, phoneList.phone); //This is not how use maps, for demonstartion purposes only
	}
}

    Diese Schleife iteriert 100 Mal über die E-Mail-Liste und erhält den richtigen Wert aus der Telefonliste, indem sie ihr Schlüssel-Wert-Paar aufruft. Listen und Karten haben ihre eigene Zeit und ihren eigenen Ort, aber in diesem Szenario führte die Verwendung von Maps zu 100x weniger Iterationen der Schleife, was zu weniger Nutzung von Rechenleistung und Ressourcen führt.

    Zusammenfassung

    • Listen, Sätze und Karten können mehrere Werte enthalten
    • A for-Schleife läuft, bis die Exit-Bedingungfalse
