Fortgeschrittener SOQL-Test

Dieser Schnelltest wurde durchgeführt, um Ihr grundlegendes Verständnis von SOQL zu testen.

  • Sie können sich die Syntax ansehen, Bonuspunkte, wenn Sie es nicht tun.
  • Die Lösung des Tests sollte idealerweise nicht länger als 10 Minuten dauern.
  • Alle Kriterien müssen erfüllt sein.
  • Die Antwort ist unter Admin2Dev GitHub Repo verfügbar.
  • Die Antwort und die vollständige Aufschlüsselung des Programms finden Sie am Ende dieser Seite.

Der Test

Frage

Write a class conUpdater that has a method updateAccount() that takes 2 String as parameter. The first parameter is Account name and the second is Description String. The method should update all the associated contact's description fields of the Account to the description string that was taken as the parameter only if the account exists.

Hinweis

  • Überprüfen Sie, ob das Konto existiert.
  • Update the contact.description field to the String
  • Überprüfen Sie die Größe der Liste mit List.size(), die einen Ganzzahlwert zurückgibt.
Starten Sie einen Timer und notieren Sie sich, wie lange Sie brauchen, um ihn zu lösen. Idealerweise sollte es ~10 Minuten dauern, aber keine Sorge, wenn Sie über Bord gehen.

Antwort

conUpdater-Klasse

public class conUpdater {
    public static void updateAccount(String accString, String descString){

      List<Account> accList = [SELECT ID, Name, (SELECT ID, Description FROM Contacts) FROM Account WHERE Name = :accString];

      List<Contact> finalList = new List<Contact>();

      if (accList.size() > 0){
        for(Contact iterator: accList[0].contacts){
          iterator.description = descString;
          finalList.add(iterator);
        }

      update finalList;
      }
      else{
        System.debug('Account does not exist');
      }

    }

}
  • We first declare our two strings that are inputs, accString and descString.
  • The accList stores the return value, the finalList is used to update our final values.
  • We check if the size of accList is more than 0, which assures that we did have some values returned.
  • Wir führen eine for-Schleife für den Kontaktteil der Liste durch (stellen sicher, dass wir Plural verwenden) und aktualisieren die Werte.
  • Wenn das Konto nicht existiert, geben wir einfach heraus, dass das Konto nicht existiert, und tun nichts.
