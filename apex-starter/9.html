Datenbankklassenmethoden

Datenbankklassenmethoden

Datenbankklassenmethoden sind alternative Möglichkeiten, DML-Anweisungen zu verwenden, außer Datenbankklassenmethoden bieten mehr Funktionalität und Flexibilität.

DML-Anweisungen Datenbankklassenmethoden
Teilaktualisierungen sind nicht zulässig Ermöglicht teilweise Updates
Keine Möglichkeit, eine Liste der erfolgreichen / fehlgeschlagenen Datensätze zu erhalten Methoden stehen zur Verfügung, um eine Liste der eingefügten oder fehlgeschlagenen Datensätze abzurufen.

Die Verwendung von Datenbankklassenmethoden erfordert Kenntnisse über integrierte Klassen.

  • Datenbankklasse
  • SaveResult-Klasse
  • Fehlerklasse

Datenbankklasse

Die Methoden der Datenbankklasse ermöglichen es Ihnen, Daten in die Organisationsdatenbank zu manipulieren und teilweise Aktualisierungen zu ermöglichen, was bedeutet, dass Sie mehrere Datensätze eines Objekts bearbeiten können und der Vorgang auch dann durchgeführt würde, wenn einige fehlschlagen. Die Syntax lautet"

Database.statement(Record_Or_List, allOrNone);
  • DatabaseIst das Schlüsselwort
  • statement is the operation we are doing on a single record or list. This can be insert, update, upsert, delete orundelete
  • Single_Or_ListKann ein einzelner Datensatz, der eingefügt wird, oder eine Liste von Objekten sein.
  • allOrNone can be either true or false to indicate if the records should be operated on despite some of them failing.

Fügen wir nun 2 Datensätze in das Kontaktobjekt ein, einer, der erfolgreich ist, und der andere schlägt fehl.

List<Contact> finalList = new List<Contact>();

Contact con = new Contact();
con.lastName = 'Smith';
finalList.add(con);

Contact failCon = new Contact();
failCon.firstName = 'John';
finalList.add(failCon);

Database.insert(finalList, true);

Wenn dieser Code ausgeführt wird, wird ein Fehler ausgegeben, dass Nachname ein Pflichtfeld ist, aber wenn die letzte Anweisung auf false gesetzt wird, werden die Werte eingefügt, die eingefügt werden können, und der Rest wird weggelassen.

Database.insert(finalList, false);

Beim Öffnen unseres Konsolenprotokolls und beim Betreten des Abschnitts "Nur Fehler" gibt es kein Protokoll. Das liegt daran, dass wir die SaveResult-Klasse benötigen, um die Ergebnisse zu speichern.

SaveResult

SaveResult class is the result of an insert or update operation returned by the Database Class. Every Database operation returns an array (arrays are like List, but use [ ] instead of <>) of SaveResult. To use it, we just save the result of a Database operation into a Database.SaveResult[] variableName. Let’s add this to our code and loop over it:

List<Contact> finalList = new List<Contact>();

Contact con = new Contact();
con.lastName = 'Smith';
finalList.add(con);

Contact failCon = new Contact();
failCon.firstName = 'John';
finalList.add(failCon);

Database.SaveResult[] resultList = Database.insert(finalList, false); //Allow partial updates

for (Database.SaveResult iterator: resultList){
    System.debug(iterator);
}

Sie erhalten Folgendes in Ihren Nur Debug-Protokollen:

01:53:17:158 USER_DEBUG [14]|DEBUG|Database.SaveResult[getErrors=();getId=0036F00003XBWVJQA5;isSuccess=true;]

01:53:17:167 USER_DEBUG [14]|DEBUG|Database.SaveResult[getErrors=(Database.Error[getFields=(LastName);getMessage=Required fields are missing:[LastName];getStatusCode=REQUIRED_FIELD_MISSING;]);getId=null;isSuccess=false;]

You will notice the successful field has getErrors, getId and isSuccess fields, which can actually be used in our code to filter out successful results and get their IDs, and only output the failed fields. Let’s update our for loop to reflect this change and pretty up our logs:

for (Database.SaveResult iterator: resultList){
    if (iterator.isSuccess()){
        System.debug('Successfuly inserted: ' + iterator.getID());
    }
    else {
        System.debug(iterator);
    }
}

Great! now to deal with the errors, we make use of the methods available in Error class.

Fehler

Die Error-Klasse wird für die Fehlerberichterstattung verwendet und verfügt über 3 Methoden:

  • getFields()Um ein Array von Feldern zu erhalten, die fehlgeschlagen sind
  • getMessage()Um herauszufinden, warum der Datensatz fehlgeschlagen ist
  • getStatusCode()Um den Statuscode des Fehlers zu erhalten

Dies wird in Kombination mit der Datenbankklasse und seiner .getErrors()-Methode verwendet.

Fügen wir dies in unserer For-Schleife hinzu

for (Database.SaveResult iterator: resultList){
    if (iterator.isSuccess()){
        System.debug('Successfuly inserted: ' + iterator.getID());
    }
    else {
        for (Database.error errorIterator: iterator.getErrors()){
            System.debug(errorIterator.getStatusCode() + ' ' + errorIterator.getMessage() );
            System.debug('The following fields were affected: ' + errorIterator.getFields());
        }
    }
}

Das Protokoll lautet jetzt:

02:10:12:126 USER_DEBUG [15]|DEBUG|Successfuly inserted: 0036F00003XBWWvQAP

02:10:12:143 USER_DEBUG [19]|DEBUG|REQUIRED_FIELD_MISSING Required fields are missing: [LastName]

02:10:12:143 USER_DEBUG [20]|DEBUG|The following fields were affected: (LastName)

Jetzt sind die Protokolle viel sauberer und geben unsere Fehler sehr deutlich aus! Die Datenbankklasse hat mehr Methoden, die wir im Laufe der nächsten Tage lernen werden.

Zusammenfassung

  • Datenbankklassenmethoden haben erweiterte Funktionen und ermöglichen teilweise Aktualisierungen
  • SaveResult-Klasse ermöglicht das Speichern der Ergebnisse der Datenbankvorgänge
  • Error-Klasse ermöglicht die Navigation durch Fehler
  • An Array is similar to a list, with the only exception of [ ] instead of < >
