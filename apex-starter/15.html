Chargen-Apex-Test

Dieser Schnelltest wurde durchgeführt, um Ihr grundlegendes Verständnis von Batch Apex zu testen.

  • Sie können sich die Syntax ansehen, Bonuspunkte, wenn Sie es nicht tun.
  • Die Lösung des Tests sollte idealerweise nicht länger als 15 Minuten dauern.
  • Alle Kriterien müssen erfüllt sein.
  • Die Antwort ist unter Admin2Dev GitHub Repo verfügbar.
  • Die Antwort und die vollständige Aufschlüsselung des Programms finden Sie am Ende dieser Seite.

Der Test

Frage

  • Insert 250 contacts with the last name "Test n" where n is a number starting from 1. For reference, the names should look like Test 1, Test 2 .. .Test 250.
  • If a Contact with the last name DeleteBatch is created, it should run a Batch Apex Class that deletes all contacts with the last name "Test n" where n is a number.
  • Wenn die Batch Apex ausgeführt wird, sollte sie die Beschreibung des "Löschen"-Kontakts mit "Gelöschte Datensätze mit IDJobID" aktualisieren, wobei JobID die ID der Batch Apex-Klasse sein sollte.
Hinweis
  • Use getJobID() method on Database.batchableContext variable to get the Job ID.
  • Use the escape character \ to add ' to your Strings. So to write 'Test', you have to write \'Test\' instead.
  • Die Änderungen sind möglicherweise nicht sofort sichtbar. Geben Sie ihm etwa 5 Minuten, bevor Sie sehen, wie Kontakte gelöscht werden, und Beschreibungsaktualisierungen.

Starten Sie einen Timer und notieren Sie sich, wie lange Sie brauchen, um ihn zu lösen. Idealerweise sollte es ~15 Minuten dauern, aber keine Sorge, wenn Sie über Bord gehen.

Antwort

Einfügen von 250 Datensätzen

  • Sie können entweder eine Klasse erstellen und sie aus dem Anon Exec Window aufrufen oder direkt 250 Datensätze ohne Klassen einfügen.
List<Contact> finalList = new List<Contact>();

for(Integer i = 1; i<=250; i++){
    Contact con = new Contact();
    con.lastName = 'Test ' + i;
    finalList.add(con);
}
insert finalList;

System.debug('Inserted ' + finalList.size() + ' records.');

Chargenspitze

  • Meine Batch-Klasse heißt batchDeleter.
global class batchDeleter implements Database.Batchable<sObject> {

    global Database.QueryLocator start(Database.BatchableContext bc) {
        return Database.getQueryLocator('SELECT Id, LastName FROM Contact WHERE LastName LIKE \'Test%\'');
    }

    global void execute(Database.BatchableContext bc, List<sObject> deleteList){
        delete deleteList;
    }
    global void finish(Database.BatchableContext bc){
        List<Contact> con = [SELECT Id, LastName FROM Contact WHERE LastName = 'DeleteBatch' LIMIT 1];
        List<Contact> finalList = new List<Contact>();
        for (Contact iterator: con){
            iterator.Description = 'Deleted records with ID ' + bc.getJobId();
            finalList.add(iterator);
        }
        update finalList;

    }
}
  • First we return all values that have Last Name Test% where % is a wildcard.
  • In der execute-Methode löschen wir einfach die gesamte Liste.
  • In der Finish-Methode wählen wir den Kontakt 'DeleteBatch' aus und um wirklich sicher zu sein, dass wir nur 1 Wert erhalten, verwenden wir das Limit-Schlüsselwort.
  • Jetzt iterieren wir über unsere Liste und aktualisieren ihre Beschreibung, damit sie mit der Batch-Job-ID übereinstimmt.
    • bc comes from the method parameter Database.BatchableContext bc and running .getJobId() returns the ID.
  • Then the values are commited to the database using the update statement.

Auslöser

  • Wir möchten, dass die Batch-Klasse ausgeführt wird, wenn der Datensatz eingefügt ist, daher ist das Schreiben eines Triggers mit einer Hilfsklasse die einzige Möglichkeit, die Ausführung unserer Batch-Klasse zu steuern.

Trigger-Datei

trigger LastName on Contact(after insert){
    if (Trigger.isAfter){
        if (Trigger.isInsert){
            LastNameHelper.deleteContacts(trigger.new);
        }
    }
}

Tigger Helper-Datei

public class LastNameHelper{

    public static void deleteContacts(List<Contact> newList){

        for(Contact iterator: newList){
            if (iterator.LastName == 'DeleteBatch'){
                batchDeleter runBatch = new batchDeleter();
                Database.executeBatch(runBatch);

            }
        }

    }
}
  • In unserer Helferklasse verwenden wir jedes Mal, wenn ein neuer Kontakt eingefügt wird, Trigger.new, um alle Datensätze zu übergeben, und in der Helferklasse überprüfen wir auf Nachnamen. Wenn der Nachname mit unserer Bedingung übereinstimmt, führen wir die Batch-Klasse aus.
