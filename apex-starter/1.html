Einführung in die Programmierung und Problemlösung

Was ist Programmierung?

Die Programmierung ist das Schreiben einer Reihe von Anweisungen, um einzelne oder mehrere Aufgaben auszuführen. Nicht technisch gesehen schreibt es die Lösung für ein Problem.

Probleme lösen

Nehmen wir ein Szenario. Es ist 2 Uhr morgens, während ich diesen Kurs schreibe, und ich bin müde (das Problem) und ich werde dies durch die Zubereitung einer Tasse Kaffee (Lösung) beheben. So mache ich meinen Kaffee.

  • In die Küche gehen
  • Holen Sie meine Tasse raus
  • Holen Sie sich etwas Kaffee
  • Messen Sie den Kaffee
  • Gießen Sie es in meine französische Presse
  • Etwas Wasser kochen
  • Fügen Sie das Wasser hinzu
  • Warten Sie 4 Minuten
  • Gießen Sie Kaffee in die Tasse

1 Problem. Unendliche Lösungen.

Sie könnten Ihren Kaffee in einer Espressomaschine zubereiten, oder Sie konnten überhaupt keinen Kaffee machen! Das gleiche Problem, sich müde zu fühlen, hat mehrere Lösungen. Kaffee, Tee, Energieaufnahmen, Power Naps. Die Möglichkeiten sind nahezu endlos.

Als Programmierer lösen Sie Probleme, und das tun Sie auf Ihre Weise. Es geht darum, wie Sie denken, dass das Problem gelöst werden sollte, und wenn Sie einen logischen Fehler in Ihrem Code sehen, ist das nicht nur ein Fehler, sondern ein Fehler in der Art und Weise, wie Sie denken.

Programmierung der Kaffeezuherstellung

Entgegen der gängigen Meinung ist das Schreiben von Code der letzte Teil der Programmierung. Eine gute Möglichkeit, Probleme zu lösen, ist der dreistufige Ansatz:

  • Identifizieren Sie das Problem
  • Die Lösung als Flussdiagramm
  • Schreiben Sie Ihren Code

Identifizieren Sie das Problem

Problem: Der Mensch ist müde und braucht die Energie, um wach zu bleiben. Piepton
Lösung: Holen Sie dem Menschen Kaffee.

Lösung als Flussdiagramm

Der nächste Schritt umfasst die Anleitung für die Lösung. Welche Schritte sind zu befolgen? Was benötigen Sie? Wie lange und oft sollte es laufen?

Von oben nach unten ist das Flussdiagramm äußerst detailliert über die Schritte, die zur Herstellung einer Tasse Kaffee zu befolgen sind. Wenn Sie Flow Builder schon einmal verwendet haben, mag dies vertraut erscheinen.

Schreiben des Codes

Jetzt haben wir unsere detaillierte Lösung für das Problem. Schreiben wir Code in einfachem Englisch.

Genau wie Flussdiagramme ist die Programmierung ein Schritt-für-Schritt-Prozess.

Zuerst sammeln wir unsere Materialien.

get cup = 1 unit;
get coffee = 100 grams;
get water = 500 ml;
get pot;
get timer;
get frenchPress;

Legen wir sie jetzt auf einen Tisch und kochen wir das Wasser:

put cup, frenchPress, pot on table;
pour water(500 ml) in pot;
start pot;
set timer = 60 seconds;

Nachdem der Timer fertig ist, brauen wir den Kaffee:

put coffee(100 grams) in frenchPress;
pour 1/5 water from pot in frenchPress;
set timer = 30 seconds;

pour water from pot in frenchPress;
set timer = 5 minutes;

Jetzt, da unser Kaffee fertig ist, lassen Sie uns ihn in unsere Tasse gießen und trinken!

pour everything from frenchPress to cup;
drink;

Jetzt ist unser Kaffee fertig! Der endgültige Code sieht etwa so aus:

get cup = 1 unit;
get coffee = 100 grams;
get water = 500 ml;
get pot;
get timer;
get frenchPress;

put cup, frenchPress, pot on table;
pour water(500 ml) in pot;
start pot;
set timer = 60 seconds;

put coffee(100 grams) in frenchPress;
pour 1/5 water from pot in frenchPress;
set timer = 30 seconds;

pour water from pot in frenchPress;
set timer = 5 minutes;

pour everything from frenchPress to cup;

drink;

Herzlichen Glückwunsch! Sie sind jetzt Programmierer!

Zusammenfassung


  • Programmierung ist das Schreiben Ihrer Lösungen für ein Problem.
  • Das Befolgen des dreistufigen Prozesses vereinfacht die Programmierung und Problemlösung.
  • Programmieren lernen ist ein Marathon und kein Sprint. Es ist in Ordnung, sich Zeit zu nehmen.
