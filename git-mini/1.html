Einführung in Git und CLI

Was ist Git?

Git ist ein Versionskontrollsystem zur Verfolgung von Codeänderungen. Obwohl es sich an Programmierer richtet, kann es verwendet werden, um Änderungen in fast jeder Art von Datei 1 zu verfolgen.

Um Git zu verwenden, brauchen wir Git-Anbieter. Die drei beliebtesten Git-Anbieter sind

  • GitHub
  • GitLab
  • BitBucket

Was ist also der Unterschied zwischen einem Git-Anbieter und Git? Stellen Sie sich das als "git" vor, Ihre Fahrkünste sind. Sie können jedes Auto fahren, das Sie wollen. Ein Git-Anbieter ist ein Auto. Sie nutzen also Ihre Fahrkünste (git), um ein Auto (git provider) zu verwenden.

Warum Git notwendig ist

Versionierung

Git ist für die Verfolgung von Änderungen Ihres Codes gedacht. Mit den Anforderungen entwickeln sich auch der Code und manchmal möchten Sie in die Vergangenheit zurückgehen und ein Codestück wiederverwenden, das entfernt wurde, oder Änderungen verfolgen, die von anderen Teammitgliedern vorgenommen wurden.

Zusammenarbeit

Mit mehreren Zweigen und Gabelungen des Codes ist es möglich, eine persönliche Version des Codes zu erstellen und ihn dann mit dem Hauptzweig des Masters zusammenzuführen.

CLI

Wir werden Git ausschließlich von der CLI oder der Befehlszeilenschnittstelle aus ausführen. Es ist das schwarze Fenster mit weißem / grünem Text, das wir immer in Hackerfilmen sehen. Es mag auf den ersten Blick einschüchternd werden, aber sobald Sie den Dreh raus haben, wird CLI nur ein weiteres Werkzeug in Ihrem Arsenal als Entwickler sein.

Hinweis:

Diese Seite, Admin2Dev, wird vollständig auf GitHub mit Git und CLI verwaltet! Git ist also nicht nur ein weiteres Tool zur Verfolgung von Änderungen, es kann auch verwendet werden, um Websites und andere Inhalte in der Produktion bereitzustellen!
