Introducción a Git y CLI

¿Qué es Git?

Git es un sistema de control de versiones para rastrear los cambios en el código. Aunque está dirigido a programadores, se puede utilizar para rastrear cambios en casi cualquier tipo de archivo 1.

Para usar git, necesitamos proveedores de git. Los tres proveedores de git más populares son

  • GitHub
  • GitLab
  • BitBucket

Entonces, ¿cuál es la diferencia entre un proveedor de git y git? Piensa en esto como si fuera tu habilidad de conducción. Puedes conducir cualquier coche que quieras. Un proveedor de git es un coche. Así que usas tus habilidades de conducción (git) para usar un coche (git provider).

Por qué es necesario Git

Versionado

Git está diseñado para rastrear los cambios de su código. A medida que las necesidades evolucionan, también lo hace el código y, a veces, desea retroceder en el tiempo y reutilizar un fragmento de código que se eliminó o para realizar un seguimiento de los cambios realizados por otros miembros del equipo.

Colaboración

Con múltiples ramas y bifurcaciones del código, es posible crear una versión personal del código y luego fusionarlo con la rama principal maestra.

CLI

Ejecutaremos git puramente desde la CLI, o interfaz de línea de comandos. Es la ventana negra con texto blanco / verde que siempre vemos en las películas de hackers. Puede ser intimidante al principio, pero una vez que lo aprendas, CLI será solo otra herramienta en tu arsenal como desarrollador.

Notas:

¡Este sitio, Admin2Dev, se administra completamente en GitHub usando Git y CLI! Así que Git no es solo otra herramienta solo para rastrear los cambios, ¡también se puede utilizar para implementar sitios web y otro contenido en producción!
