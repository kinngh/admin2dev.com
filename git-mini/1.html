Introduction à Git et CLI

Qu'est-ce que Git

Git est un système de contrôle de version pour le suivi des modifications de code. Bien qu'il s'adresse aux programmeurs, il peut être utilisé pour suivre les modifications dans presque tous les types de fichier 1.

Pour utiliser git, nous avons besoin de fournisseurs git. Les trois fournisseurs de git les plus populaires sont

  • GitHub
  • GitLab
  • BitBucket

Alors, quelle est la différence entre un fournisseur git et git ? Pensez à cela comme à git est vos compétences de conduite. Vous pouvez conduire n'importe quelle voiture que vous voulez. Un fournisseur de git est une voiture. Vous utilisez donc vos compétences de conduite (git) pour utiliser une voiture (fournisseur de git).

Pourquoi Git est nécessaire

Versionnage

Git est destiné à suivre les modifications de votre code. Au fur et à mesure que les besoins évoluent, le code évolue également et parfois vous voulez remonter le temps et réutiliser un morceau de code qui a été supprimé ou suivre les changements effectués par d'autres membres de l'équipe.

Collaboration

Avec plusieurs branches et fourches du code, il est possible de créer une version personnelle du code, puis de le faire fusionner avec la branche principale principale.

CLI

Nous allons exécuter git uniquement à partir de l'interface de ligne de commande, ou interface en ligne de commande. C'est la fenêtre noire avec du texte blanc / vert que nous voyons toujours dans les films de pirates. Cela pourrait devenir intimidant au début, mais une fois que vous aurez compris, l'interface de ligne de commande ne sera qu'un autre outil dans votre arsenal en tant que développeur.

Note :

Ce site, Admin2Dev, est entièrement géré sur GitHub à l'aide de Git et CLI ! Git n'est donc pas seulement un autre outil pour suivre les changements, il peut également être utilisé pour déployer des sites Web et d'autres contenus en production !
