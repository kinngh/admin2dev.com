Verwenden von Git

Nun, auf die tatsächliche Verwendung von Git. Erstellen Sie zunächst einen Ordner namens git (ja, alle Kleinbuchstaben) auf Ihrem Desktop. Alles, was wir jetzt tun, wird in der Terminal- / Eingabeaufforderung sein, also los geht's!

Identität

Lassen Sie uns zunächst git sagen, wer wir sind, indem wir unseren Namen und unsere E-Mail definieren, zu denen alle unsere Commits beigetragen werden. Dies ist ein einmaliger Prozess. Geben Sie in einem Terminalfenster

git config --global user.name "FIRST_NAME LAST_NAME"
git config --global user.email "name@email.com"

There will be no "Name/Email added successfully" after you press enter, and that is completely normal. Alternatively, you can use the command git config --list --show-origin to show your current settings for git.

Klonen

Ein Klon eines Repos bedeutet, dass Sie eine Kopie davon auf Ihr Gerät herunterladen. Es gibt zwei Möglichkeiten, dies zu tun

Browser-Download

Sie können direkt eine ZIP-Datei des gesamten Repos herunterladen, indem Sie auf die Seite des Repos gehen und den Code verwenden.

Terminal

Klonen wir unser Markdown-Repo aus dem vorherigen Kapitel auf unserem Gerät. Geben Sie im Terminal cd ein und ziehen Sie Ihren Git-Ordner vom Desktop auf das Terminal und drücken Sie die Eingabetaste. (Erinnerung: CD lässt Sie Ihr Verzeichnis ändern)

Jeder Befehl für die Arbeit mit Git beginnt mit, *Drum Roll*git

First get your git repo link, which is different from your URL link by heading over to the clone section and copying the url ending with .git.

To clone the repo, in terminal type git clone <url>.

Jetzt wird es einen neuen Ordner in Ihrem Git-Ordner mit Ihrem Repo-Namen geben!

Ziehen

Pull ist es, Ihren Code aus dem Repo auf dem Server zu aktualisieren. Aktualisieren wir unsere README.MD-Datei in unserem Repository. Gehen Sie zu Ihrem Repository auf GitHub und fügen Sie den folgenden Code ein:

# Markdown Repo
This is updated text in my repo!

Now in your terminal, go inside your repo folder by typing cd folder-name and type git pull and open README.MD

Hinzufügen

Der Befehl git add ist der erste Befehl in einer Reihe von Vorgängen, die verwendet werden, um die Änderungen, die Sie an Dateien und Ordnern daraus vorgenommen haben, in einem Git-Repository zu speichern. Der Befehl sendet die von Ihnen vorgenommenen Änderungen an einen Stagingbereich. Sie können dann git commit verwenden, um diese Änderungen in das Haupt-Projektarchiv zu übertragen. Es gibt zwei Möglichkeiten, Dateien aus Ihrem lokalen Repository zur Staging hinzuzufügen.

  • git add path/to/file.extension
  • git add .

Sie können entweder bestimmte Dateien über ihren Pfad hinzufügen, oder Sie können einen Punkt / vollen Stopp verwenden, um jede einzelne Datei, die geändert wurde, zur Stagingumgebung hinzuzufügen.

Begehen

Before you commit changes, the git add command is required. The commit command takes files added from the git add stage and creates a snapshot with date / time stamp and changes made to the code in the project histroy timeline LOCALLY. To commit, we simply type git commit -m "Commit Message". Do not forget to add -m "Commit Message", because if you do, it will open a text editor in terminal depending on what platform you're using and it's a little difficult to work around. You can use multiple git add commands before you solidify it using a git commit.

Drücken

The git push command is used to upload our changes to the git server(in our case, to GitHub).

Übung

Üben wir schnell alles, was wir gelernt haben.

  • Nehmen Sie Änderungen an Ihrer README.MD-Datei vor. Es könnte ein Haiku sein, etwas über sich selbst, alles, was Sie mit dem Markdown-Format schreiben möchten. Fügen Sie einen Titel, einen Untertitel und eine Liste von Artikeln für Ihren Lebensmitteleinkauf später morgen hinzu. Die Welt ist dein Spielplatz
  • Erstellen Sie eine neue Datei in Ihrem Repo-Ordner und nennen Sie sie LINK.MD. Schreiben Sie hier mehr Text, der Inhalt spielt keine Rolle.
  • Verknüpfen Sie beide Dateien miteinander. Sie README.MD und LINK.MD sollten eine Schaltfläche "Gehe zu X" haben, die Sie, wenn Sie angeklickt haben, zur anderen Datei führt.
  • Fügen Sie in Ihrem Terminal beide Dateien hinzu und übertragen Sie sie in Ihr Repository.

Lösung

  • To link files, all you have to do is add [Go To Readme.md](README.MD).
  • So fügen Sie / übertragen Sie in Ihrem Repo-Ordner im Terminaltyp
git add .
git commit
