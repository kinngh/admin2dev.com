Utilisation de Git

Maintenant, sur l'utilisation réelle de git. Tout d'abord, créez un dossier appelé git(oui, toutes minuscules) sur votre bureau. Tout ce que nous faisons maintenant sera dans l'invite de commandes / terminale, alors allons-y !

Identité

Tout d'abord, disons à git qui nous sommes en définissant notre nom et notre e-mail auxquels tous nos commits seront contribués. Il s'agit d'un processus unique. Dans une fenêtre de terminal, tapez

git config --global user.name "FIRST_NAME LAST_NAME"
git config --global user.email "name@email.com"

There will be no "Name/Email added successfully" after you press enter, and that is completely normal. Alternatively, you can use the command git config --list --show-origin to show your current settings for git.

Cloner

Un clone d'un dépôt signifie que vous en téléchargez une copie sur votre appareil. Il y a deux façons de le faire

Téléchargement du navigateur

Vous pouvez télécharger directement un .zip de l'ensemble du dépôt en vous rendant sur la page du dépôt et en utilisant le code.

Terminal

Clonons notre dépôt de markdown du chapitre précédent sur notre appareil. Dans le terminal, tapez cd et faites glisser votre dossier git du bureau sur le terminal et appuyez sur Entrée. (Rappel : cd vous permet de changer votre répertoire)

Chaque commande pour travailler avec git commence par *roulement de batterie*git

Commencez par obtenir votre lien git repo, qui est différent de votre lien URL en vous rendant dans la section clone et en copiant l'url se terminant par .git.

To clone the repo, in terminal type git clone <url>.

Maintenant, il y aura un nouveau dossier dans votre dossier git avec votre nom de dépôt !

Tirer

Pull consiste à mettre à jour votre code à partir du dépôt dans le serveur. Mettons à jour notre fichier README.MD dans notre dépôt. Rendez-vous sur votre dépôt sur GitHub et collez le code suivant :

# Markdown Repo
This is updated text in my repo!

Now in your terminal, go inside your repo folder by typing cd folder-name and type git pull and open README.MD

Ajouter

La commande git add est la première commande d'une série d'opérations utilisées pour enregistrer dans un dépôt Git les modifications que vous avez apportées aux fichiers et aux dossiers à partir de ceux-ci. La commande envoie les modifications que vous avez apportées à une zone de transit. Vous pouvez ensuite utiliser git commit pour valider ces modifications dans le dépôt principal. Il y a deux façons d'ajouter des fichiers de votre dépôt local à la mise en scène.

  • git add path/to/file.extension
  • git add .

Vous pouvez soit ajouter des fichiers spécifiques par leur chemin d'accès, soit utiliser un point / point final pour ajouter chaque fichier qui a été modifié à l'environnement de transit.

S'engager

Before you commit changes, the git add command is required. The commit command takes files added from the git add stage and creates a snapshot with date / time stamp and changes made to the code in the project histroy timeline LOCALLY. To commit, we simply type git commit -m "Commit Message". Do not forget to add -m "Commit Message", because if you do, it will open a text editor in terminal depending on what platform you're using and it's a little difficult to work around. You can use multiple git add commands before you solidify it using a git commit.

Pousser

The git push command is used to upload our changes to the git server(in our case, to GitHub).

Pratique

Mettons rapidement en pratique tout ce que nous avons appris.

  • Apportez des modifications à votre fichier README.MD. Cela pourrait être un haïku, quelque chose sur vous-même, tout ce que vous voulez écrire en utilisant le format markdown. Ajoutez un titre, un sous-titre, une liste d'articles pour vos courses plus tard demain. Le monde est votre terrain de jeu
  • Créez un nouveau fichier dans votre dossier de dépôt et appelez-le LINK.MD. Écrivez plus de texte ici, le contenu n'a pas vraiment d'importance.
  • Liez vos deux fichiers ensemble. Vous README.MD et LINK.MD devraient avoir un bouton "Aller à X" qui, lorsqu'il est cliqué, vous mène à l'autre fichier.
  • Dans votre terminal, ajoutez les deux fichiers et validez-les dans votre dépôt.

Solution

  • To link files, all you have to do is add [Go To Readme.md](README.MD).
  • Pour ajouter / valider, alors que vous êtes dans votre dossier de dépôt, dans le type terminal
git add .
git commit
