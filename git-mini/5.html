Uso de Git

Ahora, en realidad usando git. Primero, crea una carpeta llamada git (sí, en minúsculas) en tu escritorio. Todo lo que hagamos ahora estará en el símbolo del sistema de terminal, ¡así que vamos!

Identidad

Primero, digamos a git quiénes somos definiendo nuestro nombre y correo electrónico a los que se contribuirán todos nuestros commits. Este es un proceso único. En una ventana de terminal, escriba

git config --global user.name "FIRST_NAME LAST_NAME"
git config --global user.email "name@email.com"

There will be no "Name/Email added successfully" after you press enter, and that is completely normal. Alternatively, you can use the command git config --list --show-origin to show your current settings for git.

Clonar

Un clon de un repositorio significa que está descargando una copia del mismo en su dispositivo. Hay dos maneras de hacer esto

Descarga del navegador

Puede descargar directamente un .zip de todo el repositorio dirigiéndose a la página del repositorio y usando el código.

Terminal

Clonemos nuestro repositorio de rebajas del capítulo anterior en nuestro dispositivo. En terminal, escriba cd y arrastre su carpeta git desde el escritorio en terminal y presione Intro. (Recordatorio: cd le permite cambiar su directorio)

Cada comando para trabajar con git comienza con, *rollo de tambor*git

Primero obtenga su enlace de repositorio git, que es diferente de su enlace URL dirigiéndose a la sección de clonación y copiando la URL que termina con .git.

Para clonar el repositorio, en terminal escriba git clone <url>.

¡Ahora, habrá una nueva carpeta en tu carpeta git con tu nombre de repositorio!

Tire

Pull es actualizar su código desde el repositorio en el servidor. Actualicemos nuestro archivo README.MD en nuestro repositorio. Diríjase a su repositorio en GitHub y pegue el siguiente código:

# Markdown Repo
This is updated text in my repo!

Now in your terminal, go inside your repo folder by typing cd folder-name and type git pull and open README.MD

Añadir

El comando git add es el primer comando de una serie de operaciones utilizadas para guardar en un repositorio Git los cambios que realizó en los archivos y carpetas del mismo. El comando envía los cambios que realizó a un área de ensayo. A continuación, puede usar git commit para confirmar esos cambios en el repositorio principal. Hay dos maneras de agregar archivos de su repositorio local a la preparación.

  • git add path/to/file.extension
  • git add .

Puede agregar archivos específicos por su ruta, o puede usar un punto / punto completo para agregar cada archivo que se cambió al entorno de ensayo.

Confirma

Before you commit changes, the git add command is required. The commit command takes files added from the git add stage and creates a snapshot with date / time stamp and changes made to the code in the project histroy timeline LOCALLY. To commit, we simply type git commit -m "Commit Message". Do not forget to add -m "Commit Message", because if you do, it will open a text editor in terminal depending on what platform you're using and it's a little difficult to work around. You can use multiple git add commands before you solidify it using a git commit.

Empuje

The git push command is used to upload our changes to the git server(in our case, to GitHub).

Práctica

Practiquemos rápidamente todo lo que hemos aprendido.

  • Realice cambios en su archivo README.MD. Podría ser un haiku, algo sobre ti mismo, cualquier cosa que quieras escribir usando el formato de rebaja. Agregue un título, subtítulo, una lista de artículos para sus compras de comestibles más tarde mañana. El mundo es tu patio de recreo
  • Cree un nuevo archivo en su carpeta de repositorios y llámelo LINK.MD. Escribe más texto aquí, el contenido realmente no importa.
  • Vincula ambos archivos. LÉAME.MD y LINK.MD deberían tener un botón "Ir a X", que cuando se hace clic, lo lleva al otro archivo.
  • En su terminal, agregue ambos archivos y envíelos a su repositorio.

Solución

  • To link files, all you have to do is add [Go To Readme.md](README.MD).
  • Para agregar / confirmar, mientras está en su carpeta de repositorio, en el tipo de terminal
git add .
git commit
