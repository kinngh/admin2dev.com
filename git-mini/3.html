Introduction à GitHub

GitHub est la plate-forme de contrôle de version et de collaboration la plus populaire sur Internet. Rendez-vous sur GitHub et créez un compte si vous ne l'avez pas déjà fait.

Interface utilisateur GitHub

  • Le panneau de gauche répertorie tous vos dépôts, publics et privés.
  • La zone du milieu contient toutes les mises à jour des dépôts, des personnes et des organisations que vous suivez sur GitHub, y compris tous les problèmes qui peuvent être ouverts et les mises à jour à leur sujet.
  • Le panneau de droite contient des suggestions de dépôt en fonction de votre historique de navigation.

Profil GitHub

Votre profil github est disponible sur github.com/username.

Votre profil affiche des informations vous concernant, vos référentiels, projets et packages que vous avez rendus publics, un tableau de contribution qui représente votre activité sur le site et un calendrier d'activité en dessous.

Dépôts

Un dépôt, ou dépôt en abrégé, représente un projet. Chaque dépôt sur github a son propre jeu de code, problèmes, demandes de tirage, actions, projets, wiki, sécurité, informations et paramètres.

  • Code : La base de code réelle de la branche que vous consultez. (Nous parlons plus en détail des branches dans le prochain chapitre).
  • Problèmes : Tous les problèmes avec le code, comme les erreurs d'exécution, sont consignés par les contributeurs pour les signaler au développeur.
  • Demandes de tirage : Les contributeurs peuvent cloner, apporter des modifications comme corriger des bogues et demander au propriétaire du dépôt de demander une fusion de leur base de code vers la base de code principale.
  • Actions : Les actions comprennent des pipelines CI/CD (Continuous Integration / Continuous Deployment) comme "Quand un nouveau commit est effectué, poussez-le vers les serveurs Heroku".
  • Projets : Un tableau d'organisation Trello pour le développeur afin de suivre les activités et les feuilles de route, entre autres choses.
  • Wiki : Publiez un wiki pour le dépôt.
  • Sécurité : Le développeur peut configurer des stratégies de sécurité pour les contributeurs afin de signaler les vulnérabilités de sécurité pour le dépôt.
  • Insights : Analyses sur le trafic, les clones git, etc.

You can mark someone else's repo with a Star or Watch

  • Star : C'est le bouton "J'aime ce dépôt" de GitHub. Les étoiles d'un utilisateur peuvent être vues à l'adresse https://github.com/username ? tab=étoiles .
  • Regarder : Semblable à un suivi, le bouton de surveillance vous permet d'être informé des modifications apportées au dépôt.

Nous approfondirons tous ces onglets plus tard dans le cours et explorerons comment les travailler en profondeur.
