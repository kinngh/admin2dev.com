Introducción a GitHub

GitHub es la plataforma de control de versiones y colaboración más popular en Internet. Diríjase a GitHub y regístrese para obtener una cuenta si aún no lo ha hecho.

Interfaz de usuario de GitHub

  • El panel izquierdo enumera todas sus repos, públicas y privadas.
  • El área media tiene todas las actualizaciones de repositorios, personas y organizaciones que sigues en GitHub, incluidos cualquier problema que pueda estar abierto, y actualizaciones sobre ellos.
  • El panel derecho tiene sugerencias de repositorios basadas en su historial de navegación.

Perfil de GitHub

Su perfil de github está disponible en github.com/username.

Su perfil muestra información sobre usted, sus repositorios, proyectos y paquetes que ha hecho públicos, un gráfico de contribuciones que representa su actividad en el sitio y una línea de tiempo de actividad debajo de él.

Depósitos

Un repositorio, o repositorio para abreviar, representa un proyecto. Cada repositorio en github tiene su propio conjunto de códigos, problemas, solicitudes de extracción, acciones, proyectos, wiki, seguridad, información y configuración.

  • Código: La base de código real de la sucursal que está viendo. (Hablamos más sobre ramas en el próximo capítulo).
  • Problemas: Todos y cada uno de los problemas con el código, como los errores en tiempo de ejecución, son registrados por los colaboradores para que el desarrollador lo notifique.
  • Pull Requests: Los colaboradores pueden clonar, hacer cambios como corregir errores y pedir al propietario del repositorio que solicite una fusión de su base de código a la base de código principal.
  • Acciones: Las acciones incluyen tuberías CI/CD (integración continua / implementación continua) como "Cuando se realiza una nueva confirmación, envíela a los servidores Heroku".
  • Proyectos: Una junta de organización similar a Trello para que el desarrollador realice un seguimiento de las actividades y hojas de ruta, entre otras cosas.
  • Wiki: Publica un wiki para el repositorio.
  • Seguridad: El desarrollador puede configurar cosas como políticas de seguridad para que los contribuyentes informen de las vulnerabilidades de seguridad para el repositorio.
  • Perspectivas: Análisis sobre el tráfico, clones de git, etc.

You can mark someone else's repo with a Star or Watch

  • Estrella: Este es el botón "Me gusta este repositorio" de GitHub. Las estrellas de un usuario se pueden ver en https://github.com/username? tab=estrellas .
  • Reloj: Similar a un seguimiento, el botón de reloj le permite ser notificado de los cambios en el repositorio.

Vamos a profundizar en todas estas pestañas más adelante en el curso y exploraremos cómo trabajar con ellas en profundidad.
