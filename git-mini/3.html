Einführung in GitHub

GitHub ist die beliebteste Plattform für Versionskontrolle und Zusammenarbeit im Internet. Gehen Sie zu GitHub und registrieren Sie sich für ein Konto, falls Sie es noch nicht getan haben.

GitHub-Benutzeroberfläche

  • Im linken Bereich sind alle Ihre öffentlichen und privaten Repos aufgeführt.
  • Im mittleren Bereich befinden sich alle Updates von Repos, Personen und Organisationen, denen Sie auf GitHub folgen, einschließlich aller Probleme, die möglicherweise offen sind, und Updates zu ihnen.
  • Das rechte Panel enthält Repo-Vorschläge, die auf Ihrem Browserverlauf basieren.

GitHub-Profil

Ihr Github-Profil ist unter github.com/username verfügbar.

Ihr Profil zeigt Informationen über Sie, Ihre Repositorys, Projekte und Pakete, die Sie veröffentlicht haben, ein Beitragsdiagramm, das Ihre Aktivitäten auf der Website darstellt, und einen Zeitplan der Aktivitäten darunter.

Speicher

Ein Repository, kurz Repo, stellt ein Projekt dar. Jedes Repo auf Github hat seinen eigenen Codesatz, Probleme, Pull-Requests, Aktionen, Projekte, Wiki, Sicherheit, Einblicke und Einstellungen.

  • Code: Die tatsächliche Codebasis des Zweigs, den Sie sich ansehen. (Wir sprechen im kommenden Kapitel mehr über Zweige).
  • Probleme: Alle Probleme mit dem Code, wie Laufzeitfehler, werden von den Mitwirkenden protokolliert, um sie dem Entwickler zur Kenntnis zu bringen.
  • Pull Requests: Mitwirkende können klonen, Änderungen vornehmen, wie Fehler beheben und den Besitzer des Repos bitten, eine Zusammenführung von ihrer Codebasis zur Hauptcodebasis anzufordern.
  • Aktionen: Zu den Aktionen gehören CI/CD-Pipelines (Continuous Integration / Continuous Deployment) wie "Wenn eine neue Übertragung vorgenommen wird, schieben Sie sie auf Heroku-Server".
  • Projekte: Ein Trello-ähnliches Organisationsboard für den Entwickler, um unter anderem Aktivitäten und Roadmaps zu verfolgen.
  • Wiki: Veröffentlichen Sie ein Wiki für das Repo.
  • Sicherheit: Der Entwickler kann Dinge wie Sicherheitsrichtlinien einrichten, damit Mitwirkende Sicherheitslücken für das Repository melden können.
  • Einblicke: Analysen zu Datenverkehr, Git-Clones usw.

You can mark someone else's repo with a Star or Watch

  • Stern: Dies ist GitHubs Schaltfläche "Ich mag dieses Repo". Die Sterne eines Benutzers können unter https://github.com/username? tab=Sterne.
  • Watch: Ähnlich wie bei einem Follow können Sie mit der Schaltfläche "Watch" über Änderungen im Repository benachrichtigt werden.

Wir werden später im Kurs ausführlich auf all diese Registerkarten eingehen und untersuchen, wie wir mit ihnen arbeiten können.
