Abschlag

The Markdown syntax is how we format text. A markdown file ends with the .md or .markdown extension. Before we get into it, let's make a new repository on GitHub.

Nachdem Sie sich angemeldet haben, klicken Sie auf die grüne Schaltfläche "Neu" auf der linken Seite, um ein neues Repository zu erstellen.

Geben Sie nun den Namen und die Details zu Ihrem neuen Repo ein. Stellen Sie sicher, dass "Dieses Repository mit einer README.MD initialisieren" aktiviert ist.Jetzt werden Sie mit Ihrem neuen Repo begrüßt! Klicken Sie auf die Datei "README.MD"Klicken Sie jetzt auf die Schaltfläche BearbeitenSie werden mit einem ähnlichen Bildschirm begrüßt, auf dem Sie Ihren Text eingeben können.Es gibt auch den Vorschauabschnitt, der Ihnen zeigt, was geändert, entfernt und in Ihrer Datei hinzugefügt wurde. Wir werden in einem späteren Kapitel mehr über Diffs erfahren. Im Moment lernen wir Markdown!

Überschrift

Headings range from 1 to 5, with 1 being the largest and 5 the smallest. To write a line of heading, we add # before the text.

Abschlag Ausgang
# Heading 1

Rubrik 1
## Heading 2

Rubrik 2
### Heading 3

Rubrik 3
#### Heading 4

Rubrik 4
##### Heading 5
Rubrik 5

Betonung

Um Text hervorzuheben, können wir fett, kursiv oder streiken.

Abschlag Ausgang
**bold** Fett gedruckt
_italics_ Kursiv
~strike~ Streik

Verzeichnisse

Markdown ermöglicht es Ihnen, drei Arten von Listen zu haben.

Bestellte Listen

Abschlag Ausgang
1. Eins
2. Zwei
3. Drei
  1. Eins
  2. Zwei
  3. Drei

Ungeordnete Listen

Abschlag Ausgang
- Eins
- Zwei
- Drei
  • Eins
  • Zwei
  • Drei

Einzüge und Kombination

Sortierte und ungeordnete Listen können kombiniert werden, zusammen mit dem Einzug mit Registerkarten, um verschachtelte Listen zu erstellen.

Abschlag Ausgang
1. Eins
2. Zwei
- Nest 1
- Nest 2
  1. Eins
  2. Zwei
    • Nest 1
    • Nest 2

Links

Markdown ermöglicht es, auf externe und interne (Repo-)Bilder und -Dateien zu verweisen.

Bilder

Die Syntax für das Hinzufügen von Bildern lautet![Image alt description](imageLink)

Intern

Abschlag Ausgang
![Logo](folder/image/a2d_logo.png)

Extern

Abschlag Ausgang
![Admin2Dev Logo](https://admin2dev.com/images/logo.png)

Dateien

Die Syntax für die Referenzierung von Dateien lautet[Description Name](file Link)

Intern

Abschlag Ausgang
[File Name](folder/file.txt) Dateiname

Extern

Abschlag Ausgang
[Batch Apex](https://github.com/Admin2Dev/apex-snippets/blob/master/batch/README.md) Chargenspitze

Code

Es gibt zwei Möglichkeiten, einen Textblock als Code in Markdown zu markieren.

Abschlag Ausgang
`einzeilig` single line
```
Multi
Linie
Code
```		 
multi
line
code

Tisch

Das Erstellen von Tabellen in Markdown kann manchmal verwirrend (und irritierend) werden, so dass es viel einfacher ist, einen Dienst wie Markdown Table Generator zu verwenden und den Code einzufügen.
