Markdown

The Markdown syntax is how we format text. A markdown file ends with the .md or .markdown extension. Before we get into it, let's make a new repository on GitHub.

Après vous être connecté, cliquez sur le bouton vert "Nouveau" à gauche pour créer un nouveau référentiel.

Remplissez maintenant le nom et les détails de votre nouveau dépôt. Assurez-vous que "Initialiser ce référentiel avec un README.MD" est coché.Maintenant, vous serez accueilli avec votre nouveau pension ! Cliquez sur le fichier "README.MD"Cliquez maintenant sur le bouton ModifierVous serez accueilli par un écran similaire à celui-ci où vous pourrez entrer votre texte.Il y a aussi la section aperçu qui vous montre ce qui a été modifié, supprimé et ajouté dans votre fichier. Nous allons en savoir plus sur les différences dans un chapitre ultérieur. Pour l'instant, apprenons le markdown !

Rubrique

Headings range from 1 to 5, with 1 being the largest and 5 the smallest. To write a line of heading, we add # before the text.

Markdown Sortie
# Heading 1

Rubrique 1
## Heading 2

Rubrique 2
### Heading 3

Rubrique 3
#### Heading 4

Rubrique 4
##### Heading 5
Rubrique 5

Accent mis sur

Pour mettre l'accent sur le texte, nous pouvons utiliser le gras, l'italique ou la grève.

Markdown Sortie
**bold** gras
_italics_ italique
~strike~ grève

Listes

Markdown vous permet d'avoir trois types de liste.

Listes commandées

Markdown Sortie
1. Un
2. Deux
3. Trois
  1. Un
  2. Deux
  3. Trois

Listes non ordonnées

Markdown Sortie
- Un
- Deux
- Trois
  • Un
  • Deux
  • Trois

Indentations et combinaison

Les listes ordonnées et non ordonnées peuvent être combinées, ainsi que l'indentation avec des onglets pour créer des listes imbriquées.

Markdown Sortie
1. Un
2. Deux
- Nid 1
- Nid 2
  1. Un
  2. Deux
    • Nid 1
    • Nid 2

Liens

Markdown permet de référencer des images et des fichiers externes et internes (repo).

Images

La syntaxe d'ajout d'images est![Image alt description](imageLink)

Interne

Markdown Sortie
![Logo](folder/image/a2d_logo.png)

Externe

Markdown Sortie
![Admin2Dev Logo](https://admin2dev.com/images/logo.png)

Fichiers

La syntaxe de référencement des fichiers est[Description Name](file Link)

Interne

Markdown Sortie
[File Name](folder/file.txt) Nom du fichier

Externe

Markdown Sortie
[Batch Apex](https://github.com/Admin2Dev/apex-snippets/blob/master/batch/README.md) Lot Apex

Code

Il y a deux façons de marquer un bloc de texte comme code dans markdown.

Markdown Sortie
`ligne unique` single line
```
multi
ligne
code
```		 
multi
line
code

Tableau

La création de tables dans markdown peut parfois devenir confuse (et irritante), il est donc beaucoup plus facile d'utiliser un service comme Markdown Table Generator et de coller le code.
