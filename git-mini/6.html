Collaboration et contribution

Contribuer aux pensions

Maintenant que nous connaissons les bases de l'utilisation de git, collaborons avec d'autres développeurs sur la plate-forme GitHub.

Repo à fourche

Rendez-vous sur Admin2Dev Learning Markdown Repo et en haut à droite, cliquez sur Fork. Pour Fork, un dépôt signifie créer une instance du dépôt pour votre usage, c'est différent du clonage d'un dépôt puisque celui-ci se télécharge du serveur sur notre appareil.

Notice how it says forked from Admin2Dev/learning-markdown right under the repo name on top left. What we have done at this point, is created an instance of the repo for ourselves. By doing this, we can work on this repo however we want without affecting the original repo.

Faisons quelques changements. Dans notre README.MD, ajoutons une ligne supplémentaire à la fin et validons-la. Vous pouvez utiliser le bouton d'édition sur GitHub ou cloner le dépôt sur votre machine et ajouter/commettre/pousser.

> May the Salesforce be with you!

Demande de tirage

Une demande de tirage d'une fourche avise le propriétaire du dépôt d'examiner les modifications que vous avez apportées et de l'ajouter au dépôt principal. Dans ce cas, je demande à Admin2Dev, propriétaire du dépôt learning-markdown, de fusionner mes modifications dans leur dépôt, afin que tous ceux qui visitent le dépôt voient cette version.

Cliquez sur la demande de tirage et soumettez une demande.

Vous remarquerez qu'il répertorie toutes les modifications apportées, ainsi qu'un écran qui compare les modifications qui ont été apportées. Dans ce cas, nous voyons une ligne verte avec notre jeu de mots dedans, indiquant que nous avons ajouté une nouvelle ligne dans le fichier README.MD. Allez-y et cliquez sur Créer une demande de tirage, puis expliquez quelles modifications ont été apportées. C'est particulièrement important parce que vous devez documenter les modifications que vous avez apportées à la base de code et pourquoi vous l'avez apportée.

Passons maintenant au côté d'Admin2Dev (maintenant appelé A2D). A2D reçoit une notification indiquant qu'une nouvelle demande de tirage a été ouverte par kinngh à leur dépôt de marquage d'apprentissage.

À l'ouverture de la notification, A2D est dirigé vers la page Demande de tirage où ils peuvent voir quelles modifications ont été apportées, les messages de validation, qui a effectué les validations et d'autres informations pertinentes.

Once they confirm the merge, the Pull Request will be closed and the status will be updated to Merged, and the code base was updated with the commit message Merge pull request #1 from kinngh/master. You can now safely delete the original fork you made.

Vous pouvez consulter les git repos ici :

  • kinngh/learning-markdown
  • Admin2Dev/learning-markdown

Contribuer.md

Beaucoup de projets Open Source auront un fichier appelé CONTRIBUTING.md dans leur dépôt qui décrit comment vous devriez contribuer. Cela comprend la garantie que les tests sont réussis (comme l'écriture de classes de test pour Apex), le code est formaté à l'aide d'un linter spécifique, entre autres directives. Vous pouvez consulter CONTRIBUTING.md d'Alamofire/Alamofire repo qui décrit comment poser des questions, signaler les problèmes, signaler les problèmes de sécurité, le code de conduite de base, etc.

C'est une bonne pratique de suivre les pointeurs du fichier CONTRIBUTING.md d'un dépôt avant d'ouvrir des problèmes ou de tirer des demandes.

Licences

Tous les morceaux de code sur GitHub ne sont pas disponibles pour un usage commercial, ou même open source d'ailleurs. Beaucoup de projets publient leur base de code sur GitHub à des fins de transparence, c'est pourquoi c'est toujours une bonne idée de regarder dans le fichier LICENSE.MD d'un dépôt et de le suivre réellement. GitHub fait un excellent travail de détection des licences et de présentation d'un résumé rapide de la façon dont le dépôt peut être utilisé, comme celui-ci à partir de plausible/analyses

Voici une liste de sites que vous pouvez lire en détail sur les licences :

  • OpenSource.org
  • Comprendre les licences de logiciel libre et open source sur support
  • Licence open source sur Wikipedia
